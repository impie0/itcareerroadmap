# SR20 · Technical Writer / Documentation Engineer
## Deep Dive: Career Pathways, Salary, Skills & Market Positioning

**Last updated:** April 30, 2026  
**Author:** IT Career Roadmap Project  
**Status:** Verified

---

## Overview

**Technical Writing** (TW) is a specialized discipline spanning three distinct market tiers, each with different compensation, skill expectations, and career trajectories:

1. **Corporate Technical Writer** — Pharma, manufacturing, legal, insurance, utilities. Stable, lower pay. $55–130K. Commodity skill.
2. **Enterprise/SaaS Product Documentation** — Salesforce, ServiceNow, Oracle, Microsoft. Moderate pay. $80–160K. Requires information architecture + user empathy.
3. **Developer Documentation Engineer (DevX)** — FAANG, devtools, open-source sponsors. High pay. $170–260K+. Requires docs-as-code, API expertise, build infrastructure.

The **2026 inflection point**: corporate TW salaries stagnant; developer documentation salaries rising (GitHub Copilot for docs, AI tooling, API-first architecture). Remote-first roles cluster in Categories 2 & 3.

---

## Career Bifurcation: Corporate vs. DevX (2026)

### Corporate Technical Writer Path
- **Employer type:** Pharmaceutical, medical devices, industrial equipment, legal, insurance, financial services, telecom, utilities.
- **Core work:** Regulatory compliance docs, user manuals, training materials, safety documentation, process documentation.
- **Pay ceiling:** $120–160K (rare). Median ~$95K (senior).
- **Salary growth:** 2–3% annually. Dictated by union scales or HR bands, not market demand.
- **Remote-friendly:** Moderate (some on-site, hybrid common).
- **Career mobility:** Lateral moves within industry; switching industries resets seniority.
- **AI impact (2026):** High—manual drafting of boilerplate manuals being automated. Emphasis shifting to compliance review + human authorization.
- **Skills shelf life:** 5–10 years. Style guides matter; tooling less.

### Developer Documentation / DevX Engineer Path
- **Employer type:** FAANG (Google, Apple, Amazon, Meta, Microsoft); devtools (GitHub, GitLab, HashiCorp, JetBrains, Stripe, Twilio, Auth0); startups (Series B+); open-source sponsors.
- **Core work:** API reference docs, SDK guides, getting-started flows, conceptual architecture, interactive tutorials, code examples, doc build pipelines, API documentation automation.
- **Pay ceiling:** $260K+. Median senior = $180–220K total comp (salary + stock). Staff/Principal = $200–300K.
- **Salary growth:** 15–25% with role changes; stock vesting 4–5 years.
- **Remote-friendly:** Very (often 100% remote; distributed teams).
- **Career mobility:** High—skills portable across FAANG. DevX engineers hired by startups at equity upside.
- **AI impact (2026):** Moderate-to-high. LLM-generated first drafts + human refinement workflow emerging. Competitive advantage: custom doc tooling + proprietary knowledge systems.
- **Skills shelf life:** 3–5 years. Hugo/Gatsby/Docusaurus cycles quickly; Git + shell scripting + API knowledge stable.

**Key insight:** A senior corporate TW retraining to DevX can expect a 50–120% pay bump, but faces skill ramp (6–12 months). Once inside a tech-native environment, growth compounds.

## Career Stages & Competencies

### Stage 1: Junior Technical Writer / Documentation Specialist (0–2 years)
**Salary:** $55–85K | **Title common at:** Larger enterprises, SaaS, tech startups.

**Responsibilities:**
- Write and edit technical content from subject-matter expert (SME) interviews.
- Maintain existing documentation sites (copy updates, link fixes, deprecation notices).
- Create internal onboarding / release-note documentation.
- Assist with information architecture (IA) for a single product area.
- Moderate style guide compliance within team.

**Required skills:**
- Excellent English (grammar, clarity, concision).
- Familiarity with **one** docs tool (Confluence, GitBook, or simple Markdown).
- Basic HTML / CSS reading ability.
- Version control basics (Git read-only; may not commit).
- Structured writing (numbered steps, warnings, callouts).

**Career moat:** Industry knowledge + domain writing (fintech TW vs. biotech TW command domain premium). Willing to learn tooling quickly.

**Exit/pivot:** Stay 18–24 months to build portfolio, then pivot to **Senior TW** (same company after promotion) or **DevX** (if retrain on docs-as-code).

---

### Stage 2: Technical Writer (2–7 years)
**Salary:** $80–130K | **Title common at:** Startups (Series A+), SaaS, enterprises with product-led growth.

**Responsibilities:**
- Owns documentation for 1–2 product lines end-to-end (architecture through release).
- Drives information architecture for new features (works with product, UX, engineering).
- Mentors junior writers (informal; not title-based).
- Establishes style consistency across cross-functional teams.
- Manages documentation sprints (2-week cycles, integrated with dev).
- Contributes to tool selection (Docusaurus vs. Hugo vs. custom build).
- API documentation ownership (OpenAPI spec → human-readable reference).

**Required skills:**
- **Docs-as-code:** Markdown, reStructuredText, or AsciiDoc authoring + Git workflows.
- **Information architecture:** Navigation, taxonomy design, search optimization.
- **Single-sourcing:** Content reuse patterns (includes, variables, conditional branches).
- **API documentation:** OpenAPI/Swagger parsing, Postman, endpoint examples.
- **Stakeholder management:** Extracting content from engineers + product teams.
- **Analytics literacy:** Heatmaps (Hotjar), navigation patterns, search logs.

**Specialization emerging:** Developer documentation vs. product documentation vs. knowledge-base engineering.

**Career moat:** IA expertise + tooling depth + the ability to unblock engineering teams (writer who grasps code architecture is rare).

**Exit/pivot:**
- Promote to **Senior TW** (leadership + strategy).
- Pivot to **Product Manager** (doc role is PM apprenticeship; many PMs start as technical writers).
- Pivot to **DevX / Documentation Engineering** (add CI/CD, automation, APIs).
- Lateral move to **Technical Marketing** (white papers, comparison guides, case studies).

---

### Stage 3: Senior Technical Writer (5–12 years)
**Salary:** $115–165K (corporate); $150–210K (FAANG/startup) | **Title variants:** Senior Technical Writer, Lead Technical Writer, Documentation Lead, Senior Documentation Engineer.

**Responsibilities:**
- Owns documentation strategy for entire product (or multiple products in portfolio).
- Builds and mentors a team of 2–5 writers.
- Information architecture review: establishes patterns, trains others, audits consistency.
- Drives doc tooling decisions (Docusaurus upgrade, search system, versioning strategy).
- Collaborates with Product, UX, Engineering on release cadence + doc schedule.
- Owns customer-facing API documentation + developer experience (DevX) strategy.
- Doc analytics + feedback loops: NPS for docs, search analytics, customer feedback triage.
- Participates in hiring, training, style-guide governance.

**Required skills:**
- All Stage 2 skills, plus:
- **Content strategy:** Long-term doc roadmap (quarterly/annual planning).
- **Team leadership:** Hiring, feedback, career development.
- **Tool evaluation + customization:** CI/CD pipelines, custom Jekyll/Hugo/Docusaurus themes, doc automation.
- **Accessibility:** WCAG 2.1 AA compliance, assistive technology testing.
- **Localization workflows:** Translation memory, glossary management, RTL/CJK considerations.
- **Advanced API docs:** GraphQL documentation, webhook design, auth flows, rate-limit docs.
- **Technical depth:** Code reading (Python, JavaScript, Go); understands architecture.

**Specialization solidified:** Developer documentation engineers vs. product documentation leaders vs. knowledge-base architects.

**Career moat:** Institutional knowledge of product + credibility with engineering + demonstrated ability to grow & retain writers.

**Exit/pivot:**
- Promote to **Staff / Principal** (rare; org must support it).
- Pivot to **Product Strategy** or **Head of Product** (next logical career step).
- Launch **independent consulting** (doc strategy for startups).
- Launch **documentation agency** (Docs Done Right, etc. models).

---

### Stage 4: Staff / Principal Documentation Engineer (8–15+ years)
**Salary:** $170–240K base; $240–350K TC (FAANG/devtools) | **Title variants:** Staff Documentation Engineer, Principal Technical Writer, Director of Documentation, Head of Documentation.

**Responsibilities:**
- Sets documentation standards + tooling strategy for entire org.
- Builds multi-platform doc systems (product docs + API + internal wiki + support KB).
- Drives company-wide content reuse architecture (shared components, glossaries, templates).
- Mentors senior writers + leads hiring for documentation org.
- Participates in company architecture decisions (API design affects doc burden; influencing both).
- Owns doc infrastructure: CI/CD pipelines, search indexing, analytics, i18n infrastructure.
- External voice: speaks at Write the Docs, publishes articles on doc strategy.
- Budget ownership: tools, training, hiring.

**Required skills:**
- All Stage 3 skills, plus:
- **Systems thinking:** Cross-org impact; sees docs as a product.
- **Infrastructure as code:** Terraform/CloudFormation for doc hosting; GitHub Actions workflows.
- **API design literacy:** Can review API endpoints for documentation feasibility.
- **Business acumen:** ROI of doc investments, cost justification, metrics that matter.
- **Advanced automation:** Custom doc generators, API mock servers, interactive tutorials (using Jupyter, Runkit, etc.).

**Career moat:** Rare skill; few documentation leaders truly understand infrastructure + content strategy together. Network within FAANG + devtools communities.

**Exit/pivot:**
- Promote to **VP of Developer Relations** or **Head of DevX** (next step, especially in devtools).
- Pivot to **VP of Product** (seen in startup context; doc leadership → product leadership common).
- **Freelance doc strategy consultant** (high rates: $200–400/hr).
- **Author / Educator** (Write the Docs community leadership; teaching at conferences).

## Sub-Specialties & Career Branches

### 1. Developer Documentation (DevDocs)
**Focus:** API reference, SDK guides, getting-started flows, code examples, conceptual architecture, troubleshooting.

**Employers:** FAANG, devtools (Stripe, Twilio, Auth0, GitHub), backend-as-a-service platforms.

**Salary:** Senior DevDocs engineer = $150–210K (FAANG: $200–350K TC).

**Skills:**
- OpenAPI / Swagger parsing + customization.
- API design review + versioning communication.
- Code example generation (multi-language: Python, Node.js, Go, Ruby, C#, etc.).
- Interactive docs (Postman docs, ReadMe, Stoplight).
- API mocking + sandbox environments.
- Changelog + deprecation communication.

**Tools:** Redoc, Swagger UI, ReadMe, Postman, Stoplight, Mintlify, Stripe-style custom sites.

**Books:** *Docs for Developers* (Bhatti, Corleissen, Lambourne, Nunez, Waters; Apress, free at https://docsfordevelopers.com/).

---

### 2. Product Documentation / User Guides
**Focus:** Help articles, user guides for SaaS/enterprise software, features walkthroughs, video tutorials, in-app guidance.

**Employers:** SaaS companies (Salesforce, ServiceNow, Atlassian, Figma, Slack, Notion), enterprise software, UX-first startups.

**Salary:** Senior product TW = $130–180K (FAANG SaaS: $180–250K TC).

**Skills:**
- Information architecture for end-users (not developers).
- Feature documentation from roadmap.
- Search optimization (SEO for internal search + Google).
- Video script writing + screen recording (Loom, Camtasia).
- Accessibility + inclusive design.
- Feedback loops (support tickets, heatmaps, in-doc rating).
- Single-sourcing (conditional docs for different user roles).

**Tools:** GitBook, Confluence, custom Docusaurus, HelpScout, Zendesk, Notion.

**Books:** *Strategic Writing for UX* (Torrey Podmajersky; O'Reilly). *The Product is Docs* (Splunk team; Routledge).

---

### 3. Documentation Engineering / DevOps for Docs
**Focus:** Documentation infrastructure, build pipelines, static site generation, versioning, search, analytics, i18n automation.

**Employers:** GitHub, GitLab, HashiCorp, Stripe (docs infrastructure team), AWS (doc tooling), open-source sponsors.

**Salary:** Documentation Engineer = $160–220K (FAANG: $240–350K TC). **Highest paying specialty in TW field.**

**Skills:**
- **Languages:** Python, Go, Bash, JavaScript/TypeScript.
- **Build tools:** Hugo, Gatsby, Docusaurus, Jekyll, Sphinx, Antora, Mkdocs.
- **CI/CD:** GitHub Actions, GitLab CI, Jenkins.
- **Search:** Algolia, Elasticsearch, Meilisearch configuration + tuning.
- **Versioning:** Multi-branch doc sites (v1, v2, v3 parallel).
- **i18n:** Crowdin, Lokalise integration.
- **Analytics:** Custom event tracking, Hotjar + Fullstory dashboards, Search analytics (Google).
- **Security:** S3/CloudFront for doc hosting, SSL, docs-in-airgap environments.
- **Container/Kubernetes:** Docker for reproducible doc builds.

**Tools:** GitHub Actions, Stripe's custom doc platform (open-source: Docusaurus), Hugo modules, Algolia (free tier for open-source).

**Conference:** Write the Docs Portland (dedicated DevOps track).

---

### 4. Technical Marketing Writing
**Focus:** White papers, case studies, comparison guides, competitive positioning, product marketing collateral.

**Employers:** DevTools marketing teams, B2B SaaS marketing, analyst relations, agencies (Moxie, Hotwire).

**Salary:** Technical Marketing Manager = $110–160K (senior: $160–200K). Lower than pure documentation roles but higher than pure marketing.

**Skills:**
- Audience segmentation (developer persona vs. decision-maker persona).
- Competitive intelligence + win/loss analysis.
- Thought leadership positioning.
- Case study structure + customer interview skills.
- Email marketing copywriting.
- Demand generation metrics.

**Transition path:** Senior Product TW → Technical Marketing Manager (common move; writers → marketers common as well).

---

### 5. Knowledge Base / Support Documentation Engineer
**Focus:** Self-service help articles, support content, FAQ, troubleshooting trees, internal knowledge base.

**Employers:** SaaS support teams (Zendesk, HubSpot), enterprise support, customer success, financial services (compliance docs).

**Salary:** KB Engineer = $70–120K (senior: $120–160K). Lowest-paid TW specialty, but often fastest entry point.

**Skills:**
- Support software (Zendesk, Freshdesk, Intercom, Help Scout).
- Search optimization (knowledge base discoverability).
- Content reuse + single-sourcing.
- Analytics (article views, search terms, support ticket reduction ROI).
- Video tutorials (Loom, Wistia).

**Transition path:** KB Engineer → Product Documentation (common jump; KB engineers understand search + user pain points).

## Tools, Platforms & Ecosystems

### Docs-as-Code Platforms
- **Docusaurus** (Meta/Facebook; React-based, popular for dev + product docs)
- **Hugo** (Go; fastest static site generator; large ecosystem)
- **MkDocs** (Python; minimal, great for API docs)
- **Sphinx** (Python; industry standard for libraries + RTD integration)
- **Antora** (AsciiDoc + component model; enterprise Java/XML community)
- **Gatsby** (React; modern, but heavier than Hugo)
- **Jekyll** (Ruby; GitHub Pages native; older but stable)
- **Mintlify** (modern, AI-enabled, VC-funded; rising star 2024–2026)

**Trend (2026):** Docusaurus + MkDocs dominate startups; Sphinx entrenched in Python ecosystem; Antora in regulated industries.

### API Documentation Platforms
- **Postman** (collections + documentation generator; enterprise focus)
- **ReadMe** (visual, interactive, popular with SaaS)
- **Stoplight** (OpenAPI-native; design + documentation)
- **Swagger UI / ReDoc** (open-source, API spec visualization)
- **Mintlify** (AI-augmented, code-to-docs, rising 2025–2026)
- **Stripe Docs** (custom, in-house standard; GitHub-hosted as inspiration)

**2026 note:** Mintlify's LLM-augmented doc generation (code comment → formatted docs) gaining adoption in startups; GitHub Copilot Docs in beta.

### Search & Analytics
- **Algolia** (SaaS search; free for open-source; $45–500/mo for commercial)
- **Meilisearch** (open-source alternative; lighter than Elasticsearch)
- **Elasticsearch** (enterprise search; complex but powerful)
- **Google Site Search** (deprecated 2017; use Google Custom Search Engine + GA4)
- **Hotjar** (heatmaps + session recording; $39–115/mo for docs use)
- **Fullstory** (session replay + analytics; expensive but fine-grained)
- **Google Analytics 4** (free, event-based; becoming standard)

### Localization & i18n
- **Crowdin** (cloud translation management; $99–400/mo)
- **Lokalise** (similar; $80–600/mo)
- **Phrase** (enterprise; $2000+/mo)
- **Gettext** (open-source; command-line; widely used)

### Design & Content Management
- **Figma** (design collaboration; increasingly used for documentation wireframes)
- **Confluence** (Atlassian wiki; enterprise standard, but increasingly replaced by GitHub + Markdown)
- **Notion** (modern wiki; popular for internal docs; now being challenged by custom Markdown + GitHub)
- **GitBook** (modern, beautiful, GitHub-integrated)
- **Coda** (competitor to Notion; emerging for doc + project management fusion)

## Education, Certs & Communities

### Formal Certifications

#### 1. Society for Technical Communication (STC) Certified Professional Technical Communicator (CPTC)
- **Issuer:** STC (https://www.stc.org/).
- **Levels:** Foundation, Practitioner, Expert.
- **Cost:** Foundation exam ~$350; Practitioner ~$400; Expert ~$450.
- **Prereq:** 1 year TW experience for Foundation; 3 years for Practitioner; 5 years for Expert.
- **Timeframe:** Typically 3–6 months prep.
- **Recognition:** Moderate in USA; low in Europe/APAC. Employers rarely require, but certification boosts salary negotiation ~3–5%.
- **Study resources:** STC official study guides; practice exams (STC sells).

#### 2. Tekom Certified Technical Communicator (Germany / Europe)
- **Issuer:** Tekom Deutschland (https://www.tekom.de/).
- **Levels:** Foundation (tekom Basis), Practitioner (tekom Experte), Specialist certifications (API Docs, SaaS, etc.).
- **Cost:** €300–600 per exam.
- **Recognition:** Very high in Germany, Austria, Switzerland; moderate in UK; low in USA.
- **Common among:** European TW candidates; prerequisite for some pharma/automotive roles (DIN EN 82079-1 compliance).

#### 3. Write the Docs Learning Path
- **Issuer:** Write the Docs (https://www.writethedocs.org/).
- **Format:** Free + paid courses (Udemy partnerships, annual conference scholarships).
- **Cost:** Free webinars; paid workshops €99–299.
- **Recognition:** High credibility within devtools + open-source communities; not formal credential.
- **Popular courses:** "Introduction to Technical Writing," "API Documentation," "Information Architecture."

#### 4. Google Technical Writing Courses
- **Issuer:** Google (https://developers.google.com/tech-writing).
- **Format:** Free, self-paced online; courses for developers + engineers.
- **Cost:** Free.
- **Recognition:** High within Google ecosystem + startups; not a formal cert (no exam).
- **Coverage:** Plain-language writing, active voice, technical writing basics.

## Style Guides & Standards

### Industry Style Guides (Rank by Adoption)
1. **Microsoft Writing Style Guide** (https://learn.microsoft.com/en-us/style-guide/welcome/)
   - **Scope:** Plain language, tone, formatting, terminology.
   - **Best for:** SaaS, enterprise, product documentation.
   - **Adoption:** Microsoft, Apple, many startups.
   - **Notable rules:** Active voice, 2nd person ("you"), short sentences, minimize jargon.

2. **Google Developer Documentation Style Guide** (https://developers.google.com/style)
   - **Scope:** Developer documentation, API docs, code samples.
   - **Best for:** DevDocs, API documentation teams.
   - **Adoption:** Google, many open-source projects, startups.
   - **Notable rules:** Third person allowed for APIs; example-driven; emphasis on accuracy.

3. **Apple Style Guide** (proprietary, available to registered developers; https://developer.apple.com/design/human-interface-guidelines/)
   - **Scope:** UI terminology, tone, accessibility.
   - **Best for:** iOS/macOS documentation, Apple ecosystem.
   - **Notable rules:** Precision over brevity; Apple-specific terminology (e.g., "tap" not "click").

4. **The Chicago Manual of Style** (17th ed., University of Chicago Press)
   - **Scope:** Grammar, punctuation, citations, formatting.
   - **Best for:** Academic / regulated (legal, pharma) documentation.
   - **Adoption:** Textbooks, legal documents, formal reports.

5. **STC Handbook** (Society for Technical Communication; curated best practices)
   - **Scope:** Industry-neutral technical writing fundamentals.
   - **Best for:** Regulated industries, compliance documentation.

6. **DIN EN 82079-1** (ISO/IEC standard; EU/Germany)
   - **Scope:** Safety information and instructions for use (IFU).
   - **Best for:** Medical devices, pharma, machinery (regulatory requirement in EU).
   - **Adoption:** Mandatory in Germany, Austria, Switzerland for regulated products.

## Career Entry Paths

### Path A: English / Journalism → Technical Writer
**Typical timeline:** 6–12 months ramp.

**Process:**
1. Build Markdown + Git portfolio (write blog posts in Jekyll or Hugo).
2. Study Microsoft Style Guide + "Modern Technical Writing."
3. Create sample technical documentation (create 2–3 portfolio pieces: API guide, user guide, getting-started tutorial).
4. Apply to junior TW / Doc Specialist roles at startups + SaaS.
5. First role: $55–70K; ramp to mid-level ($80–100K) in 2–3 years.

**Advantage:** Natural writers often excel at clarity + audience empathy; disadvantage is technical ramp (learning to read code, API design).

---

### Path B: QA Tester / Support Engineer → Technical Writer
**Typical timeline:** 3–6 months ramp (less learning curve than journalism path).

**Process:**
1. Leverage existing product knowledge + customer empathy (QA already understands feature edge cases).
2. Learn docs tools (Markdown, Hugo, Git).
3. Move laterally: QA → Documentation role (often same company; internal mobility).
4. First TW role: $70–85K; faster growth to senior ($130K+) due to product fluency.

**Advantage:** Domain knowledge + testing mindset = faster ramp. Disadvantage: often move from "QA TW" → "Product TW" (rarely to DevX without deliberate reskilling).

---

### Path C: Software Engineer → Technical Writer (Rare but High-Paying)
**Typical timeline:** 12–18 months (difficult transition).

**Process:**
1. Engineer growing tired of coding; interest in documentation / developer experience.
2. Internal move or external hire: engineering manager sponsors "doc engineer" role.
3. Learn docs-as-code + information architecture (come in with code literacy).
4. First doc eng role: $100–130K (premium paid for engineering background).

**Advantage:** Rare combination (engineers who can write). Highest ceiling: $240–350K TC as Staff doc engineer.

**Disadvantage:** Imposter syndrome common; engineers often struggle with audience empathy (write too technical for end-users). Requires explicit mentorship on writing for non-expert audiences.

## Salary Negotiation & Market Dynamics (2026)

### Negotiation Toolkit
1. **Data sources (order by credibility):**
   - Levels.fyi (anonymized, real comp; best for FAANG/devtools).
   - Glassdoor (high volume; includes non-tech TW salary dragging average down).
   - STC Annual Salary Survey (community-specific; biased toward established members).

3. **For corporate (non-tech) roles:**
   - Ceiling ~$130K senior. Negotiating space limited; focus on title + annual raise %.
   - Domain expertise (pharma TW, finance TW) commands 5–15% premium over generic TW.

4. **For DevX roles (FAANG/devtools):**
   - Massive negotiation space. $50K+ difference between offer + negotiated comp common.
   - Stock/RSU is usually negotiable (especially post-IPO for late-stage startups; pre-IPO is fixed).
   - Signing bonus: negotiate +$20–50K for senior hires (counteroffer to competitors).

## 2026 Market Inflection Points

### AI-Augmented Documentation (2026 Reality)
1. **LLM-generated first drafts:** GitHub Copilot for Docs, Mintlify AI, OpenAI API integration tools now generate documentation from code comments + specifications. **Adoption:** 20–30% of startups actively trialing.

2. **Human review mandatory:** Generated docs require human expert review (accuracy, completeness, tone). Job displacement risk is **low**; workflow shifts from "write from scratch" to "edit AI draft."

3. **Docs-as-data:** Emphasis on structured, machine-readable documentation (JSON Schema, OpenAPI, GraphQL SDL) for AI parsing. Technical writers must understand data modeling basics.

4. **New specialization emerging:** "Documentation QA Engineer" (reviews AI-generated docs, tests code examples, ensures accuracy). Salary: $80–120K (growth specialty).

### DocOps Maturity
- **Investment:** Companies allocating dedicated budget to doc infrastructure (CI/CD, search, analytics, i18n).
- **Role expansion:** Senior TW increasingly includes "doc ops" responsibilities (builds doc pipelines, manages tooling, tracks metrics).
- **Hiring:** "Documentation Engineer" title becoming more common; traditional "Technical Writer" title declining (especially in dev tools).

### API-First Architecture Acceleration
- **Impact:** Every product now has an API; developer documentation critical to adoption.
- **Salary consequence:** DevDocs specialists command highest premium in TW field.
- **Example:** Stripe's famous documentation investment (2012–present) drove adoption > than marketing spend. Copycat strategy widespread now.

### Documentation as a Product (DaaP)
- **Trend:** Documentation itself is a competitive differentiator (e.g., Stripe vs. rivals; same payment processing, but docs are the moat).
- **Consequence:** TW roles elevated from "cost center" to "revenue driver."
- **Investment:** Companies budgeting $500K–2M+ annually for documentation operations (staff + tooling).
- **Hiring:** Aggressive recruiting for doc team leads + senior doc engineers (especially at Series B+ startups).

## Skill Acquisition Roadmap (DIY, 6–12 months)

### Month 1–2: Foundations
1. **Writing fundamentals:** Read *Modern Technical Writing* (Andrew Etter; 4 hours). Study Microsoft Style Guide (3 hours).
2. **Markdown:** Learn Markdown syntax (4 hours). Write 5 blog posts in GitHub-hosted Jekyll/Hugo (16 hours).
3. **Git basics:** Learn Git workflow (commit, push, PR) via GitHub Skills (4 hours).
4. **Create first portfolio piece:** Write a user guide for a project you know (API, tool, open-source project). Target: 2000 words. (16 hours)

**Deliverable:** GitHub repo with 5 blog posts + 1 user guide.

### Month 3–4: Docs-as-Code Tooling
1. **Pick a static site generator:** Docusaurus (React, popular) or Hugo (Go, minimal). Deploy via GitHub Pages.
2. **Build a practice docs site:** Document a fake SaaS product (include getting-started, feature guide, troubleshooting, FAQ). Target: 25+ pages. (24 hours)
3. **Learn Markdown advanced:** Includes, variables, conditional branching, front matter (4 hours).
4. **Search + analytics:** Set up Algolia (free tier) for search; add Google Analytics 4 (4 hours).

**Deliverable:** Live docs site on GitHub Pages + portfolio link.

### Month 5–6: API Documentation
1. **Learn OpenAPI/Swagger:** Read OpenAPI spec, study Postman collection basics (8 hours).
2. **Write API documentation:** Document 3 hypothetical APIs (REST endpoints, auth, rate limits, examples in 2+ languages) (16 hours).
3. **Interactive docs:** Explore Postman docs, ReadMe, or Stoplight (4 hours).
4. **Deploy:** Host API docs on Vercel or Netlify (2 hours).

**Deliverable:** 3 API docs examples + live hosted API reference.

### Month 7–8: Information Architecture & Analytics
1. **Study IA principles:** Read chapter on IA in "Docs for Developers" (4 hours). Study Stripe docs structure (why is it organized that way?).
2. **Redesign your practice docs site:** Reorganize for better findability (taxonomy, navigation, search optimization) (8 hours).
3. **Analytics deep dive:** Set up Hotjar heatmaps; analyze your own site (where do users get stuck?). Make improvements (8 hours).
4. **Accessibility:** WCAG 2.1 AA audit of your docs site (4 hours); fix issues.

**Deliverable:** Restructured docs site + analytics report.

### Month 9–10: CI/CD & Automation
1. **GitHub Actions:** Build a simple CI/CD pipeline: commit → build site → deploy to GitHub Pages (8 hours).
2. **Versioning:** Set up multi-branch docs (main / v1 / v2). Deploy different versions (8 hours).
3. **Spell-check & link verification:** Add GitHub Actions workflow for automated checks (4 hours).
4. **Internationalization (i18n):** Set up Crowdin for pseudo-localization testing (4 hours).

**Deliverable:** Automated docs build pipeline.

### Month 11–12: Specialization & Capstone
**Pick one specialization (choose one path):**
- **Path A (DevDocs):** Build a realistic API documentation site for a public API (Stripe, GitHub, Twitter API) with custom styling, examples, debugging guide (24 hours).
- **Path B (Product Docs):** Write complete user guide + video tutorials for a SaaS product (free tier). Include heatmap analysis + UX improvements (24 hours).
- **Path C (DevX/Tooling):** Build custom Docusaurus theme + docs site generator plugin (JavaScript/TypeScript). Deploy to npm (24 hours).

**Deliverable:** Capstone project + write-up (1000-word reflection on what you learned).

**End-state portfolio:** GitHub profile + 8–10 polished portfolio pieces + live docs site + capstone. Ready for junior TW interviews.

## Common Mistakes & Pitfalls

### Mistake 1: Over-engineering Documentation Tools
**Problem:** Choosing Sphinx because it's "enterprise," only to struggle with reStructuredText syntax. Or building custom static site generator instead of using Hugo.

**Solution:** Start with Docusaurus or Hugo (biggest ecosystems, most examples). Optimize tooling *after* you have content + users.

---

### Mistake 2: Writing Without User Research
**Problem:** Creating comprehensive documentation nobody reads; low search rankings; users still calling support.

**Solution:** Gather user feedback early: heatmaps (Hotjar), support tickets, feedback widget on docs. Let data guide reorganization.

---

### Mistake 3: Ignoring Accessibility (WCAG 2.1)
**Problem:** Beautiful docs that exclude screen-reader users. Accessibility lawsuits increasing 2024–2026.

**Solution:** Make WCAG 2.1 AA compliance non-negotiable. Test with axe DevTools, screen reader (NVDA, JAWS, VoiceOver). Budget 10% of time for remediation.

---

### Mistake 4: Not Investing in Style Guide Consistency
**Problem:** Each writer uses different terminology, formatting, tone. Docs feel disjointed.

**Solution:** Establish style guide early (even a 5-page reference is better than nothing). Use automated linting (Vale for markdown, write-good for plain language).

---

### Mistake 5: Letting Documentation Decay Post-Launch
**Problem:** v1.0 shipped, documentation published, then ignored. Feature updates ship without doc updates. Users confused.

**Solution:** Documentation is maintenance work. Budget: 20% of engineers' time for doc updates post-launch. Own docs in dev sprints, not as afterthought.

---

### Mistake 6: Fabricating Citations (Citation Rule)
**Problem:** Citing "studies show" without source. Claiming salary data from nowhere.

**Solution:** Every claim requires a source. If unsourced, omit. See Citation Rule (§ Introduction).

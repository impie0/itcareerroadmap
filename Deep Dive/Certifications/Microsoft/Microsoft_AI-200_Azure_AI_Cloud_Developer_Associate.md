## About

The Azure AI Cloud Developer Associate certification validates the ability to build, integrate, and deploy AI-augmented cloud solutions on Microsoft Azure. Launched in beta in April 2026, AI-200 replaces and expands upon AZ-204 to cover modern AI workloads including containerized AI services, vector-enabled databases (PostgreSQL with pgvector, Cosmos DB), retrieval-augmented generation (RAG), Copilot Studio integration, event-driven AI pipelines, and Azure OpenAI services. This certification is aimed at backend developers and cloud engineers who are transitioning to AI-focused application development.

## Topics covered

The AI-200 exam blueprint covers six main domains of AI cloud development:

- **Develop Azure AI Foundry agents and applications** — Build agentic solutions using Azure AI Foundry, orchestrate multi-step AI workflows, integrate Azure OpenAI, implement guardrails and content filtering.
- **Develop solutions with Copilot Studio** — Create conversational AI and process-mining experiences via Copilot Studio, integrate with backend Azure services, publish and monitor copilots at scale.
- **Implement retrieval-augmented generation (RAG) and grounding** — Design vector-indexed knowledge bases, implement hybrid search, ground LLM responses with proprietary data, optimize retrieval patterns for accuracy and latency.
- **Use Azure AI services for real-world use cases** — Integrate Azure OpenAI, Azure AI Search, Document Intelligence, Speech Services, Language Services, Computer Vision, and other cognitive APIs into application workflows.
- **Implement security, governance, and compliance** — Configure managed identity and Entra ID authentication, manage secrets with Azure Key Vault, implement role-based access control (RBAC), audit AI model usage, ensure data residency and encryption.
- **Deploy AI applications to Azure compute** — Package AI solutions for Azure Functions (serverless), App Service, Azure Container Apps, Azure Kubernetes Service (AKS), configure observability with Application Insights and OpenTelemetry, implement health checks and scaling policies.

*Source:* [Course AI-200T00-A: Develop AI Cloud Solutions on Azure](https://learn.microsoft.com/en-us/training/courses/ai-200t00)

## Skills validated

*Technical capabilities and services specifically tested by the AI-200 exam.*

- **Azure AI Foundry** — Agent orchestration, model configuration, workflow deployment, monitoring and evaluation.
- **Copilot Studio** — Conversational design, topic management, copilot publishing, integration with backend APIs and services.
- **Azure OpenAI Service** — Prompt engineering, deployment management, model configuration, token usage optimization.
- **Vector databases** — PostgreSQL with pgvector, Cosmos DB for NoSQL vector search, semantic indexing, hybrid search (keyword + semantic).
- **Retrieval-Augmented Generation (RAG)** — Knowledge base indexing, prompt grounding, response evaluation, citation and traceability.
- **Azure AI Services** — Document Intelligence (form and document parsing), Speech Service (speech-to-text, text-to-speech), Language Service (NER, sentiment, summarization), Computer Vision, AI Search (full-text + semantic).
- **Azure compute services** — Azure Functions (serverless event handlers, durable functions, triggers), App Service (containerized and non-containerized deployments), Container Apps (Dapr state management, secrets, scaling), Azure Kubernetes Service (Helm, service mesh basics).
- **Event-driven architecture** — Service Bus (message queues, topics), Event Grid (event routing), Event Hubs (streaming ingestion), trigger patterns with Azure Functions.
- **Security and identity** — Entra ID (formerly Azure AD) managed identity, role-based access control (RBAC), Azure Key Vault (secret and certificate management), network security groups, TLS/mTLS configuration.
- **Observability** — Application Insights (metrics, logs, distributed tracing), OpenTelemetry instrumentation, custom health checks, performance monitoring.
- **Deployment and CI/CD** — Azure DevOps (pipelines, artifact repositories), GitHub Actions, Azure Container Registry, blue-green and canary deployments, infrastructure-as-code (Bicep, ARM templates).

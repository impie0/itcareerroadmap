# Microsoft Azure AI Engineer Associate (AI-102)

**`AI-102`** · ● **Retiring (June 30, 2026)** · **Associate** · _Microsoft_

> **Critical notice:** This exam retires on June 30, 2026 at 11:59 PM Central Standard Time. The replacement exam is **AI-103** (Azure AI App and Agent Developer Associate), which launches June 2026 and shifts focus from Azure AI service integration toward generative AI applications and autonomous agent development. AI-102 holders will retain their credential indefinitely; the exam retires but the certification does not expire.

## About

The Microsoft Azure AI Engineer Associate certification (AI-102) validates intermediate-level expertise in designing, building, and deploying AI solutions on Microsoft Azure. Launched in January 2021, it became the standard credential for software developers, cloud engineers, and AI specialists integrating Azure Cognitive Services (now Azure AI Services), Azure Machine Learning, Azure OpenAI, and knowledge mining tools into production applications. The exam covers six domains: planning/managing Azure AI solutions, implementing decision support, computer vision, natural language processing, knowledge mining, and generative AI. As of May 2026, Microsoft is retiring AI-102 in favor of AI-103, which emphasizes generative AI applications and autonomous agent development using Azure AI Foundry—a strategic shift reflecting the market move toward agentic systems. Candidates have until June 30, 2026 to earn AI-102; those holding the certification retain it indefinitely.

## Topics covered

The AI-102 exam blueprint covers six domains with the following weight distribution:

- **Plan and Manage an Azure AI Solution (15–20%)** — Select appropriate Azure AI services for solution requirements; plan resource deployment; implement content moderation and safety controls; manage authentication, costs, and monitoring; apply responsible AI principles.

- **Implement Decision Support Solutions (10–15%)** — Build machine learning pipelines; train and evaluate models; use Azure Machine Learning for regression, classification, and clustering; deploy models as web services; implement AutoML for rapid prototyping.

- **Implement Azure AI Vision Solutions (15–20%)** — Use Azure Computer Vision API for image classification, object detection, optical character recognition; deploy Form Recognizer for document intelligence; implement facial recognition; apply ethical constraints.

- **Implement Natural Language Processing Solutions (15–20%)** — Build solutions with Azure Language Service for sentiment analysis, entity extraction, key phrase recognition; implement Azure Translator for multilingual support; deploy named entity recognition (NER); handle language-specific edge cases.

- **Implement Knowledge Mining Solutions (10–15%)** — Extract structured data from unstructured sources using Azure AI Search; implement indexing and retrieval pipelines; use Document Intelligence for form and table extraction; build knowledge bases and Q&A systems.

- **Implement Generative AI Solutions (10–15%)** — Integrate Azure OpenAI Service (GPT models); implement prompt engineering and prompt flows; design retrieval-augmented generation (RAG) architectures; apply safety guardrails; optimize costs and latency.

- Integrate Azure Cognitive Services (Computer Vision, Language Service, Translator) into applications
- Design and deploy Azure Machine Learning pipelines for classification and regression
- Implement content moderation and safety controls (content filtering, bias detection)
- Build Azure AI Search solutions for knowledge mining and semantic retrieval
- Deploy and manage Azure OpenAI Service endpoints with cost optimization

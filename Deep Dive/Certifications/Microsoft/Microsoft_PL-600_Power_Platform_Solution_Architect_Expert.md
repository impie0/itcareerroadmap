## About

The PL-600 is Microsoft's expert-level certification for Power Platform Solution Architects. Candidates demonstrate mastery of designing enterprise-scale solutions using Power Apps, Power Automate, Power BI, and related Power Platform components. Designed for functional consultants (PL-200), developers (PL-400), or equivalent professionals with hands-on experience who are ready to lead architecture and implementation decisions at scale. **Critical deadline: this exam retires June 30, 2026**, making it one of the final assessment opportunities for current candidates.

## Domain context — Architecture / Low-Code

Low-code and no-code platforms are increasingly central to digital transformation, enterprise process automation, and rapid solution delivery. Power Platform serves mid-to-large organisations needing citizen-developer velocity without sacrificing governance or integration complexity. Solution Architects at this level bridge business requirements, technical constraints, compliance, and scalability — a role that commands significant salary premium and career upside.

## Topics covered

From the official exam blueprint, PL-600 assesses three weighted domains:

- **Perform solution envisioning and requirement analysis** (35–40%)
  - Stakeholder discovery, business process analysis, data mapping, feasibility assessment
  
- **Architect a solution** (40–45%)
  - Security architecture, integration patterns, app and automation design, scalability, compliance, governance
  
- **Implement the solution** (15–20%)
  - Configuration, customisation, deployment pipelines, testing, user adoption

*Source:* [PL-600 Study Guide ↗](https://learn.microsoft.com/en-us/credentials/certifications/resources/study-guides/pl-600)

## Common skills at Architecture · Expert

*Shared competencies for enterprise-grade solution architecture roles.*

- **Enterprise security architecture** — identity, data governance, network segmentation, least-privilege access
- **Scalability and performance tuning** — load testing, caching, query optimisation, component reuse patterns
- **Integration design** — synchronous vs. asynchronous patterns, API gateways, event-driven architecture, data consistency
- **Governance and compliance** — audit trails, data residency, regulatory frameworks (GDPR, SOC 2, etc.)
- **Stakeholder alignment and communication** — translating business needs into technical design, risk assessment
- **Vendor evaluation and TCO analysis** — build-vs-buy, migration pathways, cost-benefit trade-offs

## Skills validated

*Specific technologies and methodologies tested by PL-600.*

- **Power Apps architecture** — canvas apps, model-driven apps, portals, component library design, scalability
- **Power Automate design** — cloud flows, RPA, process advisor, integration patterns, error handling
- **Power BI and analytics** — data models, semantic layer design, governance, row-level security (RLS)
- **Dataverse** — table design, relationships, business rules, plugins, security roles
- **Integration patterns** — connectors, custom APIs, webhooks, on-premises data gateways
- **Security and governance** — licensing, Application Lifecycle Management (ALM), DevOps pipelines
- **Dynamics 365 integration** — Sales, Customer Service, Field Service, Finance and Operations
- **AI Builder** — model building, orchestration, automated document processing
- **Performance and scalability** — throttling, batch operations, delegation, query optimisation

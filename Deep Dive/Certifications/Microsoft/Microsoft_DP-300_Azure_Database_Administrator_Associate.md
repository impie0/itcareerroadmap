# Microsoft Azure Database Administrator Associate

**`DP-300`** · ● **Active** · **Associate** · _Microsoft_

> The Azure Database Administrator implements and manages the operational aspects of cloud-native and hybrid data platform solutions built on Azure SQL services and SQL Server. DP-300 is an associate-level certification requiring 1–2 years of hands-on database administration experience. No formal prerequisites, but T-SQL and SQL Server/Azure SQL knowledge strongly recommended. Launched as the successor to legacy on-premises DBA tracks; current as of April 2026.

## About

DP-300 is Microsoft's associate-level certification for database administrators managing Azure SQL solutions. Launched in 2019 and updated April 2026, it targets professionals transitioning from on-premises SQL Server or database roles into cloud-managed SQL environments. The exam validates hands-on ability to plan data platform resources, implement security controls, monitor and optimize database performance, automate administrative tasks, and architect high-availability and disaster-recovery solutions using Azure SQL Database, SQL Managed Instance, and hybrid SQL Server on Azure. Unlike foundational DP-900, this is an operational/practitioner cert — assumes working knowledge of T-SQL, relational databases, and Azure infrastructure. Renewal is straightforward (free assessment annually).

## Topics covered

The exam blueprint organizes topics across five weighted domains:

- **Plan and implement data platform resources** (20–25%) — Azure SQL Database vs. Managed Instance vs. SQL Server on Azure VM; elastic pools; resource allocation; deployment options; migration strategies
- **Implement a secure environment** (15–20%) — authentication (Azure AD, SQL auth); encryption at rest and in transit (TDE, Always Encrypted); network isolation (VNets, service endpoints, private endpoints); firewall rules; auditing and compliance
- **Monitor, configure, and optimize database resources** (20–25%) — performance monitoring (Azure Monitor, Query Store, DMVs); indexing strategies; query tuning; wait statistics; resource utilization; alerting
- **Configure and manage automation of tasks** (15–20%) — SQL Agent (on-premises) vs. Automation Account (Azure); backup and restore automation; maintenance plans; Update Management; runbooks
- **Plan and configure a high availability and disaster recovery (HA/DR) environment** (20–25%) — failover groups; geo-replication; Always On Availability Groups; restore strategies; RTO/RPO design; backup retention

*Source:* [Official exam study guide ↗](https://learn.microsoft.com/en-us/credentials/certifications/resources/study-guides/dp-300)

## Skills validated

*Cert-specific — what this exam actually tests.*

- Azure SQL Database provisioning, configuration, and scaling (elastic pools, DTU vs. vCore)
- SQL Managed Instance architecture and hybrid connectivity
- T-SQL administration (users, roles, permissions, backup/restore scripts)
- Azure SQL Database security (Azure AD integration, firewall, TDE, Always Encrypted)
- Performance tuning (indexes, query execution plans, DMV queries, Azure Monitor)
- Backup and restore automation (full, differential, transaction log, long-term retention)
- Disaster recovery design (active geo-replication, failover groups, RTO/RPO targets)
- Azure Automation runbooks and SQL Agent jobs for scheduled maintenance
- Network isolation (VNets, private endpoints, service endpoints, NSGs)
- Compliance and auditing (SQL Audit, Advanced Data Security, Data Classification)

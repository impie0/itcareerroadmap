## About

DP-700 is Microsoft's flagship certification for data engineers using the **Microsoft Fabric** platform, released in March 2025 as a complete replacement for the retired DP-203 (Azure Data Engineer Associate). Unlike DP-203, which covered separate Azure services (Synapse, Data Factory, Databricks), DP-700 validates expertise in Fabric's unified, SaaS-first approach to data pipelines, lakehouse architecture, and real-time analytics. The exam targets professionals responsible for designing, building, and optimizing enterprise-scale data solutions in Fabric.

## Topics covered

The exam is organized into three equally weighted objectives (30–35% each):

- **Implement and manage analytics solutions in Microsoft Fabric (30–35%)**
  - Design and create Fabric workspaces and manage capacity
  - Manage data engineering assets (Notebooks, Spark jobs, pipelines)
  - Configure monitoring, security, and governance controls
  
- **Ingest and transform data (30–35%)**
  - Create data ingestion pipelines (Data Factory–style, now in Fabric)
  - Transform data using Spark, SQL, and Fabric notebooks
  - Design dimensional and fact tables (star schema, medallion architecture)
  - Handle incremental loads and slowly changing dimensions (SCD)
  
- **Monitor and optimize (30–35%)**
  - Query performance tuning (Fabric SQL warehouse, Lakehouse)
  - Cost optimization and capacity management
  - Data quality checks and anomaly detection
  - Troubleshoot pipeline failures and data lineage

## Skills validated

*What DP-700 specifically tests — distinct from the shared Associate-level data skills above.*

- **Microsoft Fabric workspace and capacity architecture**
- **Lakehouse design and management** (one-logical-copy principle, delta-parquet format)
- **Fabric Data Engineering UI** — Notebooks, Pipelines, Dataflows Gen2
- **Apache Spark on Fabric** — PySpark jobs, language bindings
- **Fabric SQL Warehouse** — T-SQL, performance tuning, materialized views
- **Data ingestion** — ADLS Gen2 connectors, Dataflow Gen2, API-based sources
- **Real-time analytics** — Real-Time Hub, Streaming, KQL (Kusto Query Language)
- **Monitoring and diagnostics** — Fabric admin portal, Capacity Metrics, job logs
- **Data governance and security** — Row-level security (RLS), workspace roles, item permissions
- **Cost optimization** — Capacity utilization, auto-scaling, compute-on-demand

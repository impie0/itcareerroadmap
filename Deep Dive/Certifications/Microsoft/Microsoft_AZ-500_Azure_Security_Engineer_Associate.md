## About

The Azure Security Engineer Associate (AZ-500) validates your ability to implement, manage, and monitor security for resources in Azure, hybrid, and multi-cloud environments. This Associate-level certification is designed for IT professionals and security engineers responsible for securing Azure infrastructure, identities, data, and operations. As of August 31, 2026, AZ-500 will retire and be replaced by SC-500 (Cloud and AI Security Engineer), which expands coverage to include AI security posture and governance alongside traditional cloud security. Organizations using Azure should plan migrations to SC-500 before the retirement deadline.

## Topics covered

The AZ-500 exam is weighted across four core security domains, reflecting real-world Azure security responsibilities:

- **Secure Identity and Access (15–20%)** — Microsoft Entra ID (Azure AD) configuration, role-based access control (RBAC), conditional access policies, multi-factor authentication (MFA), managed identities, service principals, and hybrid identity synchronization.
- **Implement Platform Protection (20–25%)** — Network security groups (NSGs), Azure Firewall, DDoS Protection, Web Application Firewall (WAF), virtual network segmentation, and bastion hosts.
- **Secure Compute, Storage, and Databases (20–25%)** — Azure Key Vault, encryption at rest and in transit, confidential computing, container security, Azure Container Registry, SQL/Cosmos DB security, and storage account hardening.
- **Manage Security Operations (30–35%)** — Microsoft Defender for Cloud, Microsoft Sentinel, threat detection, security alerts, incident response workflows, vulnerability management, and compliance monitoring.

*Source:* [AZ-500 Study Guide (Microsoft Learn) ↗](https://learn.microsoft.com/en-us/credentials/certifications/resources/study-guides/az-500)

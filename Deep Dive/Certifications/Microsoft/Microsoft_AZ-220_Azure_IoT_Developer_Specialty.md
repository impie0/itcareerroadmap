## About

The Azure IoT Developer Specialty certification validated professionals' ability to implement, deploy, and manage Internet of Things (IoT) solutions on Microsoft Azure. Released in February 2019 as a specialty-level credential, it targeted developers building cloud and edge components of IoT systems, including device connectivity, cloud-to-device and device-to-cloud messaging, data processing, and security. The certification was retired on July 31, 2023, as Microsoft shifted focus toward role-based paths spanning developer, architect, administrator, and data analyst roles. No direct replacement exam was announced; professionals currently pursuing IoT expertise are directed to combination paths across multiple Azure certifications (AZ-104, AZ-204, AZ-305) or to Azure Data Engineer Associate (DP-203) for data-stream and analytics components.

## Skills validated

*Concrete technologies and competencies this exam assessed, distinct from shared "Common skills" above.*

- Azure IoT Hub architecture, device messaging, custom endpoints, and routing
- Azure IoT Hub Device Provisioning Service (DPS) for at-scale enrollment
- Azure IoT Edge runtime and module development (containerized workloads)
- Device-to-cloud and cloud-to-device message patterns and protocols
- Azure Stream Analytics for real-time telemetry processing
- Azure Time Series Insights for time-series data analysis
- Azure Digital Twins for IoT asset modeling and simulation
- Microsoft Defender for IoT (device threat detection and compliance)
- Azure Sphere for microcontroller-grade OS and security
- Device twin and module twin query and update operations
- MQTT and AMQP protocol configuration for IoT devices
- Certificate-based and symmetric-key device authentication
- RBAC and shared-access-key authorization for IoT Hub access
- Cosmos DB and Azure Blob Storage integration for IoT data persistence
- IoT Edge gateway deployment and offline capabilities
- Azure Functions for serverless edge processing

## About

MS-102 is Microsoft's Expert-level certification for Microsoft 365 administrators, launched in September 2023 as a single-exam replacement for the previous two-exam path (MS-100 + MS-101, both retired September 30, 2023). Unlike its predecessor, MS-102 consolidates all core M365 admin competencies into a single exam, reducing study burden while raising the cognitive bar. It serves as the primary credential for professionals responsible for deploying, configuring, and managing Microsoft 365 tenants, hybrid identity environments, security orchestration via Defender XDR, and enterprise compliance via Purview. The certification is gated behind Associate-level prerequisites (MD-102, MS-200, or SC-300), reflecting its intermediate-to-expert target audience.

## Topics covered

The exam blueprint reflects four major administrative functions, each weighted by cognitive complexity:

| Domain | Weight | Key Topics |
|---|---|---|
| Deploy and manage a Microsoft 365 tenant | 25–30% | Tenant creation and configuration, multi-geo data residency setup, subscription and license management, admin role delegation (RBAC), device management policies, Microsoft 365 compliance center setup, billing and usage reporting |
| Implement and manage Microsoft Entra identity and access | 25–30% | Hybrid identity sync (Azure AD Connect, cloud sync), federated identity (ADFS, SAML, OAuth), conditional access policies and risk-based authentication, Privileged Identity Management (PIM), application registration and SSO integration, guest and B2B access |
| Manage security and threats by using Microsoft Defender XDR | 30–35% | Defender for Endpoint (EDR), Microsoft 365 Defender (unified alert management), threat analytics and investigation, incident response workflows, attack chain visualization, vulnerability management, antiphishing and malware policies |
| Manage compliance by using Microsoft Purview | 10–15% | Data lifecycle governance and retention policies, eDiscovery workflows, sensitivity labels and information protection, data loss prevention (DLP) rules, records management, regulatory compliance alignment (GDPR, HIPAA, SOC 2, industry-specific frameworks) |

## Common skills at Endpoint/Identity/M365 · Expert

Shared competencies across Expert-level M365, identity, and endpoint roles — not specific to MS-102 alone, but prerequisites for its exam objectives:

- Microsoft Entra ID (formerly Azure AD) architecture design: hybrid sync (Azure AD Connect, cloud sync), federation (ADFS, Okta integration), and identity proofing at enterprise scale
- Tenant governance and subscription strategy: multi-tenant architectures, licensing models, cost optimization, and admin role delegation
- Conditional Access and Zero Trust: policy design, risk-based authentication, device compliance, and adaptive access controls
- Privileged Identity Management (PIM): JIT access, approval workflows, and role activation audit trails
- Microsoft 365 Defender orchestration: alert triage, incident response, threat hunting, and continuous posture improvement
- Data governance: DLP policies, retention schedules, eDiscovery workflows, and regulatory compliance mapping (GDPR, HIPAA, SOC 2)
- PowerShell scripting for M365 automation: bulk user/group management, reporting, and service configuration across Exchange, Teams, SharePoint
- Modern workplace productivity: OneDrive, Teams, SharePoint governance, and organizational change management

## Skills validated

This exam explicitly tests the following concrete technologies and frameworks—not high-level concepts, but operational depth:

- **Microsoft 365 tenant architecture:** multi-geo setup, hybrid environments, subscription strategy, admin role delegation
- **Microsoft Entra ID (Azure AD):** hybrid synchronization (Azure AD Connect, cloud sync), ADFS federation, SAML/OAuth, application integration
- **Conditional Access & Zero Trust:** policy design, device compliance, risk-based authentication, legacy auth blocking, passwordless (Windows Hello, FIDO2)
- **Privileged Identity Management (PIM):** JIT access activation, approval workflows, role assignment audits, activation time limits
- **Microsoft 365 Defender XDR:** Defender for Endpoint (EDR), Defender for Cloud Apps (CASB), email/collaboration threat detection, incident response workflow
- **Microsoft Purview compliance:** data retention and lifecycle policies, eDiscovery hold/export workflows, sensitivity labels, DLP rule creation
- **Information Protection:** sensitivity label enforcement, content marking, encryption policy application
- **Licensing and cost:** SKU assignment, license optimization, usage reporting, subscription management
- **PowerShell scripting:** bulk user/group operations, Exchange Online cmdlets, Teams management, SharePoint administration
- **Regulatory alignment:** audit logs, compliance reporting, mapping to GDPR / HIPAA / SOC 2 / industry-specific frameworks

## How to use this guide

1. **Starting point:** If you're new to M365 administration, start with one of the Associate prerequisites (MD-102, MS-200, or SC-300) first.
2. **Study path:** Microsoft Learn (free official modules) + one paid course (Pluralsight/A Cloud Guru) + practice exams (MeasureUp/Whizlabs) + book (Exam Ref MS-102).
3. **Study time:** Budget 8–12 weeks for someone with prior Associate-level certification + 1–2 years hands-on M365 experience. New to M365? Allow 3–4 months minimum.
4. **Exam readiness:** Score 85%+ consistently on paid practice exams before booking the real exam.
5. **Career trajectory:** MS-102 is a stepping stone to Solutions Architect, Enterprise Administrator, or Security Operations roles. Use it as a credential to transition laterally into compliance, identity, or security teams within the same organization.

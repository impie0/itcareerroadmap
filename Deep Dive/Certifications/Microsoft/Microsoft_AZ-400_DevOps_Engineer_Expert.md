## About

The Microsoft Certified: DevOps Engineer Expert (AZ-400) is a professional-level certification for engineers who design and implement DevOps practices, continuous integration/continuous delivery (CI/CD) pipelines, infrastructure automation, and instrumentation strategies on the Microsoft Azure platform. It is the highest-level certification in the Azure DevOps track, building on foundational Azure administration or development skills. The exam was significantly updated on April 24, 2026, to reflect evolving DevOps practices including Site Reliability Engineering (SRE) strategies and modern instrumentation.

---

## Domain context — DevOps/Automation

DevOps fuses development and operations practices to automate software delivery, reduce lead time, improve reliability, and enable continuous feedback. On Azure, DevOps engineers implement version control strategies, build and release pipelines, automated testing, infrastructure-as-code, monitoring, and incident response. This certification validates expertise across the full CI/CD spectrum—source control, artifact management, deployment automation, observability, and security gates.

## Topics covered

1. **Define and implement continuous integration** (20–25%)
   - Design and implement Azure Pipelines
   - Manage build agents and build infrastructure
   - Manage code repositories and version control
   - Configure build triggers and pipeline parameters

2. **Define and implement a continuous delivery and release management strategy** (10–15%)
   - Design deployment strategies
   - Implement automated releases and approvals
   - Create and manage release pipelines
   - Manage deployment environments

3. **Manage source control** (10–15%)
   - Design source control strategies
   - Implement branching and merge strategies
   - Configure Git repositories
   - Integrate source control with CI/CD pipelines

4. **Facilitate communication and collaboration** (10–15%)
   - Configure Azure Boards for work tracking
   - Integrate monitoring and feedback loops
   - Promote knowledge sharing and transparency
   - Design communication workflows

5. **Develop a security and compliance plan** (10–15%)
   - Implement dependency scanning
   - Configure security gates in pipelines
   - Manage secrets and credentials
   - Ensure compliance and audit logging

6. **Develop a Site Reliability Engineering (SRE) strategy** (5–10%)
   - Design reliability targets and SLOs
   - Implement chaos engineering principles
   - Plan incident response and recovery
   - Define on-call and escalation procedures

7. **Develop an instrumentation strategy** (5–10%)
   - Design monitoring and alerting
   - Configure Application Insights
   - Implement distributed tracing
   - Collect and analyze telemetry

## About

The **SC-300** certifies hands-on expertise in administering Microsoft Entra ID (formerly Azure AD) and identity platforms within enterprise cloud and hybrid environments. Released in 2023, it is the primary Associate-level identity certification for professionals managing user lifecycles, access controls, authentication policies, and application integrations across Microsoft 365, Azure, and on-premises infrastructure. SC-300 is a practical, lab-heavy exam designed for identity engineers, M365 administrators, and access control engineers who implement real-world identity solutions. It is neither theoretical (like SC-900) nor architectural (like SC-100), but rather focused on operational execution.

## Topics covered

The SC-300 exam blueprint is organized into four domains with the following weights:

- **Implement an identity management solution (25–30%)**
  - Create, configure, and manage user and guest identities in Microsoft Entra ID
  - Configure identity lifecycle management with Lifecycle Workflows
  - Manage Azure AD Connect Sync and cloud sync for hybrid scenarios
  - Implement device identity and join strategies
  - Manage administrative units and role-based access control (RBAC)

- **Implement an authentication and access management solution (25–30%)**
  - Configure multi-factor authentication (MFA) and authentication methods
  - Implement Conditional Access policies and device-based policies
  - Manage passwordless sign-in (Windows Hello for Business, FIDO2, Microsoft Authenticator)
  - Implement and manage Azure AD Password Protection and self-service password reset
  - Configure session lifecycle and sign-in risk policies

- **Implement access management for applications (10–15%)**
  - Register and manage application identities in Microsoft Entra
  - Implement OAuth 2.0 / OIDC authorization flows
  - Configure app roles, delegated permissions, and application permissions
  - Manage service principals, managed identities, and application proxies
  - Implement single sign-on (SSO) for enterprise applications

- **Plan and implement identity governance (25–30%)**
  - Manage entitlements with access reviews and access packages (Entra ID Governance)
  - Implement privileged identity management (PIM) for administrative roles and resources
  - Configure Microsoft Entra Verified ID for decentralized identity claims
  - Implement threat protection and risk remediation
  - Manage B2B external identities and guest access policies

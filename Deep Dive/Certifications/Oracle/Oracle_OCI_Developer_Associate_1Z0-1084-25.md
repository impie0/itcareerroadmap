## About

The Oracle Cloud Infrastructure 2025 Developer Professional certification validates advanced skills in designing, building, deploying, and securing cloud-native applications on Oracle's public cloud platform. Released in 2025, this professional-level certification is designed for software engineers, architects, and DevOps professionals who build serverless functions, microservices, containerized workloads, and event-driven systems using OCI's developer services. Passing earns the badge and demonstrates mastery of OCI Functions, API Gateway, OKE (Kubernetes), Streaming, Events, and integration with OCI's broader infrastructure and observability stack.

## Topics covered

Exam blueprint per Oracle's official learning path (2025):

- **Cloud-Native Fundamentals** (15%) — 12-factor app methodology, microservices architecture, distributed systems principles, cloud-native design patterns
- **Cloud-Native Applications and Containerization** (20%) — Docker, container images, Oracle Cloud Infrastructure Registry (OCIR), Oracle Kubernetes Engine (OKE), container networking, service mesh, Helm charts, image scanning and security
- **Leveraging Serverless Technologies** (25%) — Oracle Functions (Fn), API Gateway, OCI Streaming, OCI Events, OCI Queue, function packaging and dependencies, event-driven architecture, integration patterns
- **Infrastructure-as-Code (IaC) and Automation** (10%) — Terraform, OCI Resource Manager, Infrastructure-as-Code best practices, automation pipelines
- **Identity and Access Management (IAM) for Developers** (5%) — Application authentication (Signature Version 4, OAuth 2.0), API authentication, secrets management with OCI Vault
- **Testing and Securing Cloud-Native Applications** (10%) — Unit testing, integration testing, security practices, OCI Vault for secret management, mTLS, certificate management, CORS policies
- **Monitoring & Troubleshooting Cloud-Native Applications** (10%) — OCI Monitoring (metrics), OCI Logging (structured logging), OCI Application Performance Monitoring (APM), observability best practices, tracing and debugging
- **Developer Services Integration** (5%) — Autonomous Database integration, Oracle Database Cloud Service, MySQL Database Service, integration with core OCI services (Compute, Storage, Networking)

*Source:* [Oracle MyLearn Developer Professional 2025 Learning Path ↗](https://mylearn.oracle.com/ou/learning-path/become-an-oci-developer-professional-2025/147839)

## Skills validated

*Cert-specific — what this exam actually tests, distinct from the shared "Common skills" above.*

- Oracle Functions (Fn) — function packaging, dependencies, invocation patterns, event-driven triggers
- API Gateway — API creation, management, authentication (OAuth 2.0, custom CA/mTLS), rate limiting, CORS, request/response transformation
- Oracle Kubernetes Engine (OKE) — cluster deployment, workload management, networking, service mesh integration, Helm
- Oracle Container Infrastructure Registry (OCIR) — image management, image scanning, vulnerability assessment
- OCI Streaming — Kafka-compatible streaming service, topic/partition management, producer/consumer patterns
- OCI Events — event patterns, rules, event routing to Functions, notifications, and other targets
- OCI Queue — asynchronous message queuing, poison queue handling
- OCI Resource Manager — Terraform-based infrastructure automation, stack management
- Infrastructure-as-Code (IaC) with Terraform for OCI resources
- Application Performance Monitoring (APM) — tracing, span analysis, service dependency mapping
- OCI Monitoring and Logging — metrics, logs, alarms, observability dashboards
- OCI Vault — secrets management, certificate authority, key management
- Autonomous Database integration with cloud-native applications
- MySQL Database Service and Oracle Database Cloud Service deployment
- Security best practices for cloud-native apps (RBAC, encryption, secrets rotation, mTLS)
- 12-factor application methodology and microservices patterns
- Container security scanning and compliance in OCI Registry
- CI/CD pipeline integration with OCI DevOps

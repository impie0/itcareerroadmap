## About

The Splunk O11y Cloud Certified Metrics User (SOCCMU) is an associate-level certification demonstrating foundational competency in metrics-driven observability using Splunk's Observability Cloud platform. Candidates prove ability to deploy the OpenTelemetry Collector, ingest and visualize metrics, configure dashboards and alerts, and troubleshoot issues using metrics. Introduced in 2022–2023 as part of Splunk's shift toward observability-focused certifications (following Cisco's 2023 acquisition of Splunk), this exam targets SREs, DevOps engineers, platform engineers, and reliability practitioners new to metrics-based monitoring.

---

## Domain context — Observability / APM / Metrics

Real-time visibility into distributed systems through metrics, logs, traces, and profiling. Splunk Observability Cloud is a commercial APM and observability platform competing with Datadog, New Relic, Elastic, and vendor-neutral tooling (Prometheus, Grafana, OpenTelemetry). This certification validates metrics-focused competency within a managed SaaS ecosystem.

## Topics covered

- **Getting Metrics Into Splunk Observability Cloud** – OpenTelemetry concepts, OTel Collector deployment (Linux, Kubernetes, Windows), receiver/processor/exporter configuration, agent vs. gateway modes
- **Metrics Concepts and Fundamentals** – Gauge, counter, histogram, summary metric types; metric cardinality; rollups and aggregation; time-series data principles
- **Monitoring Using Built-In Content** – Dashboard consumption, pre-built detectors and dashboards, infrastructure monitoring templates, navigating the UI
- **Visualizing Metrics Data** – Dashboard design, chart types (time-series, heatmap, single-value), metric scoping, filtering, drill-down behavior
- **Analytics and Insights** – Using the MetricsQL-like query language, analytics functions, time-window comparisons (hourly, daily, weekly), trend identification
- **Alerting and Detectors** – Creating and customizing detectors, alert thresholds, notification routing, escalation policies, PagerDuty integration basics
- **Splunk Observability Cloud Ecosystem** – Relationship to logs (Splunk Enterprise), APM agents, team collaboration, permissions model

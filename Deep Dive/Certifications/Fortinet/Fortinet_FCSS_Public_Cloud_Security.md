# Fortinet Certified Solution Specialist — Public Cloud Security

**`FCSS_PUBCLOUDSEC-7.6`** · ● **Active** · **Professional** · _Fortinet_

> **Certification naming transition (2026):** Fortinet is transitioning from "FCSS" (Certified Solution Specialist) nomenclature to "NSE" (Network Security Expert) levels 1–8 effective **July 15, 2026**. The FCSS Public Cloud Security certification will rebrand to **NSE 7 — Cloud Security**. This exam is currently valid under the FCSS brand; after July 15, 2026, it will be designated as NSE 7. Both naming conventions are transitionally valid through the date of rebranding.

## About

The Fortinet Certified Solution Specialist — Public Cloud Security exam validates expertise in designing, deploying, monitoring, and troubleshooting Fortinet security solutions in public cloud environments (AWS, Azure, GCP, OCI). This Professional-tier certification focuses on multi-cloud security architectures, cloud-native threat prevention, and integrated Fortinet cloud products including FortiGate Virtual, FortiCNAPP (Cloud Native Application Protection Platform), FortiCWP (Cloud Workload Protection), and CASB (Cloud Access Security Broker). Introduced as FCSS in 2024–2025 and rebranding to NSE 7 effective July 15, 2026, this credential targets cloud security engineers, solutions architects, and infrastructure engineers managing hybrid and multi-cloud deployments. Holders typically possess 2+ years of cloud security or networking experience and study for 4–8 weeks before exam.

## Domain context — Security / Cloud

Public cloud infrastructure security (AWS, Azure, GCP, OCI) combined with vendor-specific threat prevention and compliance. Cloud-native security engineering — distinct from traditional datacenter security — emphasizes Infrastructure-as-Code (IaC), ephemeral resources, API-driven security, and cloud-native workload protection. FortiGate VM deployments, cloud-centric CNAPP/CASB tooling, and multi-cloud federation architectures validate security across hyperscale platforms.

## Topics covered

The exam assesses the following competencies (per FCSS Public Cloud Security official blueprint):

- **Cloud Infrastructure & Platform Security** (~12%) — AWS/Azure/GCP/OCI architecture, IaaS security models, cloud-native vs. traditional firewall deployment, platform-specific security features (e.g., Azure Bicep, AWS CloudFormation)
- **FortiGate Virtual in Public Cloud** (~18%) — FortiGate VM deployment in AWS/Azure/GCP/OCI, licensing for cloud instances, scaling policies, high-availability failover in cloud environments, cost optimization
- **Cloud Network Security & SDN Connectors** (~15%) — AWS Transit Gateway, Azure Virtual Hub, GCP VPC peering, SD-WAN cloud connectivity, software-defined networking (SDN) integration with FortiGate, network segmentation
- **FortiCNAPP & Cloud Workload Protection** (~14%) — Cloud Native Application Protection Platform (CNAPP), workload segmentation, runtime protection, vulnerability scanning, compliance monitoring across cloud resources
- **FortiCWP & Container Security** (~12%) — FortiCWP (Cloud Workload Protection), container and microservices security, Kubernetes security policies, image scanning, compliance checks
- **Cloud Access Security Broker (CASB) & API Security** (~10%) — CASB deployment, API threat prevention, data exfiltration detection, shadow IT visibility, multi-SaaS governance
- **Automation & Infrastructure-as-Code** (~10%) — Terraform, Ansible, AWS CloudFormation, Azure Bicep, automation frameworks for cloud provisioning, policy-as-code
- **Monitoring, Logging & Incident Response** (~7%) — Cloud-native logging (CloudWatch, Azure Monitor, Cloud Logging), centralized log aggregation, threat hunting in cloud, incident response playbooks

_Source:_ [Fortinet FCSS Public Cloud Security Blueprint ↗](https://training.fortinet.com/local/staticpage/view.php?page=library_public-cloud-security)

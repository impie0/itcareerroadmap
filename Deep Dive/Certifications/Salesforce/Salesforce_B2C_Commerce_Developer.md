## About

The Salesforce Certified B2C Commerce Developer credential validates expertise in designing and implementing full-stack e-commerce storefronts using Salesforce B2C Commerce (SFCC). Originally Demandware, acquired by Salesforce in 2016 and rebranded to B2C Commerce in 2018, this platform powers storefronts for global brands in retail, fashion, beauty, and luxury goods. The exam requires 3–6 months hands-on experience with cartridge development, SFRA customization, ISML templating, JavaScript controllers, and integrations via OCAPI. Candidates typically progress from entry-level JavaScript development or full-stack web roles into specialized B2C Commerce roles; the certification is a gateway to mid-level and senior developer positions in the e-commerce ecosystem.

## Topics covered

- **B2C Commerce Architecture & Data Model** (10–15%)
  - Site organization, catalogs, products, inventory, pricing, customer profiles
  - Data model relationships and custom attributes
  
- **Storefront Reference Architecture (SFRA)** (20–25%)
  - SFRA folder structure and cartridge hierarchy
  - Client-side and server-side rendering patterns
  - Form handling and validation
  
- **Cartridges & Reusable Components** (15–20%)
  - Cartridge creation, dependencies, and modularization
  - Controllers and routes
  - Business logic encapsulation
  
- **ISML Templating** (15–20%)
  - ISML template syntax, includes, and components
  - Local and remote includes
  - Template inheritance and composition
  
- **JavaScript Controllers & Server-Side Logic** (15–20%)
  - Request/response cycle
  - Session and cookie management
  - Error handling and middleware
  
- **OCAPI & API Integration** (10–15%)
  - Open Commerce API (OCAPI) for headless commerce
  - Custom endpoints and integrations
  - Authentication and authorization
  
- **Pipeline Architecture & Hooks** (5–10%)
  - Pre/post pipeline execution
  - Job framework for background tasks

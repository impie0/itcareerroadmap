## About

The Salesforce Certified Integration Architect is an expert-level credential that validates mastery of designing, building, and scaling enterprise integration solutions on the Salesforce platform. Unlike entry and professional certifications, it is a **standalone exam** (not composite) and focuses exclusively on systems integration, middleware patterns, API architecture, data synchronization, and operational resilience at scale.

This certification is one of four prerequisites for the **Salesforce Certified System Architect** designation, which in turn is a mandatory stepping stone to the Salesforce Certified Technical Architect (CTA)—the highest individual certification on the Salesforce platform. Integration Architects design how Salesforce connects to off-platform systems, manage data flow across enterprise landscapes, implement fault tolerance, and architect solutions that span cloud, hybrid, and on-premise ecosystems.

Released as a distinct credential around 2020, the Integration Architect exam has evolved to reflect MuleSoft Anypoint Platform adoption, modern API-led connectivity patterns, and event-driven integration architectures. It is required by organizations building complex integrations (healthcare, financial services, retail, manufacturing) where Salesforce must interoperate with legacy ERP, data warehouses, and third-party SaaS.

## Topics covered

*Based on official Salesforce exam blueprint for Integration Architect (Int-Arch-301):*

- **Evaluate Current System Landscape (20%)**
  - System inventory and dependencies
  - Data sources, APIs, and protocols
  - Integration touchpoints and bottlenecks
  - Legacy system constraints and cloud readiness assessment

- **Evaluate Business Needs (15%)**
  - Integration requirements capture
  - Frequency and timeliness (real-time vs. batch)
  - Data volume, velocity, and variety
  - Availability and disaster recovery targets

- **Translate Business Needs to Integration Requirements (15%)**
  - SLA and performance requirements
  - Error handling and retry strategies
  - Consistency models (eventual vs. immediate)
  - Compliance and security controls

- **Design Integration Solutions (30%)**
  - Integration patterns (point-to-point, hub-and-spoke, event-driven, API-led)
  - Salesforce APIs (REST, SOAP, GraphQL, Bulk API 2.0)
  - Platform events and streaming
  - MuleSoft Anypoint Platform and connector frameworks
  - Data mapping and transformation
  - Idempotency and duplicate handling

- **Build and Deploy Solutions (20%)**
  - Apex integration patterns (callouts, scheduled jobs, batch)
  - Flow integration and automation
  - API versioning and lifecycle management
  - Testing, monitoring, and logging
  - Change management and blue-green deployments

## Skills validated

*Specific technologies and competencies this exam validates.*

- **Salesforce APIs:** REST, SOAP, GraphQL, Bulk API 2.0, Metadata API
- **Integration patterns:** Point-to-point, hub-and-spoke, event-driven, API-led connectivity, service-oriented
- **MuleSoft Anypoint Platform:** Design Center, Anypoint Exchange, connectors, API management
- **Platform Events and Streaming:** Event-driven architecture, change data capture, streaming channels
- **Apex integration:** HTTP callouts, async patterns, batch processing, scheduled jobs, queueable jobs
- **Data mapping and transformation:** XSLT, DataWeave, formula language, custom transformation logic
- **API design and governance:** Versioning strategies, lifecycle management, rate limiting, security policies
- **Middleware and integration middleware:** Enterprise Service Bus (ESB) concepts, message queuing, orchestration
- **Error handling and resilience:** Retry logic, idempotency, circuit breakers, failover strategies, dead-letter handling
- **Data synchronization:** Master data management (MDM), eventual consistency, bidirectional sync, conflict resolution
- **Security and compliance:** OAuth 2.0, API security, encryption, audit logging, compliance frameworks (HIPAA, PCI, GDPR)
- **Performance optimization:** Throughput, latency, scalability, bulk operations, governor limits in integration context
- **Monitoring and observability:** Logging, alerting, tracing, analytics for integration health
- **Testing strategies:** Unit testing, integration testing, performance testing, chaos engineering concepts

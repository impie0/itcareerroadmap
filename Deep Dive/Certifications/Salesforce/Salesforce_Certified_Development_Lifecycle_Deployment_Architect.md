## About

The Salesforce Certified Development Lifecycle and Deployment Architect (DLDA) is an expert-level credential that validates mastery of designing, managing, and optimizing the complete software development lifecycle within Salesforce organizations. This is a **standalone exam** focusing specifically on governance, change management, deployment strategies, CI/CD pipelines, testing frameworks, and operational stability — distinct from the application architecture or integration design roles.

One of four mandatory components of the **Salesforce Certified System Architect** designation, the DLDA cert is required for organizations implementing enterprise-scale Salesforce solutions where code quality, release cadence, and deployment risk management are mission-critical. DLDA-certified architects design sandbox strategies, implement version control integration (Salesforce DX with Git/Bitbucket/GitHub/Azure DevOps), architect CI/CD workflows, and oversee multi-team release planning and rollout coordination.

The credential reflects modern Salesforce DevOps practices circa 2024–2026, including metadata-driven development, source control as the source of truth, automated testing at scale, and data migration governance. It is pitched at lead architects, DevOps engineers transitioning into Salesforce, and senior administrators stepping into architectural governance roles.

## Topics covered

*Based on official Salesforce exam blueprint for Development Lifecycle and Deployment Architect (Plat-Arch-202):*

- **Plan and Design Organization's Environment Strategy (25%)**
  - Sandbox environments (dev, test, staging, UAT, production)
  - Environment refresh strategy and data masking
  - Organizational capacity planning for concurrent development
  - Metadata-driven development and Salesforce DX adoption
  - Environment URLs, API versions, and compatibility management

- **Analyze and Recommend Change Management and Deployment Solutions (30%)**
  - Release planning and release windows
  - Change advisory boards (CABs) and governance process
  - Regression and testing strategies
  - Deployment order dependencies and impact analysis
  - Rollback and emergency procedures
  - Deployment automation and tools integration

- **Plan and Implement Version Control and DevOps Solutions (20%)**
  - Source control systems (Git, Bitbucket, GitHub, Azure DevOps)
  - Salesforce DX and source-driven development
  - Package management and package versions
  - CI/CD pipeline architecture
  - Automated testing integration (unit, integration, system testing)
  - Build servers and deployment orchestration

- **Plan, Design, and Implement Testing Frameworks (15%)**
  - Test planning and test case management
  - Automated testing (Apex test classes, Selenium, etc.)
  - Test coverage requirements (75% minimum org-wide)
  - Data seeding and test data management
  - Performance and load testing

- **Plan, Design, and Implement Data Deployment Strategies (10%)**
  - Data validation and reconciliation in deployments
  - Data migration tools (Data Loader, Gearset, Copado, etc.)
  - Data dependencies and timing coordination
  - Post-deployment verification

*Source:* [Salesforce Development Lifecycle and Deployment Architect Exam Blueprint ↗](https://trailhead.salesforce.com/credentials/developmentlifecycledeploymentarchitect)

## Skills validated

*Specific technologies and competencies this exam validates.*

- **Salesforce DX (SFDX)** — CLI tooling, project structure, package development, deployment workflows
- **Source control systems** — Git fundamentals, Bitbucket, GitHub, Azure DevOps integration
- **Sandbox strategy and management** — Sandbox types (dev, partial, full), refresh schedules, capacity planning
- **Version control as source of truth** — Metadata-driven development, branching strategies (GitFlow, trunk-based), merge conflicts
- **CI/CD architectures** — Jenkins, GitLab CI/CD, GitHub Actions, Azure Pipelines, Bitbucket Pipelines
- **Automated testing** — Apex test classes, test data management, code coverage (75% minimum org-wide), test execution in pipelines
- **Change management governance** — Release planning, CAB processes, impact analysis, risk assessment, rollback procedures
- **Deployment tools and frameworks** — Gearset, Copado, Flosum, Deployment Pro, native Salesforce change sets and ANT migration tool
- **Data migration and validation** — Data Loader, API-based data sync, reconciliation post-deployment
- **Monitoring and observability** — Salesforce logs, performance insights, post-deployment validation
- **Security and compliance in deployment** — Code review practices, secrets management, audit trails, compliance documentation
- **Environment management** — Production stability, disaster recovery, data backup and recovery in Salesforce
- **Release communication and coordination** — Stakeholder management, pre-deployment testing coordination, go/no-go decisions
- **Performance optimization** — Build time optimization, parallel testing, query performance in deployments

# Salesforce Certified Application Architect

**`Arch-App`** · ● **Active** · **Expert** · _Salesforce_

> **Composite architecture designation (no single exam).** Validates mastery of Salesforce platform customization, data modeling, integration, and security governance. Earned by passing five constituent specialist exams: Platform Developer I, Platform App Builder, Data Architecture and Management Designer, Sharing and Visibility Designer, and Integration Architect. Gateway credential toward Salesforce Certified Technical Architect (CTA).

## About

The Salesforce Certified Application Architect (Application Architect) is a **composite designation** that validates deep expertise in designing, building, and governing Salesforce solutions at the application layer. Unlike single-exam certifications, it is earned by passing five tightly integrated specialist exams covering the full application development, integration, security, and customization lifecycle: Platform Developer I (Apex & code-based customization), Platform App Builder (declarative customization & UX), Data Architecture and Management Designer (data modeling & governance), Sharing and Visibility Designer (security & org-wide access control), and Integration Architect (APIs, middleware, event-driven architecture).

Introduced as a formal designation around 2015, the Application Architect credential has become the **primary stepping stone to Salesforce's expert-level CTA (Certified Technical Architect)** examination. To earn CTA, candidates must simultaneously hold both an Application Architect and a System Architect designation, making the Application Architect one of the two mandatory prerequisites in Salesforce's highest-achievement pathway.

The Application Architect designation signals to employers that a candidate can design end-to-end Salesforce solutions, model complex data requirements, implement security governance, and write production-quality Apex code—the full spectrum of skills required to lead application architecture initiatives on the Salesforce platform.

## Constituent exams & topics

### 1. Salesforce Certified Platform Developer I (Plat-Dev-201)

**What it covers:** Apex programming language, SOQL/SOSL queries, Lightning Web Components (LWC), triggers, asynchronous processing, testing, APIs, and governor limits.

**Weighted topics:**
- Apex Programming Fundamentals (25%)
- Object-Oriented Programming (10%)
- SOQL and SOSL (10%)
- Triggers and Asynchronous Processing (20%)
- Lightning Web Components (15%)
- Testing and Deployment (15%)
- APIs and Integration (5%)

### 2. Salesforce Certified Platform App Builder (Plat-App-201)

**What it covers:** Declarative customization, custom objects and fields, automation (Process Builder, Flow), user interface design, data validation, and formula/rollup fields.

**Key topics:**
- Object Model & Relationships
- Custom Fields & Formulas
- Automation (Flows, Approvals, Business Rules)
- User Interface Design & Navigation
- Data Management & Import
- Security & Access Control
- Testing & Deployment

### 3. Salesforce Certified Data Architecture and Management Designer (Plat-Arch-201)

**What it covers:** Data model design, storage limits, data migration strategies, backup/recovery, org consolidation, and compliance considerations (GDPR, HIPAA).

**Key topics:**
- Org Structure & Limits
- Custom Objects, Fields & Relationships
- Data Model Design Patterns
- Data Migration & ETL
- Backup, Recovery & Archival
- Master Data Management
- Analytics & Reporting Infrastructure
- Compliance & Data Governance

### 4. Salesforce Certified Sharing and Visibility Designer (Plat-Sec-201)

**What it covers:** Organization-wide defaults, role hierarchies, sharing rules, manual sharing, field-level security, record-level access patterns, and account teams.

**Key topics:**
- Org-Wide Defaults
- Role Hierarchies & Delegated Administration
- Sharing Rules (criteria-based, ownership-based)
- Manual Sharing & Account Teams
- Field-Level Security
- Record-Level Access
- Public Groups & Chatter Privacy
- Audit & Compliance Integration

### 5. Salesforce Certified Integration Architect (Plat-Int-201)

**What it covers:** API design patterns, middleware selection, asynchronous and synchronous integration, event-driven architecture, middleware frameworks (MuleSoft), governance, and security in integration scenarios.

**Key topics:**
- Integration Architecture Patterns & Protocols
- API Design & REST / SOAP best practices
- Middleware Selection (MuleSoft Anypoint, Boomi, Talend, custom webhooks)
- Synchronous vs. Asynchronous Processing
- Event-Driven Architecture & Platform Events
- Data Synchronization & Master Data Management
- Security in Integration (OAuth, mutual TLS, encryption)
- Error Handling & Retry Logic
- Monitoring & Performance Tuning
- Integration Governance & Compliance

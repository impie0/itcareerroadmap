## About

The B2C Commerce Architect credential validates mastery in designing scalable, multi-site, multi-region B2C Commerce solutions that align business strategy with technical feasibility at enterprise scale. Earned after the B2C Commerce Developer prerequisite, it signals deep expertise in solution architecture, integration patterns, performance optimization, and delivery best practices. The exam emphasizes real-world scenarios—translating complex requirements into modular, maintainable architectures that support global operations, headless commerce models, and high-volume integrations.

---

## Domain context — Commerce/Architecture

Salesforce B2C Commerce (formerly Commerce Cloud) is a SaaS platform for building headless and storefront-based ecommerce experiences at scale. Architects in this domain design solutions spanning site topology, integration topology, data architecture, and deployment strategies—often for retailers operating dozens of catalogs, price books, and locales simultaneously.

## Topics covered

Exam blueprint (5 domains, no published weights; estimates from candidate feedback):

- **Discovery & Strategy (28%)**: Translate business requirements into technical specifications; create data flow diagrams; assess site readiness; evaluate third-party solutions and APIs; document architectural decisions for stakeholders; analyze integration points and data volumes.
- **B2C Commerce Architecture (23%)**: Design multi-site/multi-region topologies; apply modularity and reusability patterns; choose between storefront vs. headless models; architect site organization (sites, catalogs, price books, customers); design data and code replication schedules.
- **Performance & Scaling (16%)**: Configure caching strategies (page cache, dynamic content cache); optimize catalog and product data structures; design indexing approaches; monitor key performance indicators; apply SFCC best practices for security and performance.
- **Integration Patterns (23%)**: Select protocols (SOAP vs. REST); choose real-time vs. batch integration; apply security strategies for web service integrations; leverage Job Framework for scheduled tasks; design custom cartridge architectures; implement service framework error handling and circuit breakers.
- **Launch & Deployment (10%)**: Set up SEO-compliant aliases; create rollback plans; plan blue-green deployments; define data migration and replication schedules; establish launch checklists and monitoring; support post-production data and code management.

*Source:* [Salesforce B2C Commerce Architect Exam Blueprint ↗](https://reviewnprep.com/certificate-details/salesforce-certified-b2c-commerce-architect)

## Common skills at Commerce · Expert

*Shared capabilities for enterprise-scale ecommerce architecture and platform design—not specific to Salesforce B2C.*

- **Multi-tenancy & Site Scaling**: Design site hierarchies, catalogs, and price book structures for hundreds of SKUs and concurrent users.
- **Headless Architecture**: Decouple presentation layer from commerce backend; design API-first solutions for omnichannel retail.
- **Data Integration at Scale**: Master batch and real-time data synchronization; design ETL workflows for inventory, orders, and customer master data.
- **Performance Tuning**: Cache strategies (content delivery, product data, customer sessions); indexing and query optimization for large datasets.
- **Search & Merchandising**: Elasticsearch integration; faceted navigation design; personalization and recommendation rules.
- **Payment & Compliance**: PCI-DSS best practices; tokenization; tax/tariff compliance across regions; GDPR/CCPA data handling.

## Skills validated

*Specific technologies and frameworks this exam certifies—distinct from general architectural competency.*

- **B2C Commerce Platform**: Site hierarchy, catalogs, price books, customer profiles, order management, inventory integration.
- **Cartridge Development**: Custom Controllers, Pipelines, SFRA (Storefront Reference Architecture), modular code patterns.
- **Integration Frameworks**: SFCC Job Framework, Service Framework, API webhooks, OData feeds, real-time vs. batch connectors.
- **Performance Optimization**: Page caching, dynamic content caching, Elasticsearch indexing, CDN integration, analytics.
- **Multi-Site Topology**: Site organization, aliases, localization, currency/tax configuration, region-specific builds.
- **Security Patterns**: OAuth 2.0, certificate management, PCI compliance, data masking, secure configuration.
- **Deployment & DevOps**: Build automation, code promotion across sandboxes, blue-green deployment, rollback procedures, monitoring.

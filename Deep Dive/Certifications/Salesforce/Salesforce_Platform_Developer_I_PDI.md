## About

The Salesforce Certified Platform Developer I (PDI) is an entry-level development credential introduced in 2010 and refined continuously through Salesforce's three annual platform releases (Spring, Summer, Winter). As of 2026, PDI is the foundational credential for Salesforce developers and is often the first hands-on coding exam professionals take after gaining platform familiarity via administration.

The PDI validates competency in **Apex programming** (Salesforce's Java-like language), **SOQL/SOSL queries**, **Lightning Web Components (LWC)**, **API integration**, **triggers**, and **asynchronous processing**. It is a prerequisite pathway to mid-level specialization (Platform Developer II, architect tracks) and pairs well with consultant, admin, and systems architecture roles.

The exam is **no longer accepting new takers for the legacy DEV-401 code** (retired 2020); all candidates now test on the current Plat-Dev-201 blueprint.

## Topics covered

The PDI exam blueprint covers six major domains. Salesforce publishes approximate weightings:

- **Apex Programming Fundamentals** (25%) — Variables, data types, control flow, OOP concepts, class/interface design, exception handling
- **Object-Oriented Programming** (10%) — Inheritance, polymorphism, encapsulation, abstract classes, interfaces, design patterns
- **SOQL and SOSL** (10%) — Query syntax, filtering, aggregation, relationship queries, query optimization
- **Triggers and Asynchronous Processing** (20%) — Trigger types, order of execution, batch apex, scheduled jobs, future methods, queueable interface, platform events
- **Lightning Web Components (LWC)** (15%) — Component lifecycle, property binding, event handling, data binding, composition, parent-child communication
- **Testing and Deployment** (15%) — Unit test best practices, code coverage (75% minimum required), assertions, mock objects, sandbox deployment strategies, change sets, metadata API
- **APIs and Integration** (5%) — REST/SOAP API basics, HTTP callouts, authentication, JSON serialization

*Source:* [Official Salesforce Exam Guide — Plat-Dev-201](https://trailhead.salesforce.com/en/credentials/platformdeveloperi)

## Skills validated

*Concrete technologies and techniques this exam validates — not shared domain skills.*

- **Apex language fundamentals** — variables, primitives, objects, collections (List, Set, Map), control structures (if/else, loops, switch)
- **Object-oriented Apex** — class definition, constructors, methods, inheritance, interfaces, abstract classes, static members, access modifiers
- **SOQL queries** — SELECT, WHERE, JOIN (relationships), aggregate functions (COUNT, SUM, MAX), ORDER BY, LIMIT, OFFSET, type casting
- **SOSL queries** — Search, IN clause, matching SOQL results
- **Triggers** — Before/After triggers, Insert/Update/Delete/Undelete context, trigger context variables, bulk operations
- **Asynchronous Apex** — Future methods, batch apex (execute(), start(), finish()), scheduled apex (Schedulable), Queueable interface, platform events
- **Lightning Web Components** — Component lifecycle (connectedCallback, disconnectedCallback, renderedCallback), property binding, event handling (@wire decorators), parent-child communication, slots
- **APIs** — REST/SOAP basics, HTTP callouts from Apex, JSON/XML parsing, authentication (OAuth), rate limit handling
- **Testing** — Test class structure (@isTest), assertions (assertEquals, assertTrue, fail), test data creation, mock callouts (HttpCalloutMock), code coverage metrics
- **Deployment** — Change sets, metadata API, sandbox refresh strategies, managed packages (basics)

## Career progression context

**Entry Path (0–6 months post-certification):**
- Junior Salesforce Developer roles at agencies, consulting firms, or mid-market companies
- Typical responsibilities: Write Apex triggers and batch jobs, maintain custom Lightning components, support integration with third-party systems
- Salary: $85K–$110K

**Intermediate (12–24 months post-PDI):**
- Can pursue [Platform Developer II](../../Certifications/Salesforce/Salesforce_Platform_Developer_II_PDII.md) (advanced patterns, architecture)
- Can laterally move to [Consultant tracks](../../Ecosystems/D09_Salesforce_Ecosystem.md#consultant-track-by-cloud) (Sales Cloud, Service Cloud, Marketing Cloud Consultant)
- Can pursue [System/Application Architect composites](../../Ecosystems/D09_Salesforce_Ecosystem.md#architect-track-40-composites) as foundation
- Typical salary bump: $110K–$135K

**Long-term (3+ years post-PDI):**
- Platform Developer II, then Architecture tracks (System Architect, Application Architect, specialty architects like Heroku Architect, CPQ Architect)
- Can pursue board-reviewed [Certified Technical Architect (CTA)](../../Ecosystems/D09_Salesforce_Ecosystem.md#top-tier-certification-certified-technical-architect-cta) (extremely selective; ~20–30% approval rate; $6K exam cost; $160K–$400K+ consulting roles)
- Can specialize in vertical industry clouds (Health, Financial Services, Manufacturing)

**Typical 5-year earnings trajectory:**
- Year 1 post-PDI: $85K–$110K (junior developer)
- Year 2–3: $110K–$140K (mid-level developer or consultant entry)
- Year 4–5: $140K–$180K (mid-level consultant, junior architect, or specialist with add-on certs)
- Year 6+: $160K–$250K (architect, specialist with multiple certs, or senior individual contributor)

## Exam strategy & tips

**Preparation timeline:** 8–12 weeks (40–60 hours total) for professionals with 6+ months hands-on Apex/development experience; up to 16 weeks for career-changers from non-dev backgrounds.

**Study sequence:**
1. **Week 1–2:** Complete free Trailhead modules (Apex Basics, SOQL, Triggers, LWC Basics)
2. **Week 3–4:** Hands-on sandboxing — write 5–10 small Apex programs (classes, triggers, batch jobs)
3. **Week 5–8:** Take structured course (Focus on Force or Mike Wheeler Udemy) + daily Trailhead modules
4. **Week 9–10:** Complete 1–2 practice exams; identify weak domains (typically Triggers and SOQL for first-timers)
5. **Week 11:** Final review of weak areas + mock exam
6. **Week 12:** Exam day

**Common weak areas:**
- **Trigger order of execution** — memorize the sequence (before triggers, validation rules, after triggers, workflows, flows, async processes)
- **SOQL aggregation** — GROUP BY, HAVING, and relationship queries often trip candidates
- **Test coverage** — ensure understanding of mock objects, assertions, and why 75% minimum is enforced
- **Asynchronous processing limits** — batch sizes, governor limits on future calls, scheduled job frequency

**Exam day tips:**
- Read each question carefully; Salesforce exam questions are precise and often test subtle edge cases
- Flag uncertain answers and review at end if time permits (PDI has no time pressure for most test-takers)
- Do not memorize code syntax exactly — focus on concepts and patterns, as questions emphasize design over memorization
- Review the official exam guide once more 48 hours before exam; Salesforce updates blueprints quarterly

## About

The Salesforce Certified Platform Developer II (PDII) is the professional-level certification for Apex and Salesforce platform development. PDII builds directly on PDI skills, requiring demonstrated mastery of advanced logic automation, integrations, UI customisation, testing, performance optimisation, and debugging. It is the primary credential for senior Salesforce developers and development leads. Released in 2016, PDII remains the de facto advanced developer certification within the Salesforce ecosystem and is a prerequisite for Solution Architect tracks.

## Domain context — Software Engineering / CRM

Customer Relationship Management (CRM) platforms enable enterprises to manage customer data, automate sales and service workflows, and integrate business systems. Salesforce dominates the CRM SaaS market, with its extensible platform (custom Apex code, Lightning components, integrations) requiring dedicated developer roles and certifications.

## Topics covered

Per the official Salesforce exam blueprint:

- **Logic and Process Automation** (33%) — Apex triggers, batch Apex, scheduled actions, process builder, flow logic, exception handling, governor limits
- **User Interface** (14%) — Lightning Web Components (LWC), Aura, controllers, JavaScript, responsive design, testing UI code
- **Integration** (15%) — REST/SOAP APIs, callouts, webhooks, middleware patterns, OAuth, error handling in integrations
- **Testing** (10%) — Apex test classes, code coverage requirements (75%+ org coverage), mocking, test data factories
- **Performance and Optimisation** (15%) — Query optimisation, SOQL limits, bulk operations, caching, asynchronous patterns, monitoring
- **Debugging / Deployment Tools** (13%) — Logs, breakpoints, change sets, source control (Git), CI/CD, ant/SFDX, rollbacks

*Source:* [Salesforce PDII Blueprint ↗](https://trailhead.salesforce.com/credentials/platformdeveloperii)

## Skills validated

*Specific technical competencies tested by the PDII exam.*

- Advanced Apex: Interfaces, abstract classes, custom iterables, exception handling, governor limit optimisation
- Asynchronous Apex: `@future`, batch Apex, scheduled Apex, queueable jobs, event-driven architecture
- SOQL and SOSL: Complex queries, relationship queries, query optimisation, avoiding N+1 pitfalls
- Triggers: Trigger patterns (handler classes, context-aware design), avoiding trigger recursion, bulk operations
- REST and SOAP APIs: Callouts, request/response serialisation, OAuth flow, external service integration
- Lightning Web Components (LWC): JavaScript, reactive properties, lifecycle hooks, composition patterns
- Aura Framework: Controllers, events, actions, debugging Aura-specific issues (legacy; LWC preferred)
- Unit Testing: Test classes, mocking (Test.startTest/stopTest), code coverage 75%+, factory patterns
- Deployment: SFDX, source control, CI/CD concepts, change sets, rollback strategies
- Performance Tuning: Index strategies, batch window optimisation, CPU time and heap usage limits
- Debugging Tools: Developer console logs, checkpoints, breakpoints, Apex log analysis

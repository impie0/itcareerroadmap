## About

The Salesforce Certified Slack Developer certification highlights ability to build powerful custom applications on the Slack platform and demonstrates knowledge of Slack APIs and skill in planning, designing, and delivering effective Slack apps that drive collaboration and productivity. This professional-level certification targets developers with at least 6 months of hands-on experience building Slack apps; no formal prerequisites exist, though fluency in JavaScript, Python, or Java is strongly recommended.

## Topics covered

The exam blueprint covers four major domains:

- **Apps in the Slack Platform** — Automating and enhancing workspace operations; recommending appropriate surfaces (App Home, modals, messages) for use cases
- **App Design & Architecture** — Designing apps that interact effectively with Slack architecture; choosing appropriate components (incoming webhooks, slash commands, shortcuts, interactive components, bots)
- **Slack APIs & Implementation** — Selecting the right Slack API (Web API, Events API, RTM) based on business needs; secure implementation patterns
- **Security & Enterprise Features** — Data protection; secure communication; Enterprise Grid integration; OAuth flows and token management
- **App Distribution** — Token generation; workspace containment; multi-workspace distribution; Slack Directory submission process

## Skills validated

*Cert-specific — what this exam actually tests, distinct from the shared "Common skills" above.*

- **Slack Bolt SDK** — JavaScript, Python, and Java framework usage for building apps
- **Block Kit UI framework** — Visual component design and layout for surfaces
- **Modals and interactive workflows** — Modal listeners, submission handling, view stacks
- **Slash commands and shortcuts** — Command routing and shortcut handling
- **Event subscriptions** — Workspace event listeners and subscription patterns
- **OAuth and token management** — App installation, token scopes, and secure token storage
- **Slack Web API** — Core API methods for messages, files, user management, channels
- **Slack Events API** — Event-driven application patterns
- **RTM (Real-Time Messaging) API** — WebSocket-based real-time communication
- **App distribution and publishing** — Single-workspace vs. multi-workspace apps; Slack Directory submission
- **Enterprise Grid features** — Org-wide apps, Enterprise Grid Sandbox, Slack Connect
- **Incoming webhooks** — Simple message posting without full app authentication

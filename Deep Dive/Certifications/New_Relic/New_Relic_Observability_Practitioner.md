## About

The New Relic Certified Observability Practitioner (also known as the Full Stack Observability Practitioner certification) is an Associate-level credential validating intermediate knowledge of the New Relic platform and modern observability practices. This certification is designed for professionals who have foundational observability knowledge and wish to demonstrate practical expertise in deploying, configuring, and managing comprehensive monitoring across applications, infrastructure, and user experience using New Relic. Prerequisites recommend completion of the New Relic Verified Foundation certification and 6+ months of hands-on experience with New Relic. Launched in 2024 as part of New Relic's structured certification track, the exam is proctored and offered free through New Relic University.

## Domain context — Observability & APM

Application Performance Monitoring (APM) and observability focus on monitoring, visualizing, and analyzing the health, performance, and behavior of software applications and their infrastructure in real time. Modern observability spans metrics, logs, traces, events, and user experience data across distributed cloud-native systems.

## Topics covered

Based on the official exam blueprint and learning paths, the NR-OP exam assesses knowledge across these core areas:

- **New Relic platform architecture** — Account structure, UI navigation, data organization, and workloads.
- **Telemetry data types (MELT)** — Metrics, Events, Logs, and Traces; understanding when and how to use each.
- **APM (Application Performance Monitoring)** — Deploying APM agents, instrumenting applications, monitoring transaction performance, error tracking.
- **Distributed tracing and OpenTelemetry** — Understanding traces, service dependencies, latency analysis, and integration with OpenTelemetry.
- **Infrastructure monitoring** — Host monitoring, container monitoring, Kubernetes orchestration visibility.
- **Browser monitoring (Real User Monitoring)** — Page performance, Core Web Vitals, user experience metrics, session replay.
- **Synthetic monitoring** — API tests, scripted browsers, uptime checks, alert integration.
- **NRQL (New Relic Query Language)** — Query syntax, data aggregation, dashboard and alert query building.
- **Dashboards and data visualization** — Creating, customizing, sharing dashboards; visualization types.
- **Alerts and notification configuration** — Alert policies, conditions, thresholds, incident workflows, notification channels.
- **Log management** — Log forwarding, parsing, log patterns, log-based alerting.
- **New Relic AI (AIOps)** — Anomaly detection, Proactive Detection, error tracking, AI-driven insights.
- **Workloads and service mapping** — Grouping related entities, understanding service dependencies.
- **Service levels (SLI/SLO)** — Defining and tracking service level indicators and objectives.

*Source:* [Full-Stack Observability Exam ↗](https://learn.newrelic.com/full-stack-observability-exam-1)

- **New Relic platform mastery** — Full configuration, dashboard creation, alert policy setup, data exploration via NRQL.

- **Distributed tracing and service dependency mapping** — Understanding trace analysis, identifying root causes, latency optimization.

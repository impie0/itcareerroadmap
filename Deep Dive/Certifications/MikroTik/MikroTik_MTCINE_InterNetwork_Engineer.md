## About

The MTCINE is MikroTik's most advanced routing certification, covering enterprise and service-provider scale inter-networking. It follows the MTCRE and requires hands-on mastery of BGP confederation and route reflection, MPLS L2/L3 VPNs, traffic engineering (TE), and large-scale OSPF/IS-IS design. The exam is administered by approved MikroTik trainers at the end of a three-day instructor-led course heavy on RouterOS labs. Open-book format emphasizes practical troubleshooting and design decision-making rather than memorization. MTCINE holders are positioned for roles in ISP network operations, carrier engineering, WISP backbone design, and complex routing architecture.

## Domain context — SP/Carrier Networking

Service-provider and carrier-grade IP routing, MPLS, and BGP convergence at continental and multi-region scale. Core skills: interdomain routing policy, label-switched paths, VPN architectures, and traffic engineering for resilience and capacity optimization.

---

## Topics covered

Official exam domains (open-book, 25 questions total):

- **BGP at scale** — Autonomous Systems, BGP transport and packet types, iBGP/eBGP with loopbacks, Private AS removal, multihop configuration, route distribution and filtering, path selection algorithm, prefix attributes, route reflectors, confederations
- **MPLS fundamentals** — Label Distribution Protocol (LDP), static labeling, penultimate-hop-popping (PHP), MPLS traceroute, LDP-based pseudowires, VPLS tunnels, Bridge Split Horizon in L2 MPLS, MTU and fragmentation handling
- **Layer 2 MPLS** — BGP-based L2 tunnels, VPLS (Virtual Private LAN Service), pseudowire configuration and troubleshooting
- **Layer 3 VPNs** — VRF (Virtual Routing and Forwarding), BGP-based L3VPN, OSPF as CE-PE protocol, route target and route distinguisher management
- **Traffic Engineering (TE)** — RSVP fundamentals, static path definition, dynamic path (CSPF), bandwidth reservation and admission control, bandwidth constraints and link utilization optimization
- **ISP routing strategies** — Multi-carrier redundancy, failover design, load balancing across eBGP peers, route filtering and policy implementation, community-based routing

## Skills validated

*Concrete technologies and protocols tested on the MTCINE exam.*

- Border Gateway Protocol (BGP) v4 — iBGP, eBGP, multihop, route reflection, confederations
- Multiprotocol Label Switching (MPLS) — LDP, pseudowires, label propagation, PHP
- Virtual Private LAN Service (VPLS) — BGP-based and LDP-based L2 VPN
- Virtual Routing and Forwarding (VRF) — multi-tenant routing, route targets
- BGP/MPLS Layer 3 VPNs (L3VPN) — inter-AS Option A/B/C scenarios
- Open Shortest Path First (OSPF) — large-area designs, CE-PE interaction
- Intermediate System to Intermediate System (IS-IS) — link-state routing at scale
- Traffic Engineering — RSVP, constraint-based path selection (CSPF), bandwidth management
- RouterOS CLI and configuration of advanced routing constructs
- Network troubleshooting tools — traceroute, BGP debug, MPLS LSP path verification
- Service Provider design principles — redundancy, failover, load balancing

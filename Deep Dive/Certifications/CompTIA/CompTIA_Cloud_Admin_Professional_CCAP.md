# CompTIA Cloud Admin Professional

**`CCAP`** · ● **Active** · **Professional** · _CompTIA_

> **Stackable credential**: Requires concurrent holding of CompTIA **Network+** and **Cloud+** certifications. No separate CCAP exam exists; the credential is awarded automatically upon passing both exams and maintaining their validity.

## About

The CompTIA Cloud Admin Professional (CCAP) is a stackable professional-level credential that validates combined competency in networking and cloud administration. Unlike traditional single-exam certifications, CCAP is awarded automatically once candidates successfully pass both CompTIA Network+ (N10-009) and CompTIA Cloud+ (CV0-004) and maintain active status in both. Designed for IT professionals with 2–5 years of experience, the credential signals mastery of cloud infrastructure management, network administration, and the intersection of networking and cloud services — critical for organizations building or operating hybrid and multi-cloud environments. The stackable model allows candidates to earn credentials incrementally while building toward professional-level validation.

## Domain context — Cloud Administration + Networking

This credential bridges two core IT domains: cloud infrastructure operations (provisioning, monitoring, security, cost optimization) and network administration (routing, switching, connectivity, SDN). Cloud administrators increasingly manage both layers as modern infrastructure merges on-premises networking with cloud services.

## Skills validated

_Concrete technologies and competencies this stackable credential tests—beyond the shared domain skills listed above._

- Routing and switching protocols (OSPF, BGP, VLAN, spanning tree)
- Cloud networking models (VPC/vNets, peering, hybrid connectivity, SD-WAN)
- Public cloud services across AWS, Azure, GCP, and OCI
- Infrastructure-as-Code tools (Terraform, CloudFormation, ARM templates, Ansible)
- Cloud security posture management (CSPM), identity and access management (IAM)
- Container orchestration (Kubernetes basics) and serverless architectures
- Cost optimization and cloud resource tagging/chargeback models
- Monitoring and observability platforms (CloudWatch, Azure Monitor, Stackdriver, Datadog)
- Troubleshooting cloud connectivity, performance, and availability issues
- Compliance and governance frameworks (FedRAMP, HIPAA, PCI-DSS, ISO 27001)

## About

CompTIA Linux+ (XK0-005, V7) was a vendor-neutral, entry-to-intermediate Linux system administration certification covering system management, security, scripting, and troubleshooting. Launched June 2022, it replaced XK0-004 and served as the primary CompTIA OS-level credential for career progression from help desk to junior sysadmin roles until retirement in January 2026. **As of May 1, 2026, candidates should pursue XK0-006 (V8) instead.**

## Topics covered

**Domain 1: System Management (32%)**
- Installation, initial configuration, kernel management
- Filesystem management (ext4, XFS, Btrfs, LVM, partitioning)
- File permissions, ownership, ACLs, setuid/setgid
- User and group management (useradd, usermod, sudoers)
- Package management (apt/apt-get, dnf/yum, zypper, pacman)
- Process management (systemd, systemctl, cron, at)
- System logging (rsyslog, journalctl, log rotation)
- Performance monitoring (vmstat, iostat, sar, top, ulimit)

**Domain 2: Security (21%)**
- SELinux and AppArmor (modes, contexts, profiles)
- Firewall implementation (iptables, nftables, firewalld, ufw)
- SSH configuration and key-based authentication
- TCP/IP basics and network tools (ss, netstat, dig, nslookup, traceroute)
- File integrity monitoring, encryption fundamentals
- Access control and privilege escalation (sudo, runlevels)

**Domain 3: Scripting, Containers, and Automation (19%)**
- Bash scripting (variables, conditionals, loops, functions, error handling)
- Text processing (grep, sed, awk, cut, sort, uniq)
- Regular expressions (POSIX BRE and ERE)
- Basic Git workflows (init, clone, add, commit, push, pull)
- Container basics (Docker concepts, docker pull/run, Dockerfile)
- Infrastructure as Code (Terraform, Ansible high-level familiarity)

**Domain 4: Troubleshooting (28%)**
- System performance analysis, bottleneck identification
- Log analysis and interpretation
- Filesystem troubleshooting (fsck, badblocks, df, du, inode issues)
- Boot process and GRUB2 troubleshooting, recovery
- Network troubleshooting (connectivity, routing, DNS, firewall)
- Service/daemon troubleshooting (systemd journal, unit dependencies)
- Hardware diagnostics, device driver issues
- Application troubleshooting (strace, ltrace, ldd)

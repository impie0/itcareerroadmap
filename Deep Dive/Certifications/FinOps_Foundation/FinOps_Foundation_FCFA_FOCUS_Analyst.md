## About

The FinOps Certified FOCUS Analyst credential certifies proficiency in FOCUS (FinOps Open Cost & Usage Specification), an open standard that normalizes billing datasets across multiple cloud and technology vendors. FOCUS v1.2 was ratified in May 2025; v1.3 is scheduled for June 2026. This specialist certification targets data analysts, FinOps practitioners, and cloud cost engineers who need to work with conformant billing data, generate FOCUS-aligned cost reports, and drive cost visibility across multi-cloud environments. Prerequisites are minimal; target audience includes those with basic data analysis experience and awareness of cloud billing structures.

## Topics covered

- FOCUS specification core concepts and governance (FOCUS schema versions, steering committee, conformance standards)
- Column definitions and metadata (dimension vs. measure columns; semantic meaning in billing contexts)
- Provider conformance (AWS CUR migration to FOCUS; Azure Billing API; GCP BigQuery export; OCI Cost Analysis)
- Data ingestion pipelines (FOCUS dataset generation; ETL from vendor formats; quality assurance)
- Data analysis workflows (querying FOCUS datasets; SQL and analytics tools; cost allocation scenarios)
- Showback and chargeback (department/team attribution; cost transfer mechanics; reporting patterns)
- Multi-cloud cost normalization (cross-provider comparison; vendor-agnostic cost metrics; unified cost reporting)
- Business intelligence and cost optimization (cost anomaly detection; forecast and variance analysis; cost trend identification)
- FOCUS in cloud-native and Kubernetes environments (container cost allocation; resource tagging; FOCUS in FinOps dashboards)

## Skills validated

*Cert-specific — what this exam actually tests.*

- FOCUS specification structure, schema, and versioning (columns, measures, dimensions; provider conformance)
- Cloud provider billing data (AWS Cost and Usage Report; Azure Billing API; GCP BigQuery export; OCI Cost Analysis)
- FOCUS data ingestion and preparation (ETL workflows; data validation; quality assurance for billing conformance)
- Cost data analysis and querying (SQL-based cost attribution; aggregation and slicing by dimension)
- Showback and chargeback mechanics (cost allocation; stakeholder reporting; business unit cost transfer)
- Multi-cloud cost normalization and reconciliation (vendor-agnostic cost metrics; cross-cloud comparison)
- FinOps in containerized and cloud-native environments (Kubernetes cost allocation; container metrics; cluster tagging)
- Cost optimization workflows using FOCUS data (anomaly detection; forecast variance; trend analysis; optimization recommendations)

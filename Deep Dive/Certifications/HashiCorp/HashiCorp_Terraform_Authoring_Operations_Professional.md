## About

The Terraform Authoring and Operations Professional certification validates advanced expertise in both Terraform configuration development and production operations. It is positioned as the next step above the Associate-level certification (003/004), targeting practitioners with extensive hands-on experience building dynamic HCL modules, managing remote state at scale, and establishing collaborative workflows in HCP Terraform (HashiCorp Cloud Platform). Unlike the Associate exam, which tests foundational knowledge, the Professional exam uses scenario-based labs where candidates must actually provision infrastructure, modify configurations, and manage state files in a proctored Linux environment. It is a role-focused certification designed for infrastructure engineers, platform engineers, and DevOps professionals who architect, build, and operate Terraform-based infrastructure at production scale.

## Topics covered

The exam assesses mastery across the following domains:

- **HCL Language & Syntax** — Complex functions, expressions, meta-arguments, dynamic blocks, custom validation, and conditional logic
- **Module Authoring & Design** — Module structure, variable/output scoping, module versioning, refactoring strategies, and module composition patterns
- **Dynamic Configuration** — Using data sources, outputs, locals, and conditional resource creation to build reusable infrastructure templates
- **Terraform CLI & Workflows** — Advanced CLI operations, state inspection, targeted operations, debugging, and refactoring workflows
- **State Management** — Remote state backends, state locking, state data sources, state splitting strategies, and cross-configuration state sharing
- **HCP Terraform Workspace Management** — Creating and organizing workspaces, variable assignment (environment variables, variable sets), workspace access controls, and team governance
- **Dynamic Credentials & Authentication** — Configuring dynamic provider credentials within HCP Terraform, role assumption, and secrets management patterns
- **Policy as Code (Sentinel)** — Writing and testing Sentinel policies, policy sets, enforcement levels, policy testing, and integration with HCP Terraform run workflow
- **Run Workflow & Automation** — Cost estimation, policy enforcement, run tasks, run triggers, and VCS integration for GitOps workflows
- **Terraform Cloud/Enterprise Operations** — Team access management, permissions scoping, audit logging, and multi-account/multi-cloud scenarios
- **Complex Infrastructure Patterns** — Multi-region deployments, workspace organization strategies, and state refactoring at scale

## Skills validated

*Specific technologies and practices this exam tests, distinct from general DevOps competencies.*

- **Terraform CLI** (v1.2+) — apply, plan, state, workspace, import, force-unlock, validate, fmt
- **HCL syntax & functions** — splat expressions, for loops, conditionals, try(), merge(), jsondecode(), local functions
- **Module authoring** — root modules, child modules, module composition, variable/output scoping, module registry integration
- **Terraform state** — state backends (S3, Terraform Cloud, Azure Storage, GCS), state locking, state inspection, state refactoring
- **HCP Terraform (TFC/TFE)** — workspace creation, variable sets, run tasks, policy sets, VCS integration, team access management
- **Sentinel policy language** — policy rules, imports, policy testing, test framework, policy sets and enforcement levels
- **Dynamic credentials** — provider configuration with assume_role, workload identity, external data sources, secrets engines
- **Provider configuration & authentication** — multiple provider instances, alias, for_each with providers, provider functions (v1.8+)
- **Data sources & locals** — data source queries, computed values, local blocks for reusable expressions
- **Advanced resource meta-arguments** — count, for_each, depends_on, lifecycle, moved blocks, preconditions/postconditions
- **Terraform Cloud operations** — cost estimation, drift detection, policy enforcement, run workflows, webhooks, API-driven runs
- **Cost management** — resource costs via Terraform Cloud, cost policy enforcement, right-sizing recommendations

## About

TA-003 validated foundational Infrastructure as Code (IaC) skills using HashiCorp Terraform Community Edition. Launched October 2022 as the primary Terraform Associate certification; it was designed for cloud engineers, operations specialists, and developers who understand basic Terraform concepts and the core workflow (init, plan, apply, destroy). The exam tested practical knowledge of Terraform CLI, modules, state management, providers, and workspaces, with awareness of HCP Terraform (Terraform Cloud) capabilities. Retired January 8, 2026, in favor of TA-004 (which introduces Terraform 1.12 language features, ephemeral values, and HCP governance).

## Topics covered

From the official TA-003 exam blueprint:

- **Terraform purpose and advantages** — IaC concepts, declarative vs. imperative, tool differentiation
- **Terraform basics and workflow** — init, plan, apply, destroy, and core CLI operations
- **Terraform configuration language (HCL)** — syntax, blocks, interpolation, functions, data sources
- **Terraform modules** — module structure, inputs/outputs, registry usage, versioning
- **State and backends** — local state, remote backends (S3, TFC, Consul, Terraform Cloud), state locking, workspaces
- **Terraform providers** — provider configuration, authentication, multiple-provider scenarios
- **Variables, locals, and outputs** — input variable types, defaults, validation, sensitive data handling
- **HCP Terraform (Terraform Cloud/Enterprise) awareness** — feature differences vs. Community Edition, VCS integration, remote runs

## Skills validated

*Distinct from shared domain skills; what TA-003 specifically tested.*

- **Terraform CLI mastery** — init, plan, apply, destroy, taint, import, fmt, validate
- **HCL syntax and interpolation** — resource declaration, variable interpolation, conditionals, for loops, splat syntax
- **Module design and composition** — input variables, outputs, locals, module sources, registry versioning
- **State file management** — local state, remote backends (S3, Azure Blob, Google Cloud Storage), state locking, state mv/rm
- **Provider configuration and authentication** — AWS IAM roles, Azure Service Principal, GCP SA keys, multiple provider blocks
- **Workspace management** — isolated state per workspace, workspace-scoped variables, use cases (dev/staging/prod)
- **Data sources and dynamic content** — querying existing cloud resources, dynamic block construction
- **Terraform Cloud/Enterprise features** — VCS integration, remote runs, cost estimation, policy as code (Sentinel)
- **Security and best practices** — sensitive variable masking, secrets management, code review workflows, module design

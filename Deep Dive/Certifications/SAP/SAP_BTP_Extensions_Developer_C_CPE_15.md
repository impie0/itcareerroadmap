## About

The SAP Certified Associate – Backend Developer certification validates knowledge of SAP's Cloud Application Programming Model (CAP) and the ability to design and deploy side-by-side extension applications on SAP Business Technology Platform (BTP). Launched in 2024 as successor to the retired C_CPE_14 (SAP BTP Extensions Developer), this Associate-level cert targets developers building ABAP Cloud and Node.js/Java back-end services using CAP, CDS (Core Data Services), OData APIs, and Fiori Elements integration. Candidates typically hold 1–2 years of hands-on SAP BTP or S/4HANA extension experience and are moving into specialist SAP cloud roles.

## Domain context — Software Engineering

Full-stack cloud-native development on vendor platforms, spanning runtime architecture, persistence, API design, and security in production. SAP BTP is a tier-1 enterprise cloud platform; CAP is the canonical SAP extension framework.

## Topics covered

The exam validates core competency in these areas:

- SAP BTP Architecture & Cloud Foundry runtime (12–15%)
- Kyma Runtime, serverless, and event-driven extensions (8–10%)
- SAP Cloud Application Programming Model (CAP) fundamentals (18–22%)
- Core Data Services (CDS) — data modeling, views, aspects (15–18%)
- ABAP Cloud & remote services integration (10–12%)
- Node.js & Java runtime stacks in CAP (10–12%)
- OData services & REST API design (8–10%)
- Fiori UI integration & content delivery (8–10%)
- Authorization, trust management, role-based access (8–10%)
- Workflow & side-by-side extensibility patterns (8–10%)

## Skills validated

- **SAP BTP Architecture**: Cloud Foundry concepts, networking, security zones, subaccounts, service binding
- **SAP Kyma Runtime**: Serverless, event mesh, function deployment, Kubernetes basics
- **SAP CAP (Cloud Application Programming Model)**: project scaffolding, full-stack development
- **Core Data Services (CDS)**: entity definitions, views, calculated elements, annotations, composition
- **Node.js & Java Runtimes**: cds-based service deployment in both languages, package management
- **ABAP Cloud & Remote APIs**: calling ABAP services from CAP, oData consumption, RFC/SOAP interop
- **OData Fundamentals**: v2/v4, query options, error handling, $metadata
- **Fiori Elements & UI5**: layout aggregation, custom pages, data binding, form validation
- **Authorization & Trust**: OAuth 2.0 / SAML, XSUAA, role-based access control (RBAC), token exchange
- **Persistence & Databases**: SQLite (local dev), SAP HANA, PostgreSQL, multi-tenancy
- **Workflow & BPM Integration**: workflow task definitions, LoB process extensibility
- **Testing & Debugging**: CAP test library, cds CLI, local sandboxes, logs
- **Deployment & Operations**: Cloud Foundry CLI, manifest files, environment variables, monitoring

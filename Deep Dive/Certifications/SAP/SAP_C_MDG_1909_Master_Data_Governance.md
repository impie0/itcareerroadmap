## About

The **C_MDG_1909** certification validates fundamental and applied knowledge in SAP Master Data Governance — a core data management discipline within the broader SAP ecosystem. Released in 2019 and actively maintained through 2026, this exam confirms the holder's ability to participate in MDG implementation projects under mentorship, configure change-request workflows, design data models, execute data quality validations, and manage data replication across systems. It is positioned as an entry-level professional credential for functional and techno-functional consultants transitioning from general SAP work (FI, MM, SD) into specialized master-data roles.

---

## Domain context — Data Governance

Data governance spans policies, processes, tools, and accountability structures that ensure data quality, integrity, security, and regulatory compliance across an organization. Within SAP ecosystems, Master Data Governance (MDG) is the vendor's centralized solution for managing golden-record creation, stewardship, and synchronization across ERP, CRM, supply-chain, and external systems. MDG consultants typically sit at the intersection of business process, IT operations, and data architecture teams.

## Topics covered

The exam blueprint distributes weight across the following domains (~12–18% each):

- **MDG Overview and Architecture** — SAP MDG technical stack, deployment options (on-premise vs. cloud), entity types (Reuse, Flex, Custom), system landscape integration
- **Data Modeling & Consolidation** — data entity definition, hierarchy modeling, consolidation logic, best-record algorithms, scoring and duplicate detection
- **Change Request Process** — workflow configuration, approval routing, status tracking, versioning, activation of golden records
- **Data Quality & Analytics** — validation rules, search-help configuration, side-panel activation, quality metrics, data auditing
- **Data Replication & Key Mapping** — replication scenarios (push/pull), key mapping strategies, system communication, Delta Link setup
- **MDG Mass Processing** — batch operations, multi-object processing, data import/export workflows
- **Business Rules Framework (BRF+)** — rules engine for data validation and conditional workflow routing
- **User Interface Configuration** — Fiori UI design for MDG, field layout customization, role-based access

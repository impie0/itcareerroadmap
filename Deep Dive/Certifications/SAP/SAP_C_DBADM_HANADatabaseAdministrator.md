## About

SAP HANA is SAP's in-memory computing platform that powers next-generation business applications. The Database Administrator certification validates your ability to install, configure, monitor, and maintain SAP HANA systems in both on-premise and cloud (SAP HANA Cloud) environments.

This certification demonstrates expertise in:
- HANA system installation and setup
- Database configuration and optimization
- Performance monitoring and troubleshooting
- Backup and disaster recovery
- User management and security
- System administration and operations
- HANA Cloud provisioning and administration
- High availability and scale-out configurations

The C_DBADM certification is essential for organizations running HANA databases in production environments, ensuring data integrity, performance, and availability.

## Domain context

SAP HANA has become the foundational technology for SAP's modern enterprise resource planning solutions, including S/4HANA. As organizations migrate to these systems, the demand for skilled HANA database administrators continues to grow.

Organizations using this certification typically:
- Are implementing or maintaining S/4HANA systems
- Have mission-critical HANA databases in production
- Require high availability and disaster recovery
- Need performance optimization expertise
- Use SAP HANA Cloud for cloud-native deployments
- Integrate HANA with business intelligence tools
- Require security and compliance management

## Topics covered

The C_DBADM_2404 exam is organized into 7 major topic areas (each 11%-20% of exam):

1. **SAP HANA Architecture** (11% - 20%)
   - HANA system architecture overview
   - In-memory computing concepts
   - Row and column store technology
   - HANA Cloud architecture
   - System components (Index Server, Name Server, etc.)
   - Distributed HANA systems
   - Scale-out and scale-up configurations
   - Replication and federation concepts

2. **Provisioning SAP HANA** (11% - 20%)
   - HANA system installation
   - SAP HANA Cloud provisioning
   - Database creation
   - File system and storage configuration
   - Network configuration
   - Security and SSL setup
   - Tenant database creation
   - Pre-installation requirements

3. **Database Administration** (11% - 20%)
   - Database startup and shutdown
   - Database configuration parameters
   - Tablespace and data volume management
   - Memory management and allocation
   - Performance parameters tuning
   - Catalog and system tables
   - Database properties and settings
   - Encryption and compression options

4. **Database Security** (11% - 20%)
   - User and role management
   - Authentication methods (LDAP, SAML, Kerberos)
   - Authorization and privileges
   - SQL permissions and object privileges
   - Audit and compliance
   - Data protection and encryption
   - Password policies
   - Single sign-on (SSO) configuration

5. **Backup and Recovery** (11% - 20%)
   - Backup types and strategies
   - Backup scheduling
   - Recovery procedures
   - Point-in-time recovery
   - Disaster recovery planning
   - Backup tools and utilities
   - Recovery testing
   - Data retention policies
   - Archive logs management

6. **Monitoring and Troubleshooting** (11% - 20%)
   - System monitoring tools
   - Performance analysis and tuning
   - Alert configuration
   - Log file analysis
   - Health check reports
   - Resource utilization monitoring (CPU, memory, disk)
   - Database locks and blocking
   - Error diagnostics

7. **User Management** (11% - 20%)
   - User creation and deletion
   - Role and privilege assignment
   - User authentication setup
   - User activity monitoring
   - Session management
   - System privileges and object privileges
   - User profile configuration
   - Multi-tenancy user management

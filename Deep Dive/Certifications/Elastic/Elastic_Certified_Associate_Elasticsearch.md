## About

The Elastic Certified Associate — Elasticsearch validates foundational competency in Elasticsearch cluster operations, data indexing, and search query fundamentals. This performance-based hands-on lab exam requires candidates to complete real-world operational tasks on a live Elasticsearch cluster within a 2-hour window, covering core topics: cluster setup, data ingestion, basic search queries, index management, and cluster health monitoring. The Associate level is an entry point for IT professionals transitioning into Elasticsearch roles or validating hands-on skills before pursuing the Professional-level Elastic Certified Engineer (ECE) credential. Valid for 2 years; candidates have 1 year from purchase to use their exam attempt.

## Domain context — Search & Analytics

Elasticsearch powers full-text search, log analytics, and real-time data exploration across applications and infrastructure. At the Associate level, practitioners validate capability to deploy single-node and basic multi-node Elasticsearch clusters, index documents, construct simple and Boolean queries, and troubleshoot fundamental cluster issues. This aligns with entry-to-mid-level roles: Elasticsearch Support Engineer, Junior Search Infrastructure Specialist, Data Engineer (early-career), and IT technicians gaining Elastic Stack competency.

## Topics covered

The Elastic Certified Associate exam validates proficiency across five primary domains:

- **Cluster Deployment & Configuration** — Single-node and two-node cluster setup, node configuration (roles: master, data, ingest), bootstrap security (initial password, TLS), Elasticsearch on Docker/Kubernetes basics, cluster startup and health checks
- **Data Indexing & Ingest** — Document creation via REST API, bulk indexing, index creation, dynamic templates, refresh intervals, basic ingest pipelines, error handling and troubleshooting ingestion failures
- **Search & Query Fundamentals** — Match queries, bool queries (must, should, filter), exact-match searches, full-text search basics, range queries, sorting results, pagination (from/size), basic aggregations (terms, sum, avg)
- **Index & Cluster Administration** — Index lifecycle (create/delete/close), shard and replica basics, node addition/removal, basic cluster settings, _cat APIs for health and status, index status indicators (green/yellow/red)
- **Monitoring & Troubleshooting** — Kibana Dev Tools and Console, basic cluster health monitoring, identifying slow queries, log file interpretation, common error messages, rebalancing basics

## About

The Workday Pro Reporting Specialist certification validates practical expertise in building operational and analytical reports within Workday's HCM and Finance modules. It is intended for HRIS analysts, business analysts, and reporting professionals responsible for planning and executing reporting strategies in organizations using Workday. Certified professionals demonstrate mastery of Workday's report writer, calculated fields, BIRT (Business Intelligence and Reporting Tools), dashboards, and composite reporting functionality. The Workday Pro program launched in 2021 and represents Workday's practical, employer-focused credentialing approach, emphasizing applied configuration and operational tasks over abstract theory.

## Topics covered

The Workday Pro Reporting Specialist exam covers hands-on reporting skills within the Workday platform, testing candidates' ability to design, build, and optimize both operational and analytical reports.

- **Calculated Fields** — Creating report-specific and system-wide calculated fields using Workday expression language; deriving new values from existing data (e.g., tenure calculations, commission formulas)
- **Report Writer Fundamentals** — Building custom reports using Workday's native report writer; data source selection; filtering, grouping, and sorting
- **BIRT Reporting (Business Intelligence & Reporting Tools)** — Advanced report design using BIRT; rich text formatting, headers, footers, page breaks, exact label placement
- **Composite Reports** — Combining multiple sub-reports with different data sources and layouts into single consolidated reports; multi-dimensional analysis across organizational departments
- **Dashboards** — Designing and deploying dashboards; leverage drag-and-drop tools; pivoting and dynamic report manipulation
- **Workday Business Object Model** — Understanding Workday's data structure; business objects, data sources, relationships; navigating the instance to identify available report fields
- **Report Security & Access Control** — Implementing row-level security (RLS); report filtering based on organizational hierarchy; field-level access; role-based report visibility
- **Data Filtering & Trending Analysis** — Advanced filtering techniques; time-series analysis; trending reports; performance tracking over time
- **HCM-Specific Reporting** — Reporting on workforce data: headcount, turnover, compensation, benefits, time tracking, talent management, organizational structure
- **Financial Reporting** — Reporting on GL transactions, journals, budgets, expenses, cost allocations, and financial performance within Workday Finance
- **Report Scheduling & Distribution** — Scheduling automated report runs; delivering reports via email, file storage, or delivery services; managing report frequency and recipients

## Skills validated

_Certification-specific technologies and competencies tested in this exam._

- **Workday Report Writer** — Building standard reports with filters, grouping, sorting, drill-down functionality
- **Workday Calculated Fields** — Expression language syntax; conditional logic; mathematical operations; date/time calculations
- **BIRT (Business Intelligence & Reporting Tools)** — Rich text formatting; page layout design; header/footer customization; exporting to PDF, Excel, CSV
- **Workday Dashboards** — Widget configuration; drag-and-drop UI; refresh scheduling; interactive pivoting
- **Composite Reports** — Multi-source report design; sub-report parameters; data synchronization across report sections
- **Workday Business Object Model** — Navigating the object tree; identifying reportable fields; understanding data sources (transactional, snapshot, custom)
- **Row-Level Security (RLS)** — Implementing org hierarchy filters; location-based access; manager-based filtering
- **Data Extraction & Transformation** — Using Workday's data tools; CSV/JSON export; ETL fundamentals within Workday
- **Workday HCM Data** — Worker master data, compensation, benefits, time tracking, absence management, performance management
- **Workday Finance Data** — GL account hierarchy, journal entries, invoices, POs, expense reports, cost allocations
- **Report Performance Optimization** — Query efficiency; caching strategies; scheduled report best practices

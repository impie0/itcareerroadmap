## About

The Workday Pro Security Administrator certification validates expertise in implementing, configuring, and maintaining Workday's configurable security framework. Candidates demonstrate applied knowledge of security groups (role-based, user-based, segment-based, location, rule-based, aggregation, intersection, and service-center types), domain security policies, business process security policies, role-based access control (RBAC), and security audit and compliance reporting. This professional-level credential is designed for Workday administrators, consultants, and support personnel responsible for designing and troubleshooting user access controls within Workday HCM and Finance environments. The exam tests practical ability to configure security models that enforce segregation of duties, manage access permissions, and maintain audit trails.

## Topics covered

The Workday Pro Security Administrator exam blueprint covers the following knowledge areas:

- **Security Architecture & Framework** — Workday configurable security model, security domain structure, and permission hierarchy
- **Security Group Types** (approximately 20% of exam) — Role-based, user-based, segment-based, location membership, rule-based, aggregation, intersection, and service-center security groups
- **Domain Security Policies** (approximately 25% of exam) — Designing and maintaining domain policies that assign View, Read, and Modify permissions to security groups across Workday domains (e.g., Worker Data, Compensation, Benefits, Payroll)
- **Business Process Security Policies** (approximately 20% of exam) — Configuring who can initiate, intervene in, and complete business process steps; participation rules and step transitions
- **Role-Based Access Control (RBAC)** (approximately 15% of exam) — Implementing RBAC patterns, constrained access, functional security roles, and supervisor hierarchies
- **Access Control Troubleshooting** (approximately 10% of exam) — Diagnosing permission issues, testing security policies, validating access, and resolving common RBAC problems
- **Audit & Compliance Reporting** (approximately 10% of exam) — Security history audits, user access reports, compliance documentation, segregation of duties (SoD) validation, and security change tracking

*Source:* [Workday Pro Certification information ↗](https://cloudshinepro.com/workday-pro-certification/), [Workday security configuration guides ↗](https://www.reco.ai/hub/configuring-workday-security-role-based-permissions)

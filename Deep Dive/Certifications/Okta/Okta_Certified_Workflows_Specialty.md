## About

The Okta Certified Workflows - Specialty credential validates the ability to design and implement identity automation solutions using the Okta Workflows no-code platform. Okta Workflows allows identity and access management professionals to automate repetitive identity processes—user provisioning, deprovisioning, password management, group rule automation, MFA enrollment, and delegated administration—without writing custom code. The exam tests proficiency with Okta Workflows architecture (flows, connectors, actions, cards), building conditional logic and error handling, integrating with external systems via connectors and HTTP calls, implementing scheduled flows and event-based triggers, and monitoring and troubleshooting flow execution. Candidates must pass a two-part exam: 30 multiple-choice questions assessing conceptual and procedural knowledge, followed by four hands-on configuration use cases requiring real-time design and deployment of Okta Workflows. This is a specialist-level credential suited for identity operations engineers, automation architects, and IAM professionals who drive efficiency through identity automation. The certification is increasingly important as organizations adopt zero-touch provisioning, self-service identity governance, and event-driven identity architectures.

## Topics covered

*Based on the Okta Certified Workflows - Specialty Exam Study Guide:*

- **Okta Workflows architecture and fundamentals** — Workflows platform overview, flows vs. connectors vs. actions vs. cards, flow types (on-demand, scheduled, triggered), event sources and event attributes
- **Building flows with no-code cards** — Using action, condition, and utility cards, card configuration and chaining, data passing between cards, variable management
- **Okta connectors** — Built-in Okta connector (user provisioning, group management, custom object operations), third-party app connectors (Slack, Microsoft, Salesforce), custom connector setup and API integration, authentication for connectors
- **HTTP and API calls within Workflows** — Making outbound HTTP requests, API authentication (Basic, OAuth, API keys), request/response mapping, handling API rate limits and errors
- **Error handling and conditional logic** — Error handling cards (try-catch patterns), condition cards (if-else, switch), loop cards (for-each), validating data before actions, fallback paths and notifications
- **User lifecycle automation with Workflows** — Pre-hire provisioning, onboarding workflows, role and access grant automation, deprovisioning and access removal, password reset and MFA enrollment automation, group rule delegation
- **Okta Expression Language (OEL) in Workflows** — Using expressions for data transformation, string/array/date manipulation, conditional expressions, calling Okta APIs with expressions
- **Scheduled and time-based flows** — Cron-based flow scheduling, time zone handling, scheduled provisioning and cleanup tasks, frequency and concurrency control
- **Delegated administration via Workflows** — Empowering admins to run custom flows via Okta admin console, restricting flow execution to specific users or groups, self-service workflows for end users
- **Monitoring, logging, and debugging Workflows** — Flow execution logs and audit trails, debugging flow issues with log output, monitoring flow health and success rates, alerting on flow failures
- **Advanced topics** — Workflows for custom provisioning rules, flows for access reviews and certifications, multi-tenant and cross-org workflows, best practices for complex flow design

*Source:* [Okta Certified Workflows - Specialty Exam Study Guide ↗](https://certification.okta.com/page/workflowscertexamstudyguide)

## About

AWS Certified Advanced Networking – Specialty validates deep expertise in designing, implementing, and managing complex hybrid and multi-cloud network architectures on AWS. Launched July 2022 as the successor to ANS-C00, this certification targets network architects and cloud infrastructure engineers with 5+ years networking background. It emphasizes VPC design at scale, Direct Connect deployment patterns, SD-WAN integration, Route 53 hybrid DNS resolution, and network security governance — making it distinct from architect certs that touch networking as one domain.

## Topics covered

Official exam blueprint weights (4 domains):

- **Network Design (30%)** — VPC architecture, hybrid connectivity models, Transit Gateway design, Direct Connect, SD-WAN, multi-account networks
- **Network Implementation (26%)** — VPC peering, TGW setup, Direct Connect MACsec/LAG/BGP, VPN acceleration, Route 53 inbound/outbound endpoints, Network Firewall
- **Network Management and Operation (20%)** — Network Manager, CloudWatch metrics, VPC Flow Logs, route optimization, traffic analysis, cloud operations
- **Network Security, Compliance, and Governance (24%)** — IAM network policies, VPC endpoints (interface/gateway/GWLB), WAF, Shield, GuardDuty network insights, traffic mirroring, compliance automation

*Source:* [AWS Certified Advanced Networking Specialty Exam Guide ↗](https://d1.awsstatic.com/training-and-certification/docs-advnetworking-spec/AWS-Certified-Advanced-Networking-Specialty_Exam-Guide.pdf)

## Skills validated

*Cert-specific — what ANS-C01 actually tests, distinct from shared Networking · Expert skills above.*

- **VPC peering vs. Transit Gateway:** Architecture selection, shared TGW design patterns, multi-account sharing via RAM
- **Transit Gateway (TGW) advanced:** TGW Connect for SD-WAN, Central Egress, RouteThru Transit, cross-region peering
- **AWS Network Manager:** Visual topology, on-prem link integration, intent-based routing, cloud WAN
- **AWS Direct Connect:** Private/Public/Transit VIFs, MACsec encryption, LAG (Link Aggregation Group), BGP communities, resilience patterns
- **Direct Connect resilience:** Multi-site active-active, failover strategies, BFD integration, prefixes announced
- **Site-to-Site VPN:** BGP dynamic routing, VPN acceleration (accelerated VPN), ECMP (equal-cost multipath), tunnel optimization
- **AWS Cloud WAN:** Centralized policy application, multi-cloud integration, SD-WAN as alternative transport
- **Route 53 hybrid DNS:** Inbound/outbound Resolver endpoints, hybrid.example.com patterns, multi-account DNS architecture
- **Route 53 Application Recovery Controller:** Cross-region failover automation, endpoint evaluation, readiness checks
- **Global Accelerator:** Static IP, DDoS protection, sub-millisecond routing, traffic steering
- **CloudFront:** Origin shield, geo-restriction, cache behaviors, Lambda@Edge security
- **Network Firewall:** Stateless/stateful rules, IDS/IPS integration, domain name filtering, encryption inspection
- **AWS WAF:** Rate-limiting, geo-blocking, IP reputation lists, rule groups
- **Shield Advanced:** DDoS detection + response team (DRT), cost protection, real-time alerts
- **VPC Flow Logs:** Capture at ENI/subnet/VPC level, analysis patterns, Security Hub integration
- **GuardDuty for VPC:** Network findings (unusual DNS queries, crypto mining, lateral movement), threat intelligence
- **VPC endpoints (Interface):** PrivateLink for SaaS access, endpoint policies, DNS resolution per-AZ
- **VPC endpoints (Gateway):** S3/DynamoDB selective routing, resource policies, multi-VPC access
- **Gateway Load Balancer (GWLB):** Network virtual appliances, appliance auto-scaling, cross-zone support
- **Traffic Mirroring:** Packet capture for IDS/IPS, SPAN equivalent, source/target rules
- **IPv6 dualstack VPC:** Dual addressing, ENI IPv6 assignment, egress-only IGW, dual-stack load balancing
- **Multi-account networking:** Service Control Policies (SCPs) for network compliance, CloudFormation StackSets for TGW expansion
- **AWS Cloud Map:** Service registry for hybrid networks, health checks, DNS-based service discovery
- **BGP path selection:** AS path prepending, local preference, MED (Multi-Exit Discriminator), weight
- **Hybrid DNS resolution:** .local vs. .internal, conditional forwarders, split-view DNS at scale
- **Network performance tuning:** Jumbo frames (MTU 9000), TCP window scaling, EC2 placement groups, network acceleration
- **Terraform/CloudFormation for IaC:** VPC module composition, TGW resource automation, drift detection

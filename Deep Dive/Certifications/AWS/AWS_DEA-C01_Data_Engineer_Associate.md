## About

The AWS Certified Data Engineer – Associate (DEA-C01) validates your ability to design, build, and manage data pipelines that process large volumes of information on AWS. Launched in beta in late 2023 and released as general availability in April 2024, DEA-C01 serves as the entry-level data engineering credential on AWS and replaces the Data Analytics Specialty (DAS-C01), which retired April 9, 2024. The exam emphasizes hands-on experience with ETL/ELT processes, data warehousing, data lakes, and orchestration using core AWS services. Target audience: data engineers (junior to mid-level), analytics engineers transitioning to AWS, and cloud professionals building data platforms.

## Topics covered

The exam covers four content domains with the following weightings (from official exam blueprint):

- **Domain 1: Data Ingestion and Transformation (34%)** — ETL/ELT design, ingest from multiple sources (JDBC, API, S3, DMS), transform using AWS Glue, Glue DataBrew, Glue Studio, Lambda; apply programming concepts
- **Domain 2: Data Store Management (26%)** — Choose optimal storage (S3, DynamoDB, Redshift), design schemas, partitioning strategies, data lifecycle management, optimize for cost and query performance
- **Domain 3: Data Operations and Support (22%)** — Orchestrate and monitor pipelines (Step Functions, EventBridge), troubleshoot failures, ensure data quality, analyze data with Athena and Redshift, implement cost optimization
- **Domain 4: Data Security and Governance (18%)** — Apply IAM policies, encryption (KMS), data cataloging (Glue Data Catalog), implement compliance, use Lake Formation for fine-grained access control

*Source:* [Official exam guide PDF ↗](https://d1.awsstatic.com/training-and-certification/docs-data-engineer-associate/AWS-Certified-Data-Engineer-Associate_Exam-Guide.pdf)

## Skills validated

*Concrete technologies and services tested on DEA-C01—what you must demonstrate hands-on experience with.*

- **AWS Glue** (ETL job creation, Glue DataBrew, Glue Studio, Data Catalog)
- **AWS Kinesis** (Data Streams, Firehose, Analytics)
- **AWS Lambda** (data transformation, event-driven pipelines)
- **Amazon S3** (object storage, data tiering, Intelligent-Tiering, S3 Select, performance)
- **Amazon Redshift** (warehouse architecture, Spectrum for S3 querying, Concurrency Scaling, RA3 clusters)
- **Amazon DynamoDB** (NoSQL design, streams, TTL)
- **Amazon Athena** (SQL queries on S3, partitioning, performance tuning)
- **AWS EMR** (Apache Spark, Hive, Hadoop clusters)
- **AWS DMS** (Database Migration Service for data ingestion from on-premises/RDS)
- **AWS Step Functions** (workflow orchestration, error handling)
- **AWS Lake Formation** (data lake setup, fine-grained access control)
- **AWS Data Pipeline** (legacy; deprecated; may appear in legacy questions)
- **AWS IAM** (data access policies, cross-account roles)
- **AWS KMS** (key management, encryption)
- **AWS Glue Data Catalog** (metadata, schema registry)
- **AWS Macie** (data discovery, PII detection)
- **EventBridge** (event-driven scheduling)
- **CloudWatch** (monitoring, logs, alarms)

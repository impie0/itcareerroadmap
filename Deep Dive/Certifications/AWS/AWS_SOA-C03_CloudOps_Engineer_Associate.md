# AWS Certified CloudOps Engineer – Associate

**`SOA-C03`** · ● **Active** · **Associate** · _AWS_

> The AWS Certified CloudOps Engineer – Associate (SOA-C03) validates your ability to deploy, manage, and operate workloads on AWS. Launched September 30, 2025, this exam replaced the AWS Certified SysOps Administrator – Associate (SOA-C02). The rebrand reflects expanded scope: containers (ECS, EKS), multi-account orchestration, infrastructure-as-code, and modern automation are now core.

## About

The AWS Certified CloudOps Engineer – Associate validates the technical knowledge and skills of cloud operations professionals, system administrators, and DevOps engineers who operate workloads on AWS. Launched September 30, 2025, this certification replaced the AWS Certified SysOps Administrator – Associate (SOA-C02), expanding its scope to include container orchestration (ECS, EKS, Fargate), infrastructure-as-code (CloudFormation, CDK, Terraform), multi-account and multi-Region architectures, and modern automation frameworks. The exam emphasizes hands-on operational skills: CloudWatch observability, AWS Systems Manager automation, compliance management, and business continuity. Recommended for engineers with 1–2 years of AWS operational experience and is a common stepping stone toward AWS DevOps Engineer Professional (DOP-C02).

## Topics covered

The SOA-C03 exam is organized into five content domains, each weighted as a percentage of scored content:

- **Domain 1: Monitoring, Logging, Analysis, Remediation, and Performance Optimization (22%)**
  - CloudWatch Metrics, Logs, Dashboards, and Alarms
  - Log aggregation and centralized logging
  - Performance troubleshooting and root-cause analysis
  - Cost optimization and AWS Trusted Advisor

- **Domain 2: Reliability and Business Continuity (22%)**
  - High availability and auto-scaling strategies
  - Backup and disaster recovery (RTO / RPO)
  - AWS Backup, Storage Gateway, Replication
  - Multi-AZ and multi-Region failover

- **Domain 3: Deployment, Provisioning, and Automation (22%)**
  - Infrastructure as Code (CloudFormation, CDK)
  - AWS Systems Manager (Patch Manager, Run Command, Session Manager)
  - EC2 Image Builder and automated deployments
  - Configuration management and state drift detection

- **Domain 4: Security and Compliance (16%)**
  - IAM identity and access management
  - Data encryption (at-rest and in-transit)
  - AWS Config and compliance monitoring
  - Security group and network ACL configuration

- **Domain 5: Networking and Content Delivery (18%)**
  - VPC design, subnets, routing, and NAT
  - DNS and Route 53 configuration
  - CloudFront distribution setup
  - VPN and hybrid connectivity (Direct Connect, Site-to-Site VPN)

## Skills validated

*Cert-specific — what SOA-C03 actually tests, distinct from broader cloud operations.*

- **CloudWatch** (Metrics, Logs, Alarms, Dashboards, Insights, Anomaly Detection)
- **AWS Systems Manager** (Patch Manager, Run Command, Session Manager, Parameter Store, State Manager)
- **Infrastructure-as-Code**: CloudFormation (templates, stacks, parameters, exports), AWS CDK, AWS Config (compliance rules)
- **EC2 Image Builder** and automated golden-image pipelines
- **Multi-AZ and multi-Region failover**: Auto Scaling, Route 53 health checks, RDS multi-AZ, Aurora global databases
- **Backup and disaster recovery**: AWS Backup, Storage Gateway, AWS DataSync, recovery time / recovery point objectives (RTO/RPO)
- **Container orchestration**: Amazon ECS, Amazon EKS, AWS Fargate (new in C03)
- **IAM Access Analyzer** and identity-based / resource-based policies
- **AWS Budgets, Cost Explorer, Trusted Advisor** for cost optimization
- **Network architecture**: VPC design, subnets, route tables, security groups, network ACLs, DNS (Route 53), CloudFront, VPN, Direct Connect
- **S3 versioning, lifecycle policies, object lock** for data protection
- **RDS** and relational database administration (backup windows, parameter groups, monitoring)
- **Lambda** automation and event-driven remediation
- **AWS Secrets Manager** and credential rotation
- **Compliance and audit logging**: CloudTrail, AWS Config, AWS Organizations for multi-account governance

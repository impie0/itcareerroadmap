## About

AWS Certified Solutions Architect – Professional (SAP-C02) is the flagship expert-level AWS credential for senior architects and principal engineers designing enterprise-scale, mission-critical solutions. Launched November 15, 2022, it supersedes SAP-C01 and reflects modern AWS services, organizational complexity patterns (multi-account strategy, Control Tower, landing zones), advanced networking (Transit Gateway, Network Firewall, Cloud WAN), and workload migration at scale. The exam validates deep hands-on experience with AWS Well-Architected Framework, cost governance, resilience patterns, and large-scale transformation initiatives. Industry pass rate approximately 55–65%; significantly harder than Solutions Architect Associate (SAA-C03).

## Topics covered

**Official exam blueprint weights (SAP-C02) — 4 scored domains:**

- **Design Solutions for Organizational Complexity (26%)** — Multi-account AWS environments, AWS Organizations, Service Control Policies (SCPs), AWS Control Tower, landing zone architecture, cross-account access (Resource Access Manager, IAM roles), compliance frameworks, scalable governance patterns
- **Design for New Solutions (29%)** — Greenfield architecture patterns, compute strategy (EC2, Fargate, Lambda, Outposts, Wavelength), storage architecture (S3, EBS, EFS, FSx), database selection (RDS, Aurora, DynamoDB, Redshift, Neptune), networking design (VPC design, Route 53, CloudFront, Direct Connect, VPN), security design (IAM, KMS, Secrets Manager, CloudHSM)
- **Continuously Improve for Existing Solutions (25%)** — Migration planning and execution (AWS Migration Hub, Application Migration Service, Database Migration Service, DataSync), cost optimization (Compute Optimizer, Right Sizing, Reserved Instances, Savings Plans, Spot Instances), performance tuning, operational excellence (CloudWatch, X-Ray, CloudTrail), disaster recovery and business continuity
- **Accelerate Workload Migration and Modernization (20%)** — Migration strategies (rehost, re-platform, re-architect, retire, repurchase), legacy system assessment, dependency mapping, parallelization patterns, data migration at scale (AWS Snow Family, DataSync, Storage Gateway), application modernization (containerization, serverless refactoring), cost-benefit analysis for migration sequencing

*Source:* [Official SAP-C02 Exam Guide ↗](https://docs.aws.amazon.com/pdfs/aws-certification/latest/solutions-architect-professional-02/solutions-architect-professional-02.pdf)

## Common skills at Cloud / Architecture · Expert

*Shared competencies for all architects at Expert level across cloud platforms.*

- Multi-cloud and hybrid strategy (AWS + Azure, AWS + GCP, or AWS-centric with on-premise)
- Cost governance at scale (chargeback models, showback/allocation, FinOps practices, TCO/ROI modeling)
- Enterprise governance and compliance (SOC2, ISO 27001, HIPAA, PCI-DSS, FedRAMP, GDPR data residency)
- Advanced networking architecture (SD-WAN, cloud interconnect, hybrid VPN/Direct Connect, network segmentation, zero-trust)
- Resilience and disaster recovery (RTO/RPO targets, multi-region active-active, failover automation, chaos engineering)
- Organizational design for cloud (two-pizza teams, cross-functional architecture reviews, guild/community models)
- Vendor strategy and platform roadmapping (make/buy/integrate decisions, CapEx vs. OpEx tradeoffs, competitive positioning)
- Executive communication and C-suite readiness (cloud ROI, business case development, risk quantification, strategic alignment)

## Skills validated

*SAP-C02-specific technologies and patterns tested on the exam.*

- **Organizational AWS (multi-account):** AWS Organizations, Service Control Policies (SCPs), AWS Control Tower, landing zone architecture (AWS Landing Zone Accelerator, Service Catalog)
- **Advanced networking:** Transit Gateway (TGW) routing, Network Firewall, AWS Cloud WAN, Direct Connect (connections, virtual interfaces, MACsec, gateway), VPN high availability (redundant VPN, Customer Gateway failover), Route 53 (health checks, geolocation routing, failover, traffic policies), PrivateLink, VPC endpoints
- **Governance and compliance:** AWS Config, AWS Compliance Manager, AWS Service Control Policies, cross-account IAM role assumption, Resource Access Manager (RAM), AWS Security Hub, Detective, CloudTrail for multi-account logging
- **Advanced security:** KMS multi-Region keys, Secrets Manager with rotation, CloudHSM, AWS Certificate Manager (ACM), WAF with custom rules and rate limiting, Shield Advanced, GuardDuty, Macie (data discovery)
- **Enterprise data architecture:** Redshift (Spectrum, Concurrency Scaling), Aurora (read replicas, global databases), DynamoDB (global tables, on-demand billing), Lake Formation, AWS Glue (data catalog, ETL), Amazon QLDB
- **Migration at scale:** AWS Application Migration Service (MGN), Database Migration Service (DMS) with CDC, AWS DataSync, AWS Storage Gateway (file, volume, tape), AWS Snow Family (Snowcone, Snowball, Snowmobile), AWS Migration Hub, Server Migration Service (SMS), AWS Datasync
- **Cost optimization:** Compute Optimizer recommendations, Right Sizing analysis, Reserved Instances (RIs) and Savings Plans (SP) purchasing strategies, Spot Instance fleets, Cost Anomaly Detection, AWS Budgets and Cost Explorer, Cost allocation tags, FinOps metrics (CUR analysis)
- **Disaster recovery:** Multi-Region active-active architecture, pilot light, warm standby, RTO/RPO targets, AWS Backup cross-Region copies, Amazon Route 53 failover, Application Load Balancer cross-zone, DynamoDB global tables, Aurora cross-Region read replicas
- **Performance at scale:** CloudFront (origin groups, geographic routing, origin shield), ElastiCache (Redis, Memcached, cluster mode), RDS (read replicas, Aurora read scaling), Kinesis (streams, firehose, analytics), SQS / SNS / EventBridge (async decoupling), Lambda (concurrency, reserved capacity)
- **Serverless at enterprise scale:** Lambda function versioning and aliases, layers (dependencies), VPC Lambda, Lambda@Edge, API Gateway (caching, throttling, request/response transformation), Step Functions (state machines, error handling, retries), AppSync (GraphQL managed service)
- **Hybrid and edge:** AWS Outposts (compute/storage on-prem), Wavelength (5G edge), Local Zones (metro extension), Site-to-Site VPN, AWS Direct Connect, Storage Gateway, Snow Family, Hybrid architectures with on-prem integration

## Exam preparation strategy

**Typical timeline:** 8–16 weeks (300+ study hours for full-time working professionals).

**Week 1–2: Baseline assessment**
- Review SAP-C02 exam guide (PDF, free from AWS)
- Take one free Tutorials Dojo practice exam to identify knowledge gaps
- Verify prerequisites: 2+ years hands-on AWS experience; SAA-C03 pass required

**Week 3–8: Core content**
- Complete Adrian Cantrill SAP-C02 course (40–60 hours video + hands-on labs)
- Deep-dive on weak domains: re-watch sections, take detailed notes, build lab environments
- Read AWS whitepapers: Well-Architected Framework, Cost Optimization Pillar, Resilience Pillar, Security Pillar
- Practice AWS CLI for multi-account scenarios, CloudFormation stack sets, cost estimation

**Week 9–14: Practice exams**
- Take 5–10 full Tutorials Dojo practice exams (timed, 180 min, under exam conditions)
- Review every incorrect answer; understand why each option is right/wrong
- Track performance by domain; re-study underperforming topics
- Aim for consistent 800+ scores on practice exams before attempting live exam

**Week 15–16: Final review**
- Review all ADRs (Architecture Decision Records) you've written in your job
- Re-watch Adrian Cantrill high-difficulty sections (multi-account, networking, migration)
- Mock exam day: take one final practice exam at exact exam time, same environment
- Skim AWS service release notes from past 6 months (new services rarely tested, but updated services appear)

**Critical success factors:**
- **Hands-on labs:** Don't just watch videos. Provision multi-account AWS environments, set up AWS Organizations, create Transit Gateway, configure Direct Connect, build DMS pipelines.
- **Business context:** SAP-C02 tests architectural thinking, not just AWS service knowledge. Understand the *why* behind decisions (cost vs. reliability vs. complexity tradeoff).
- **Scenario-based focus:** Exam questions are complex multi-part scenarios. Practice identifying business constraints, non-functional requirements, and hidden gotchas in question stems.

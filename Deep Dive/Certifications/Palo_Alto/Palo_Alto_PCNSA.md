# Palo Alto Networks Certified Network Security Administrator

**`PCNSA`** · ● **Retired** (March 31, 2025) · **Associate** · _Palo Alto Networks_

> **Retirement Notice:** The PCNSA was retired effective **March 31, 2025** as part of Palo Alto Networks' comprehensive certification restructure. Existing credential holders retain validity for two years from issue date. New candidates should pursue **Network Security Foundational** or **Network Security Professional** certs under the new role-based framework (launched August 1, 2025). This deep dive documents the historical PCNSA curriculum and its modern replacements.

## About

The Palo Alto Networks Certified Network Security Administrator (PCNSA) was the foundational certification validating hands-on knowledge of next-generation firewall (NGFW) deployment, configuration, and threat prevention on Palo Alto Networks platforms (primarily Strata firewall family). Launched in 2011 as the entry point into Palo Alto's certification ladder, PCNSA served as the gateway cert before PCNSE (professional level) and other specialist credentials. It covered core concepts including firewall architecture, policy management, threat prevention subscriptions, and Panorama multi-device management. The exam remained active for 13+ years across six major PAN-OS versions (7.x through 10.x), making it one of the industry's most widely held network security credentials. Its retirement in March 2025 marked the end of an era; Palo Alto Networks replaced the entire legacy exam structure with a modern role-based framework aligned with 2025 job market realities (network security, security operations, cloud security tracks).

## Retirement context — Certification Restructure (2024–2025)

**Timeline:**
- **Announced:** June 2024 — Palo Alto Networks published certification roadmap shift
- **Dual offering:** August 1, 2024 – March 31, 2025 — Both legacy (PCNSA/PCNSE) and new role-based certs available
- **Legacy sunset:** March 31, 2025 (PCNSA, PCCET, PCSFE) and July 31, 2025 (PCNSE, PCCSE, PCSAE, PCDRA)
- **New framework live:** August 1, 2025 — Full role-based ladder (Foundational → Professional → Specialist → Architect)

**Credential holders:** PCNSA holders retain full validity for **2 years from issue date**. Renewal after expiry requires earning a new role-based cert (e.g., Network Security Professional or NGFW Engineer).

## Topics covered (PCNSA Exam Blueprint)

The final PCNSA exam blueprint (PAN-OS 10.x) was weighted as follows:

- **Core Concepts (10%)** — Firewall architecture, OSI model review, stateful inspection, application-based routing, user identification methods (AD, LDAP, RADIUS, local databases)
- **Deploy and Configure Core Components (40%)** — NGFW interfaces and zones, security policies, NAT (source, destination, dynamic IP), address objects, service objects, threat prevention subscriptions, SSL/TLS decryption setup
- **Deploy and Configure Features and Subscriptions (28%)** — DNS security, URL filtering, User-ID integration, Syslog and log forwarding, GlobalProtect VPN, security profiles (antivirus, anti-spyware, vulnerability protection, file blocking, wildfire), app-id and traffic classification
- **Deploy and Configure Firewalls Using Panorama (12%)** — Multi-device management, template stacks, device groups, policy hierarchy, centralized log collection
- **Manage and Operate (10%)** — Firewall administration, high availability (active-active, active-passive), backup and restore, password policy, admin roles and RBAC, troubleshooting tools (packet capture, network traffic analyzer, CLI diagnostics)

*Source:* [PCNSA Final Exam Objectives (Community Archive)](https://live.paloaltonetworks.com/)

## Transition guidance (May 2026)

**If you hold an active PCNSA:**
1. Your credential is valid for **2 years from issue date** (not from now). Check your certificate.
2. Before expiry, pursue one of these **new Palo Alto role-based certs**:
   - **Network Security Professional** (breadth; 2–3 years experience recommended) — 90 min, $200 USD
   - **NGFW Engineer** (specialist depth in advanced NGFW) — 90 min, $250 USD
   - **Network Security Architect** (if 5+ years design/leadership experience) — 90 min, $300 USD
3. Use your hands-on firewall experience as foundation; new exams expect modern PAN-OS 11.x/12.x knowledge and cloud-native deployments (Prisma integration, Strata Cloud Manager).

**If you're studying for PCNSA (stopped taking new exams as of April 1, 2025):**
- Pivot to **Network Security Foundational** (entry-level, no experience required) or **Network Security Professional** (if you have hands-on NGFW time).
- PCNSA content remains relevant for foundational understanding, but new exams emphasize modern platform features and cloud-first architecture.
- Beacon free learning portal still hosts archived PCNSA study materials; cross-reference with new role-based curriculum.

## Historical significance

The PCNSA was one of the longest-running associate-level network security credentials, surviving 13+ years and six major platform revisions (PAN-OS 7.x through 10.x). Its retirement marks a watershed moment in the vendor-certification industry: as cloud-native architecture, SASE, and automated threat response reshape enterprise security, legacy exam-based credentials are being replaced with role-based frameworks that map directly to modern job functions. The PCNSA's transition represents both an end of era and a modernization imperative. Holders should leverage their hands-on firewall mastery to qualify for new role-based certs emphasizing cloud-first deployment, API-driven operations, and AI-augmented security. The knowledge remains evergreen; the credential structure does not.

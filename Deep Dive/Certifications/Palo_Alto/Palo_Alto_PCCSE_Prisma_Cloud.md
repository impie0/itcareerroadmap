## About

The Palo Alto Networks Certified Cloud Security Engineer (PCCSE) validated the knowledge, skills, and abilities required to onboard, deploy, and administer Prisma Cloud—Palo Alto's comprehensive cloud security platform. The exam covered Prisma Cloud (CSPM), Prisma Cloud Enterprise, and Prisma Cloud Compute, spanning cloud posture management, workload protection, compliance, and infrastructure-as-code security. Launched circa 2019, PCCSE became one of Palo Alto's primary cloud security credentials until its retirement July 31, 2025, replaced by role-based certifications aligned to Palo Alto's updated product taxonomy.

---

## Domain context — Cloud Security

Cloud Security encompasses the detection, investigation, and remediation of threats across public cloud platforms (AWS, Azure, GCP, OCI)—via Cloud Security Posture Management (CSPM), Container and Kubernetes security, Cloud Workload Protection (CWP), Cloud Network Security, Cloud Identity and Access Management (CIAM/CIEM), and Infrastructure-as-Code (IaC) scanning. Professional level assumes hands-on experience deploying and maintaining cloud security tools.

---

## Topics covered

Based on the official PCCSE exam blueprint, the certification assessed the following domains:

- **Cloud Security Posture Management (CSPM)** (~25%)
  - Asset discovery and inventory in cloud accounts
  - Policy creation and management
  - Compliance standard configuration
  - Automated remediation workflows

- **Cloud Workload Protection (CWP)** (~25%)
  - Protecting virtual machines and containers
  - Runtime threat protection
  - Vulnerability and compliance scanning
  - Workload visibility and segmentation

- **Cloud Network Security** (~20%)
  - Network segmentation and microsegmentation
  - Cloud firewall rules and policies
  - VPC/Virtual Network security
  - DDoS and threat prevention

- **Cloud Identity Security (CIEM) and Access Control** (~15%)
  - Cloud identity and access management
  - Privilege escalation detection
  - Key and credential management
  - Role-based access control (RBAC) across cloud platforms

- **Cloud Code Security and IaC** (~10%)
  - Infrastructure-as-Code scanning (Terraform, CloudFormation, ARM)
  - Secrets detection
  - Misconfigurations in code
  - CI/CD pipeline integration

- **Compliance and Reporting** (~5%)
  - Compliance framework reporting
  - Audit logs and visibility
  - Incident response integration

*Source:* [PCCSE Exam Blueprint PDF ↗](https://www.paloaltonetworks.com/content/dam/pan/en_US/assets/pdf/datasheets/education/pccse-blueprint.pdf)

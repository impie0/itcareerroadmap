## About

The CyberArk Certified Associate level (marketed as "Trustee") represents the foundational entry point in CyberArk's multi-tier certification path. This certification validates conceptual understanding of privileged access management (PAM), threat landscape context, and the core capabilities of CyberArk's Privileged Access Security (PAS) platform. It targets IT professionals, system administrators, security analysts, and career-changers entering the PAM domain—requiring no prior CyberArk experience. Launched as part of CyberArk University's structured certification program, the Associate tier serves as the prerequisite gateway to the Defender level (operational competency) and higher-tier credentials (Sentry, Guardian). As of November 2025, all CyberArk exams are administered in-person via Pearson VUE.

## Domain context — Identity & Access Security

Covers identity governance, privileged account management, credential lifecycle, secrets management, and zero-trust access control across hybrid and cloud environments. PAM is a critical subdomain focusing on the secure lifecycle of high-risk service accounts, human privileged identities, and just-in-time (JIT) access provisioning.

## Topics covered

Based on CyberArk's official curriculum for entry-level PAM certification:

- **Privileged Access Management (PAM) fundamentals** — threat model, password vaults, session management, least-privilege principle
- **CyberArk Privileged Access Security (PAS) platform overview** — core modules (Vault, CPM, PSM, EPM), architecture concepts
- **Credential and secrets lifecycle** — discovery, provisioning, rotation, deprovisioning
- **Identity threat analytics** — monitoring privileged activity, detecting anomalies
- **CyberArk ecosystem** — integration with SIEM, ticketing, cloud platforms (AWS, Azure, GCP)
- **Compliance and regulatory context** — HIPAA, PCI-DSS, SOX, CIS Controls alignment
- **Common PAM use cases** — database administration, infrastructure access, DevOps secrets, SaaS application credentials
- **Security best practices** — defense-in-depth, principle of least privilege, audit and accountability

## About

MuleSoft Certified Developer Level 1 validates foundational competency in building integration and API applications on the Anypoint Platform using Mule 4. Aimed at developers with basic to intermediate experience in enterprise integration, this Associate-level cert confirms hands-on ability to design, build, and deploy integrations using Anypoint Studio, connectors, transformers, and API-first design patterns. Launched in 2019 alongside the Mule 4 runtime, the cert remains the entry-level credential in the MuleSoft integration stack and is actively maintained by Salesforce.

## Topics covered

- **Anypoint Platform fundamentals** — Anypoint Exchange, Anypoint Design Center, Anypoint Runtime Manager, CloudHub
- **Anypoint Studio** — creating projects, flow design, debugging, project organization
- **Connectors and transports** — HTTP, File, Database, Salesforce, Workday, SAP; understanding transport vs. operation
- **DataWeave** — mapping, transforming payloads, functional programming (map, filter, reduce)
- **APIs and RAML** — RAML 1.0 API design, APIkit scaffolding, request/response validation, API versioning
- **Error handling and resilience** — error handlers, try-catch, async/blocking patterns, retry policy
- **Variables and expressions** — flow variables, object store, expression evaluation, MEL (Mule Expression Language) + DataWeave
- **Flows and sub-flows** — flow control, scatter-gather, batch processing, transaction scope
- **Deployment** — CloudHub, on-premises runtime, Docker, application properties, environment-specific configuration
- **Security and OAuth** — HTTPS, OAuth 2.0 client credentials, API key enforcement, policy enforcement

## Skills validated

*Cert-specific technologies and practices this exam actually tests.*

- **Anypoint Studio IDE** — project scaffolding, flow design, debugging, Maven dependency management
- **Anypoint Design Center** — API-first design, mock servers, fragment reuse
- **Mule 4 runtime** — flow execution, payload handling, event model
- **DataWeave 2.1** — transformation language, type system, functional operations
- **HTTP Connector** — request/response, headers, status codes, SSL/TLS
- **RAML 1.0 API specification** — types, traits, security schemes, APIkit scaffolding
- **Database Connector** — SQL queries, parameterized statements, connection pooling
- **Salesforce Connector** — standard objects, custom fields, batch operations, authentication
- **Error handling** — error handlers, error propagation, exception strategy patterns
- **Batch processing** — batch jobs, step tracking, aggregation
- **CloudHub deployment** — application properties, worker sizing, secure properties, monitoring
- **OAuth 2.0 client credentials** — token exchange, scope management, policy enforcement

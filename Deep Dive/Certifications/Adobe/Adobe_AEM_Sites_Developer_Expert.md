## About

The Adobe Certified Expert — AEM Sites Developer certification validates expertise in designing, implementing, and maintaining enterprise digital experiences using Adobe Experience Manager (AEM) Sites. Targeted at professional developers with 2+ years of hands-on AEM development, this expert-level certification demonstrates mastery of AEM architecture, component development, content management workflows, and production deployment patterns. Current as of 2026; vendor actively maintains and updates exam content quarterly.

## Domain context — Software Engineering / CMS

Enterprise content management and web platform development using proprietary vendor stacks. AEM represents a specialized, high-value segment of the broader CMS ecosystem, requiring deep knowledge of Java/Sling architecture, component patterns, and Adobe's deployment topology.

## Topics covered

Exam blueprint weighted as follows:

- **AEM Architecture & Core Concepts** (10%) — OSGi, Sling servlets, JCR repository model, node structure
- **Pre-sales & Discovery** (10%) — Stakeholder engagement, requirements gathering, solution fit assessment
- **Solution Design** (15%) — High-level design, component strategy, content modeling, authoring workflows
- **Implementation** (40%) — Component development (HTL/Sightly), backend services, bundles, APIs, testing, CI/CD integration
- **Solution Validation** (10%) — Testing strategies, performance profiling, security scanning, UAT coordination
- **Maintenance & Operations** (15%) — Upgrade paths, version management, troubleshooting, log analysis, monitoring
- **Best Practices & Standards** (10%) — AEM design patterns, accessibility (WCAG 2.1), responsive design, security hardening

## Skills validated

*This exam specifically tests the following technologies and concepts distinct from general software engineering:*

- **AEM Core** — OSGi bundles, Sling framework, JCR repository model, node types, access control lists
- **Component Development** — HTL (HTML Template Language), Sightly syntax, clientlibs, component dialogs, design systems
- **Content Models** — StructuredText nodes, asset management, metadata, versioning, workflow states
- **Java Backend** — Servlets, service interfaces, dependency injection (Declarative Services), bundle lifecycle
- **APIs & Integration** — Content API, Assets API, GraphQL endpoint configuration, third-party service connectors
- **Testing & Quality** — JUnit, Mockito, AEM testing libraries, performance profiling with JMeter
- **Deployment & DevOps** — Dispatcher configuration, replication agents, package deployment, CI/CD pipelines (Jenkins, Cloud Manager)
- **Security** — Authentication (SAML, OAuth), authorization, XSS prevention, CORS, content security policies
- **Performance** — Query optimization (JCR-SQL2), caching strategies, dispatcher rules, asset delivery optimization
- **Accessibility & UX** — WCAG 2.1 AA compliance, responsive design patterns, RTL support, screen reader testing

## About

The ServiceNow Certified System Administrator (CSA) is the entry-level professional credential for ServiceNow platform administration. Launched around 2012 and updated three times yearly to align with ServiceNow release cycles, it validates configuration and customization skills across the ServiceNow platform core modules: IT Service Management (ITSM), IT Asset Management (ITAM), Self-Service Catalog, and Reporting & Analytics. Recommended for IT operations professionals, service desk engineers, business analysts supporting ServiceNow deployments, and IT managers transitioning into ServiceNow ecosystem roles. No formal prerequisites; ServiceNow Now Learning free courses provide the primary self-study path.

ServiceNow-specific platform administration and ITSM configuration: instance setup and governance, role-based access control (RBAC), incident and request management, change management workflows, asset lifecycle tracking, service catalog design and automation, reporting and dashboarding, security and compliance controls within the platform.

## Topics covered

The official ServiceNow CSA exam blueprint covers these domain areas (approximate weightings):

- **User Interface and Navigation (10%)** — Platform UI fundamentals, navigation patterns, list/detail views, form layout, filters, personalization.
- **Collaboration (15%)** — Communication tools, chat, social feeds, team collaboration workflows, notifications.
- **Self-Service and Process Automation (15%)** — Service Catalog, Request Fulfillment, approval workflows, SLA configuration, workflow orchestration basics.
- **Database Administration (20%)** — Tables, records, fields, relationships (foreign keys), data types, data validation, duplicate handling, data governance.
- **Reporting and Analytics (15%)** — Reports (list, pivot, executive), dashboards, charts, filters, report groups, scheduling and distribution.
- **Service Catalog (15%)** — Catalog items, variables, item options, workflows linked to catalog items, portal configuration.
- **Tasks and Task Management (10%)** — Incidents, Requests, Changes, Problems, Task workflows, state transitions, assignment logic.

## Common skills at IT Service Management (ITSM) · Associate

*Shared foundational ITSM administration skills at the Associate level — not specific to ServiceNow.*

- ITSM process fundamentals (incident, request, change, problem management per ITIL v3/v4 concepts)

## Skills validated

*What the CSA exam specifically tests, distinct from shared Associate-level ITSM skills.*

- **Instance setup and governance** — Default configurations, system properties, user account creation, security settings, audit trails, backup/restore concepts.
- **Role-based access control (RBAC)** — Role creation, permission assignment, role inheritance, delegation rules.
- **Table and field management** — Creating tables, field types (text, numeric, dropdown, date, lookup), mandatory vs. optional fields, field validation rules.
- **Form design** — Form layouts, sections, related lists, field dependencies, conditional visibility, embedded reports.
- **Data relationships** — Reference fields, many-to-many (M2M) tables, lookups, field maps, data dependency.
- **Incident Management** — Incident creation, categorization, priority/impact matrix, assignment logic, state workflows, escalation.
- **Request Fulfillment** — Service Catalog item configuration, variables and options, request workflows, approval chains.
- **Change Management** — Change request workflows, CAB (Change Advisory Board) integration, conflict detection, rollback procedures (at admin level).
- **IT Asset Management (ITAM)** — Asset records, CI relationships, depreciation, lifecycle states, reconciliation.
- **Service Catalog** — Catalog categories, item properties, order guides, catalog search, consumer portal presentation.
- **Reporting** — List reports, pivot tables, bar/pie/line charts, filters, dashboard widgets, report scheduling.
- **Workflow automation** — Workflow editor, conditions, actions, timers, subflows, script activities (basic JavaScript).
- **SLA and workflow states** — SLA definition, conditions, targets (resolution time, response time), breach handling.
- **Self-Service Portal** — Portal configuration, catalog presentation, knowledge management integration, user personalization.
- **Business rules and script includes** — Simple table-level rules (UI policies, client scripts) for validation and automation.

## About

The CIS-EM certification validates a specialist's ability to configure, deploy, and maintain ServiceNow's Event Management application within IT Operations Management (ITOM) deployments. Candidates demonstrate proficiency in event architecture, event processing rules, alert management, external event source integration (SCOM, SolarWinds, Nagios), MID Server configuration, and correlation logic. As of 2024, CIS-EM became a "mainline" (high-stakes) exam requiring proof of eligibility before registration. This cert is intended for implementation consultants, ITOM engineers, and IT operations professionals managing event-driven automation at the enterprise scale.

## Topics covered

The CIS-EM exam is weighted across five major domains:

- **ITOM Solution & Event Management Fundamentals** (20%) — ITOM positioning, customer pain points, Event Management application structure, Operator Workspace, Alert Intelligence, dependency maps, CSDM integration
- **Architecture & Discovery** (25%) — Discovery Service integration, MID Server architecture and deployment, Event Management database schema, CMDB binding, monitoring architecture, MID Server validation and diagnostics
- **Event Processing & Configuration** (25%) — Event inbound actions, event rules, filters, thresholds, message key binding, CI binding, event scripting (JavaScript, PowerShell, regex), notification rules, event deduplication
- **Alert Management & Correlation** (20%) — Alert records and fields, alert management rules, priority scoring algorithms, incident auto-creation, alert correlation, aggregation, Service Operations Workspace, scheduled jobs for alert lifecycle
- **Event Sources & Monitoring** (10%) — Event source identification and connectivity, push vs. pull collection methods, third-party monitoring tool connectors (SCOM, SolarWinds, Nagios, generic), agent-based vs. agentless collection, troubleshooting event flow

*Source:* [Official CIS-EM Exam Blueprint ↗](https://learning.servicenow.com/lxp/en/credentials/certified-implementation-specialist-event-management-mainline?id=kb_article_view&sysparm_article=KB0011604)

- Designing event collection and routing strategies for hybrid cloud + on-premises infrastructure
- Configuring MID Servers, agent installation, and monitoring connectivity diagnostics
- Building correlation rules to reduce alert fatigue and surface root-cause events
- Integrating third-party monitoring tools (Nagios, SolarWinds, SCOM, Prometheus, Datadog) via Event Collector connectors

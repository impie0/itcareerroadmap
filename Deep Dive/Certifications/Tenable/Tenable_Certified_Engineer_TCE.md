## About

The Tenable Certified Engineer (TCE) is an associate-level certification validating foundational expertise in Tenable vulnerability management platforms. The exam assesses ability to deploy, configure, and operate Tenable.io and Tenable Security Center in enterprise environments, including asset discovery, scan configuration, vulnerability assessment, and basic troubleshooting. TCE is the entry point into Tenable's formal certification hierarchy; holders can advance to the Professional-tier Tenable Certified Security Engineer (TCSE) and related specialist certifications. The certification reflects practical platform knowledge without requiring prior Tenable experience.

## Domain context — Security/Vulnerability Management

Vulnerability management encompasses asset discovery, vulnerability scanning, risk assessment, remediation tracking, and integration with security infrastructure. Tenable is a leading vendor in this domain alongside Qualys, Rapid7, and Nessus-powered platforms. The TCE validates foundational implementation skills in Tenable-specific workflows.

## Topics covered

The TCE exam assesses the following competency areas:

- **Tenable platform architecture and deployment** — Tenable.io cloud-based deployment, Tenable Security Center (on-premises) architecture, licensing models, user provisioning
- **Asset discovery and management** — Asset identification workflows, agent deployment, scanner configuration, asset groups and tags, credential management basics
- **Vulnerability scanning fundamentals** — Nessus scan engine overview, basic scan template selection, scan scheduling, network vs. credentialed scanning, plugin types and updates
- **Scan result interpretation** — Vulnerability severity (CVSS scoring), finding plugin metadata, false positive identification, export and reporting basics
- **Access control and user management** — Role-based access control (RBAC) concepts, user and group creation, permission assignment, API key basics
- **Compliance and vulnerability reporting** — Basic compliance framework mapping (PCI-DSS, CIS), vulnerability export formats, dashboard creation, remediation tracking
- **Tenable.io vs. Tenable.sc operational differences** — When to use cloud vs. on-premises, platform strengths, data flow, backup and disaster recovery concepts
- **Troubleshooting basics** — Common scan failures (network connectivity, credentials, timeout), agent troubleshooting, log interpretation, support resources

## About

The ISTQB Certified Tester – Mobile Application Testing (CT-MAT) is a specialist-level certification for quality assurance professionals working with mobile platforms. Launched in 2021, CT-MAT extends the CTFL Foundation with mobile-specific expertise: iOS and Android testing, device diversity handling, connectivity testing, sensor interactions, usability and accessibility, security vulnerabilities, and performance optimization on resource-constrained devices. Target audience includes test engineers, QA specialists, automation engineers, business analysts, and test leads moving into mobile development projects. Certification validity spans 3 years; renewal occurs via retake after expiration.

The CT-MAT addresses unique challenges of mobile testing that differ substantially from web and desktop environments. Mobile apps must handle interrupted network transitions, multiple device orientations, varying screen sizes and resolutions, battery and memory constraints, and sensor-based inputs like GPS and accelerometers. The syllabus balances theoretical understanding with practical application of tools and techniques used in professional mobile QA teams across startups and enterprises. Completion requires passing 40 multiple-choice questions with 65% correct within 60 minutes, demonstrating mastery of mobile-specific testing methodologies, test automation approaches, and quality attribute evaluation.

## Domain context — Quality Assurance

Mobile application testing is a specialized domain within QA focused on validating software behavior across heterogeneous devices, operating systems, network states, and user interactions unique to smartphones and tablets. Unlike web or desktop testing, mobile QA must account for interrupted connectivity (WiFi to cellular transitions), device sensors (GPS, accelerometer, proximity sensor), OS-specific UI patterns (Material Design for Android, Human Interface Guidelines for iOS), screen sizes from 4.5 to 7 inches, varying pixel densities (2x, 3x displays), and performance constraints tied to battery life (3000–5000 mAh typical) and available RAM (2–12 GB).

Mobile testing encompasses manual and automated approaches, compatibility matrix challenges spanning hundreds of device and OS version combinations, performance profiling under real network conditions, and security testing aligned with OWASP Mobile Top 10 vulnerabilities (insecure storage, insufficient cryptography, insecure communication). The domain integrates with DevOps/CI-CD pipelines, cloud-based device farms (BrowserStack, Sauce Labs), and continuous testing frameworks (Bitrise, Fastlane). CT-MAT certification validates mastery of these unique requirements and demonstrates ability to design comprehensive mobile test strategies across iOS (Swift/Objective-C apps) and Android (Java/Kotlin apps) platforms. Professionals holding this certification commonly lead mobile QA initiatives, architect test automation solutions, mentor junior testers, and establish mobile testing centers of excellence.

## Topics covered

Mobile application testing encompasses the full testing lifecycle adapted for mobile constraints:

- **Mobile Ecosystem & Business Context** (15%) — Business drivers (app store economics, user acquisition cost, retention metrics), mobile analytics platforms (Firebase, AppsFlyer), app architectures (native iOS/Android, hybrid Cordova/React Native, cross-platform Xamarin/Flutter, progressive web apps), device platforms and OS versions, fragmentation challenges across manufacturers and carrier customizations, platform-specific guidelines
- **Mobile Testing Types** (20%) — Compatibility testing across device/OS combinations, performance and power consumption (battery drain analysis, CPU/memory profiling, thermal management), usability and accessibility testing (touch interactions, gestures, screen reader compatibility, color contrast), security and data protection, connectivity testing (WiFi, cellular, offline modes, network transitions), sensor and hardware interaction testing (camera, GPS, accelerometer, Bluetooth, NFC)
- **Mobile Test Automation** (20%) — Automation frameworks and tools (Appium for cross-platform, Espresso for Android, XCTest for iOS, UIKit testing), test design for mobile-specific scenarios, CI/CD pipeline integration, emulator vs. real device trade-offs, cloud device lab utilization, test reporting and metrics, flaky test management
- **Mobile Testing Tools & Environments** (15%) — Emulators and simulators (Android Emulator, iOS Simulator), physical device labs and cloud services (BrowserStack, Sauce Labs, AWS Device Farm, LambdaTest), debugging and profiling tools (Android Studio, Xcode Instruments, Charles Proxy), logcat and system logs, test case management systems, crash analytics
- **Mobile Test Strategy & Planning** (15%) — Risk-based test planning adapted for mobile constraints, test case design for device diversity matrix, coverage strategies for OS versions and network conditions, release readiness assessment, regression testing optimization
- **Mobile-Specific Quality Attributes** (15%) — Battery drain and power consumption measurement, data consumption over metered networks, interruption handling (phone calls, notifications, app backgrounding, lifecycle events), localization and internationalization testing (multiple languages, regional settings), responsive and adaptive design validation, state preservation across app lifecycle events

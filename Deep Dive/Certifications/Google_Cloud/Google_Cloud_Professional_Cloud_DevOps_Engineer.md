## About

The Google Cloud Professional Cloud DevOps Engineer certification validates expertise in applying Site Reliability Engineering (SRE) practices to Google Cloud services. This certification demonstrates proficiency with CI/CD pipelines, infrastructure automation, cloud operations, observability, and incident management. Ideal for DevOps engineers, SRE practitioners, and platform engineers who design, implement, and manage production services on GCP using automation and reliability best practices.

## Domain context — Cloud

Hyperscale public cloud (GCP). Part of the Google Cloud certification track for DevOps and infrastructure specialists.

[Read full deep dive — Google Cloud →](../../Google_Cloud_Roadmap.md)

## Topics covered

**Exam Domain Distribution:**

- **Bootstrapping and Maintaining a Google Cloud Organization** (~20%) – Design resource hierarchy; organize resources (projects, folders); configure Shared VPC and networking; implement IAM roles and policies; set up multi-project monitoring and logging
- **Building and Implementing CI/CD Pipelines** (~30%) – Design and implement CI/CD pipelines using Cloud Build and Cloud Deploy; manage pipeline configuration and secrets; implement security controls; automate testing and deployments
- **Applying Site Reliability Engineering (SRE) Practices** (~25%) – Apply SRE principles to service design; establish SLIs, SLOs, and error budgets; implement runbooks; practice healthy incident response; manage on-call rotations
- **Implementing Observability** (~13%) – Implement monitoring with Cloud Monitoring; configure logging and log analysis; set up Cloud Trace for tracing; implement Cloud Debugger; create dashboards and alerts
- **Optimizing Performance and Cost** (~12%) – Collect performance metrics; use Active Assist insights; implement FinOps practices; optimize resource utilization; configure cost controls and recommenders

**Key Technologies Covered:**
- Cloud Build (CI/CD orchestration)
- Cloud Deploy (deployment automation)
- Google Kubernetes Engine (GKE)
- Artifact Registry (container and package registry)
- Infrastructure as Code (Terraform, Deployment Manager)
- Cloud Monitoring, Cloud Logging
- Cloud Trace, Cloud Debugger
- Error Reporting, Profiler
- Pub/Sub, Cloud Tasks (event-driven automation)
- Shared VPC, VPC Service Controls
- IAM, Cloud KMS, Secret Manager
- Anthos (hybrid/multi-cloud)

## About

The Google Cloud Professional Cloud Database Engineer (PDE) certification validates expertise in designing, deploying, and maintaining scalable, highly available cloud database solutions on Google Cloud Platform. Launched in 2022, this Professional-level cert targets mid-to-senior database architects and cloud data engineers with hands-on GCP experience. PDE holders demonstrate deep knowledge of GCP's polyglot database portfolio—including AlloyDB, Cloud SQL, Cloud Spanner, Bigtable, Firestore, and Memorystore—and the ability to select and optimize the right database for specific workloads. The exam is actively maintained with quarterly blueprint reviews; no retirement is announced.

## Topics covered

The exam blueprint weights knowledge across seven major domains (official Google guide):

- **Design scalable and highly available cloud database solutions** (~30–35%)
  - Selecting appropriate GCP database services (AlloyDB, Cloud SQL, Cloud Spanner, Bigtable, Firestore, Memorystore)
  - Designing for high availability, disaster recovery, and failover
  - Multi-region and multi-zone database architectures
  - Capacity planning and instance sizing
  
- **Manage a solution spanning multiple database solutions** (~20–25%)
  - Polyglot persistence patterns and database selection frameworks
  - Use cases for each GCP database type
  - Data consistency models and eventual consistency tradeoffs
  - Cross-database transactions and data synchronization
  
- **Migrate data solutions** (~15–20%)
  - Database Migration Service (DMS) planning and execution
  - Schema migration and data transfer patterns
  - Heterogeneous database migrations (on-premises to GCP)
  - Zero-downtime migration strategies
  
- **Deploy cost-effective database solutions** (~10–15%)
  - Pricing models and cost optimization (instance types, reserved instances, commitment discounts)
  - Resource scaling and autoscaling for cost efficiency
  - Monitoring spend and optimizing reserved capacity
  
- **Build and maintain database solutions using Vertex AI and Cloud AI products** (~10–15%)
  - BigQuery ML for predictive analytics
  - Vertex AI integration with databases
  - LLM-powered data operations and automation
  - Machine learning pipelines with database backends
  
- **Observe, monitor, and tune database operations** (~15–20%)
  - Cloud Monitoring and Cloud Logging for databases
  - Query Insights and performance recommendations
  - Connection pooling with PgBouncer
  - Cloud SQL Auth Proxy and network optimization
  - Identifying and resolving performance bottlenecks
  
- **Secure databases** (~15–20%)
  - IAM roles and service account management
  - VPC Service Controls and private service access
  - Customer-Managed Encryption Keys (CMEK)
  - Network isolation and firewall rules
  - Audit logging and compliance monitoring

*Source:* [Official Google Cloud PDE Exam Guide](https://cloud.google.com/learn/certification/guides/cloud-database-engineer)

## Skills validated

*Cert-specific — what this exam actually tests, distinct from the shared "Common skills" above.*

- AlloyDB for PostgreSQL (managed, enterprise-grade, high-availability architecture)
- Cloud SQL (MySQL, PostgreSQL, SQL Server; managed relational databases)
- Cloud Spanner (globally distributed, horizontally scalable, strongly consistent)
- Bigtable (wide-column NoSQL for analytics and real-time applications)
- Firestore (document database, real-time sync, offline support)
- Memorystore (managed Redis and Memcached for caching)
- Database Migration Service (DMS) for schema and data transfer
- Polyglot persistence (selecting multiple database types within a solution)
- Cloud Monitoring and Cloud Logging for database observability
- Query Insights and performance recommendations engine
- Connection pooling strategies (PgBouncer, Cloud SQL Auth Proxy)
- BigQuery ML for in-database machine learning
- VPC Service Controls and private service access for database security
- Customer-Managed Encryption Keys (CMEK) for encryption
- IAM roles and service account management for database access

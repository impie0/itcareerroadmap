## About

The Google Cloud Professional Cloud Network Engineer (Professional Cloud Network Engineer) certification validates expertise in designing, implementing, and managing complex Google Cloud networking solutions. Targeting mid-to-senior network architects and cloud infrastructure engineers, this Professional-level cert requires hands-on GCP experience and a strong networking background. Holders demonstrate deep knowledge of GCP networking primitives—VPC design, hybrid connectivity, load balancing, DNS, security, and network operations—and the ability to design scalable, secure, and highly available networks. The exam is actively maintained with no announced retirement.

## Topics covered

The exam blueprint weights knowledge across six major domains (official Google guide):

- **Designing, planning, and prototyping a GCP network** (20–24%)
  - VPC design (shared VPC, VPC peering, multi-region)
  - IP addressing and subnet design
  - Hybrid connectivity (Cloud VPN, Cloud Interconnect, Cross-Cloud Interconnect)

- **Implementing virtual private cloud (VPC) instances** (16–24%)
  - Firewall rules and policies
  - Routes and routing protocols
  - NAT (Cloud NAT)
  - Private Google Access
  - DNS (Cloud DNS, private zones, DNS peering)
  - VPC Service Controls

- **Configuring network services** (24–28%)
  - Cloud Load Balancing (global HTTPS, regional external/internal, TCP/UDP)
  - Cloud CDN
  - Traffic Director (service mesh)
  - Network Endpoint Groups (NEGs)

- **Implementing hybrid interconnectivity** (16–20%)
  - Cloud VPN (HA VPN, classic VPN)
  - Cloud Interconnect (Dedicated, Partner)
  - BGP configuration
  - Network Connectivity Center

- **Implementing network security** (12–16%)
  - Cloud Armor (WAF, DDoS protection)
  - Certificate Manager
  - Firewall policies (hierarchical)
  - Private Service Connect
  - Secure web proxy

- **Managing, monitoring, and optimizing network operations** (8–12%)
  - Network Intelligence Center
  - VPC Flow Logs
  - Packet Mirroring
  - Network Topology
  - Connectivity Tests

*Source:* [Official Google Cloud Professional Cloud Network Engineer Exam Guide](https://cloud.google.com/learn/certification/guides/cloud-network-engineer)

## Skills validated

*Cert-specific — what this exam actually tests, distinct from the shared "Common skills" above.*

- Virtual Private Cloud (VPC) design and implementation (shared VPC, peering, multi-region)
- IP address management and subnet planning
- Cloud VPN (HA VPN and classic VPN) for hybrid connectivity
- Cloud Interconnect (Dedicated and Partner) for enterprise-grade on-premises connectivity
- BGP configuration and routing policy management
- Firewall rules, hierarchical firewall policies, and VPC Service Controls
- Cloud DNS (public and private zones, DNS peering, forwarding rules)
- Cloud NAT for secure outbound connectivity
- Private Google Access for internal service connectivity
- Cloud Load Balancing (global external, regional external, internal)
- Cloud CDN for content delivery and caching
- Traffic Director for service mesh and advanced traffic management
- Network Endpoint Groups (NEGs) for flexible service registration
- Cloud Armor for DDoS protection and web application firewall
- Certificate Manager for SSL/TLS certificate lifecycle
- Private Service Connect for private connectivity to services
- Secure Web Proxy for outbound traffic filtering
- Network Intelligence Center for topology visualization and analysis
- VPC Flow Logs for network traffic monitoring and troubleshooting
- Packet Mirroring for traffic analysis and security monitoring
- Connectivity Tests for network reachability validation
- Network observability, monitoring, and optimization

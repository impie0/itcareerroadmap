## About

The **Atlassian Certified Professional — Cloud Organization Admin (ACP-120)** validates expertise in administering Atlassian cloud products at the organization level. Released in updated form in 2023, it consolidates earlier role-specific certs (Jira Cloud Admin, Confluence Admin) into a single professional-level credential that covers user management, identity integration (SSO/SAML/SCIM), multi-product governance, security, permissions, data residency, cloud migrations, and marketplace ecosystem administration. It is the primary professional certification for those managing Atlassian cloud deployments across medium to large organizations.

## Topics covered

The ACP-120 exam blueprint covers:

- **User & License Management** (15–20%) — Adding/removing users, license assignment, user deprovisioning, user directory synchronization
- **Identity & Access Control** (15–20%) — SSO (SAML 2.0, OAuth), SCIM provisioning, Directory Sync, user authentication flows, permission schemes
- **Product Administration** (20–25%) — Jira Cloud, Confluence Cloud, Bitbucket Cloud, and Service Management Cloud basic admin (at org level, not site-specific workflows)
- **Security & Compliance** (15–20%) — IP allowlisting, audit logging, data residency, PII handling, export controls, permission audits
- **Migration & Cloud Readiness** (10–15%) — Server to Cloud migration path, Data Center to Cloud considerations, API-driven data transfer, downtime planning
- **Marketplace & App Governance** (5–10%) — Viewing installed apps, app permissions, marketplace app vetting, cloud app implications
- **Organization Settings & Governance** (10–15%) — Organization structure (domains, sites), shared resources, global policies, multi-instance administration
- **Troubleshooting & Support** (5%) — Common admin issues, logging, vendor support interaction

*Source:* Atlassian University exam blueprint (accessible post-enrollment)

## Skills validated

*Cert-specific — what the ACP-120 exam actually tests.*

- **Atlassian Cloud product suite** — Jira Cloud, Confluence Cloud, Bitbucket Cloud, Service Management Cloud
- **SAML 2.0 & OAuth 2.0** — Identity provider integration, token flows, SSO configuration
- **SCIM (System for Cross-Domain Identity Management)** — Automated user provisioning, attribute mapping, directory synchronization
- **Atlassian Directory (user management)** — User invitations, license lifecycle, inactive user handling, API-driven bulk operations
- **Atlassian Cloud security model** — IP allowlisting, audit logs, workspace isolation, data residency
- **Permission & space governance** — Project roles, space permissions, group management, permission inheritance
- **Cloud-to-Cloud migrations** — Data transfer, downtime planning, cloud readiness assessment
- **Marketplace app administration** — App security review, permission implications, installation workflows
- **Atlassian REST APIs** — User admin endpoints, site admin endpoints, scripting for bulk tasks
- **Compliance & audit** — SOC 2, GDPR, export controls, audit log review, retention policies

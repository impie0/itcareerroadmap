## About

The Cisco Certified DevNet Specialist – Data Center Automation and Programmability (300-635 DCAUTO) certifies professionals in automating Cisco data center solutions using programmability, orchestration, and automation tools. Launched in 2019, this exam is a concentration requirement for the CCNP Data Center and Cisco Certified DevNet Professional certifications. It targets network engineers, systems engineers, and IT professionals with hands-on experience in data center infrastructure automation, API-driven workflows, and infrastructure-as-code practices. The exam reflects real-world day-zero provisioning, on-box and off-box automation, and multi-vendor orchestration in modern data center environments.

## Domain context — Data Center Programmability

Network programmability and automation are foundational to modern data center operations — enabling policy-driven provisioning, reduced manual configuration, faster deployment cycles, and consistent infrastructure-as-code governance across Cisco ACI, NX-OS, UCS, and Intersight ecosystems.

## Topics covered

Per the official DCAUTO v1.1 blueprint:

- **Network Programmability Foundation (10%)** — Python fundamentals, REST APIs, JSON/YAML/XML, Jinja2 templating, code repositories (Git), basic networking concepts
- **Controller-Based Data Center Networking (30%)** — Cisco ACI REST API, ACI object model, ACI automation, fabric management, Intent-based networking, policy deployment
- **Data Center Device-Centric Networking (30%)** — NX-OS programmability, NX-API (CLI & REST), Bash scripting, Guest Shell, NETCONF/RESTCONF, YANG models, event-driven automation
- **Data Center Compute (30%)** — Cisco UCS Manager Python SDK, UCS automation workflows, service profiles, policy-driven deployment, Intersight API, HyperFlex automation, infrastructure provisioning

*Source:* [Official Cisco DCAUTO Exam Topics ↗](https://learningnetwork.cisco.com/s/dcauto-exam-topics)

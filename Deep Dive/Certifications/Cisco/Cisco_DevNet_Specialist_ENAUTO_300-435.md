# Cisco Certified DevNet Specialist — Enterprise Automation and Programmability

**`300-435`** · ● **Active** · **Professional** · _Cisco_

> Concentration exam for CCNP Enterprise and Cisco Certified DevNet Professional. Tests enterprise network automation, Python programming, APIs (NETCONF, RESTCONF), controller-based automation (Catalyst Center, SD-WAN, Meraki), and Cisco IOS XE automation. 90 minutes, $300, 50–65 scored questions. No prerequisites. Valid 3 years; renew via retake, higher exam, or 80 CE credits (hybrid: 1 prof exam + 40 CE). Pairs with DEVCOR 350-901 for full DevNet certification stack.

## About

The 300-435 ENAUTO (Automating and Programming Cisco Enterprise Solutions) is a 90-minute professional-level concentration exam introduced in 2021 that validates hands-on expertise in enterprise network automation, Python scripting, API design and consumption (NETCONF, RESTCONF), and controller-based automation platforms including Cisco Catalyst Center (formerly DNA Center), SD-WAN vManage, and Cisco Meraki. It serves as a concentration exam for both the CCNP Enterprise and Cisco Certified DevNet Professional credential tracks. The exam shifted to v2.0 in 2025, emphasizing modern automation frameworks, telemetry, and AI-enabled operations. No prerequisites exist, though 3–5 years of hands-on DevOps and automation experience is typical for successful candidates.

## Topics covered

Based on the official ENAUTO v2.0 exam blueprint:

- **Network Programmability Foundation (10%)** — Protocol overview (NETCONF, RESTCONF, YANG), data models, API design patterns, Python basics for network automation
- **Automate APIs and Protocols (10%)** — RESTCONF configuration, NETCONF operations, YANG data model concepts, XML/JSON parsing in Python
- **Network Device Programmability (20%)** — Cisco IOS XE automation via APIs, on-box Python (guest shell), streaming telemetry, gRPC, device configuration via API
- **Cisco Catalyst Center Automation (20%)** — Intent-based networking, template management, assurance and monitoring, API-driven deployments, multi-site fabric automation
- **Cisco SD-WAN Automation (20%)** — SD-WAN vManage API, policy templates, device onboarding, traffic steering automation, analytics consumption
- **Cisco Meraki Automation (20%)** — Meraki Dashboard API, organization and network management, device provisioning, client policy automation, reporting

*Source:* [Official Cisco exam blueprint ↗](https://learningcontent.cisco.com/documents/marketing/exam-topics/300-435-ENAUTO-v2.0-7-9-2025.pdf)

## Skills validated

*Specific technologies, protocols, and tools tested by the ENAUTO exam.*

- **Python 3** — scripting, data types, functions, loops, libraries (requests, json, yaml, paramiko)
- **NETCONF and RESTCONF** — configuration retrieval/editing, YANG data models, namespace operations
- **YANG** — data model structure, leafs, containers, rpc definitions, constraints
- **REST APIs** — HTTP methods, JSON/XML parsing, authentication (basic, OAuth, API keys), error handling
- **Cisco IOS XE** — on-box guest shell Python, EEM (Event Management), API GETs/POSTs
- **Cisco Catalyst Center (DNA Center)** — Intent-based networking, site management, template provisioning, device assurance, API-driven deployments
- **Cisco SD-WAN (Viptela)** — vManage API, policy and template application, device onboarding, telemetry consumption
- **Cisco Meraki** — cloud-based Dashboard API, organization/network/device management, client policies, network monitoring
- **Ansible** — playbooks, roles, variables, handlers, idempotency, network modules
- **Terraform** — HCL, state management, providers, module organization
- **Git** — version control, branching, pull requests, CI/CD integration
- **Streaming Telemetry** — gRPC, MDT (Model-Driven Telemetry), subscriptions, Telegraf/InfluxDB ingest
- **Cisco ISE (Identity Services Engine)** — EAP, 802.1X, API-driven policy, radius/tacacs+ automation
- **JSON/XML/YAML** — data serialization, templating, Jinja2 template engine
- **Docker / Containerization basics** — concepts, networking, CI/CD pipelines

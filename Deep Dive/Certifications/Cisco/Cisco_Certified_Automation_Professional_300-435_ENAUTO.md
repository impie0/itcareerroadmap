## About

The Automating and Programming Cisco Enterprise Solutions (300-435 ENAUTO) exam validates professional-level expertise in designing, deploying, and maintaining network automation solutions across Cisco enterprise platforms. Launched in 2020 following Cisco's rebranding of DevNet to "Automation," ENAUTO is a concentration exam within the Cisco Certified Automation Professional (CCAP) path and supersedes earlier DevNet Pro specializations. It assesses hands-on competency with Cisco DNA Center, SD-WAN orchestration, Meraki management APIs, and enterprise-grade programmability frameworks. ENAUTO targets network automation engineers, NetDevOps practitioners, and infrastructure automation architects working at scale in multi-domain enterprise environments.

## Topics covered

The exam validates knowledge across six primary domains:

- **Network Programmability Fundamentals (15%)** — API concepts (REST, YANG, NETCONF); authentication and authorization; security in automation; SDK and library usage; data models and serialization (JSON, XML, YAML).
- **Automation APIs and Protocols (20%)** — RESTful API design patterns; HTTP methods and status codes; request/response handling; rate limiting; error handling; webhook integrations; API documentation and testing tools.
- **Network Device Programmability (20%)** — Device-level APIs (Cisco IOS, IOS-XE, IOS-XR); SNMP automation; command execution; configuration templating; device backup and restore; multi-vendor API patterns.
- **Cisco DNA Center (20%)** — DNA Center architecture and intent-based networking; assurance workflows; device onboarding; policy configuration; telemetry and analytics; DNA Center APIs and SDK.
- **Cisco SD-WAN (15%)** — SD-WAN architecture and deployment models; vManage API integration; policy automation; edge device programming; application-aware routing; performance monitoring via APIs.
- **Cisco Meraki (10%)** — Meraki Dashboard API; device and organization management; network monitoring and alerting via APIs; wireless configuration automation; integration patterns with third-party systems.

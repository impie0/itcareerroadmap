## About

The Cisco Certified Network Professional Automation (CCNP Automation) is Cisco's professional-level certification for network engineers, software developers, and DevOps practitioners who design, deploy, and manage automation systems using Cisco platforms, APIs, and cloud-native architectures. Originally launched in December 2019 as CCNP DevNet DEVCOR, it was rebranded to CCNP Automation on February 3, 2026, aligning Cisco's cert portfolio to emphasize AI-driven automation and network programmability. The exam tests deep knowledge across software development patterns, REST/NETCONF/RESTCONF APIs, Cisco platforms (Webex, Catalyst, Meraki, Nexus, DNA Center), containerization (Docker/Kubernetes), CI/CD pipelines, infrastructure-as-code, and network security automation.

## Domain context — Networking / Automation

Specialized subset of networking focused on programmatic control, API-first architecture, and operational automation of network infrastructure. ## Topics covered

The official DEVCOR 350-901 exam blueprint (current for 2026, pre-AUTOCOR transition) covers five main domains:

- **Software Development and Design** (15–20%) — design patterns, architectural patterns (MVC, microservices), problem-solving, data structures, databases, object-oriented principles, code organization, version control
- **Using APIs** (20–25%) — REST APIs, error handling, pagination, rate limiting, flow control, OAuth 2.0, OpenAPI/Swagger, request/response formats (JSON, XML), API authentication, API versioning
- **Cisco Platforms** (15–20%) — Webex Teams/API, Cisco Firepower, Cisco Meraki, Cisco Intersight, Cisco UCS, Cisco DNA Center, AppDynamics, custom dashboards, event-driven architectures
- **Application Deployment and Security** (20–25%) — CI/CD pipelines, Docker, Kubernetes, containers, image registries, orchestration, DevSecOps, data encryption, secret management, OWASP threat models, secure coding
- **Infrastructure and Automation** (15–20%) — Infrastructure as Code (Terraform, Ansible, Puppet), model-driven telemetry (MDT), NETCONF, RESTCONF, network configuration automation, monitoring/logging, observability

*Source:* [Cisco Learning Network — DEVCOR Exam Topics ↗](https://learningnetwork.cisco.com/s/autocor-exam-topics)

## Rebranding note: DevNet → Automation (Feb 3, 2026)

On February 3, 2026, Cisco rebranded its entire DevNet certification track to **Cisco Automation Certifications**, aligning professional and expert tracks with network automation and AI-driven network management. 

- **Before:** CCNP DevNet Professional, exam DEVCOR 350-901 (5 domains)
- **After:** CCNP Automation, exam AUTOCOR 350-901 (4 domains, restructured)

All active CCNP DevNet certifications automatically migrated to the new CCNP Automation track — **no retesting required**. Candidates holding the CCNP DevNet cert are now officially certified as CCNP Automation. The exam code 350-901 remains the same; objectives were refined to emphasize network automation, infrastructure-as-code, and AI/ML integration.

**Important for test-takers (May 2026):** The legacy DEVCOR exam officially retired on Feb 3, 2026. New candidates must now register for **AUTOCOR 350-901**. Existing DEVCOR study materials (books, courses, practice tests) remain valid for 90% of coverage, but review updated exam topics on Cisco Learning Network before exam day.

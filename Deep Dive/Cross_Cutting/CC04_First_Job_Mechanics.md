# CC04: First-Job Mechanics Per IT Certification
## Practical "How Do I Actually Get Hired" Guide

**Status:** Research-backed, April 2026  
**Scope:** Entry-level + early-career certifications (A+, CCNA, Security+, AWS, Azure, Linux, etc.)  
**Cross-links:** [CC03 Career Changers](../CC03_Career_Changer_Backgrounds.md), [CC07 Visa & Geography](../CC07_Visa_and_Geographic_Factors.md)

## Overview

This guide maps **what hiring managers actually ask for**, **portfolio proof you need**, **resume strategy**, **interview formats**, **job boards to use**, and **first-90-days expectations** for 30+ entry-level IT certifications. Every claim is sourced; unsourced claims are omitted.

## FOUNDATION: Help Desk / Desktop Support

### CompTIA A+ (220-1201 / 1202)

#### What Jobs Are Actually Posted

- Help Desk Tier 1 (most common)
- Desktop Support Technician
- IT Support Specialist
- Computer Support Technician
- Technical Support Associate

**Real posting volume:** Over 5,525 active job postings require A+ certification on Glassdoor as of 2026. Employers across tech, healthcare, government, and finance actively seek A+ certified professionals for frontline IT support.

#### Portfolio / Proof of Skill

- **Home lab:** Virtual machine setup with Windows and Linux (VMware, VirtualBox, Hyper-V).
- **Hands-on projects:**
  - Troubleshoot and repair a non-booting Windows OS (document with screenshots).
  - Network two VMs (configure IP, test connectivity with ping/ipconfig).
  - Install and configure network printers; document driver issues and resolution.
  - Build a basic Active Directory domain with users/groups.
- **Write-ups:** 3–5 short posts on Medium/Dev.to explaining common A+ topics (BSoD troubleshooting, disk partitioning, BIOS/UEFI basics).

#### Resume Tips (Entry-Level A+ Formula)

**Keywords to include:** CompTIA A+, Tier 1 Support, Active Directory, Windows 10/11, macOS, networking, hardware troubleshooting, ticketing systems (ServiceNow, Zendesk, Jira Service Desk), Remote Desktop, TeamViewer.

**Structure:**
- **Certifications** section: Place A+ prominently (top of certifications if you have it).
- **Technical Skills:** List by OS (Windows 10/11, macOS, Linux basics), protocols (TCP/IP, DNS, DHCP), hardware (RAM, storage, peripherals).
- **Experience:** Quantify — "Resolved 30 support tickets/day with 95% customer satisfaction"; "Imaged 40+ systems in 2 months using Acronis."
- **Action verbs:** Troubleshot, Configured, Imaged, Deployed, Documented.

**ATS optimization:** 97% of companies use ATS to filter IT support resumes. Missing keywords like "ServiceNow," "Active Directory," or "Tier 1 Support" can disqualify you instantly. Match the job description term-for-term.

#### Interview Format Expected

**Structure:** 30–45 minutes. Icebreaker → Technical probes → Behavioral scenarios.

**Technical questions (expect):**
- Describe the OSI model and which layer handles routing.
- User can't connect to WiFi; walk me through troubleshooting.
- What's the difference between BIOS and UEFI?
- Explain the purpose of Active Directory.
- User reports slow performance; what tools would you check?

**Behavioral questions (STAR format):**
- Tell me about a difficult end-user you supported. How did you handle it?
- Describe a time you had to troubleshoot something you didn't know; how did you approach it?
- What's your process for documenting issues in a ticketing system?

**Tip:** Technical knowledge gets you the interview; communication gets you the job.

#### Where to Apply

- **Generic boards:** Indeed, LinkedIn, Glassdoor.
- **Cert-specific:** CompTIA Job Board (partner postings), Tech job boards (Dice, Stack Overflow Jobs).
- **Junior-friendly:** Wellfound (formerly AngelList Talent), Built In, Otta, College Recruiter.
- **Underrated:** Local LinkedIn recruiter DMs (search "IT recruiter" + your city), local tech meetups, staffing agencies (Tek Systems, Volt).

#### First 90 Days in Role

**Days 1–30 (Orientation):**
- Company culture immersion, compliance training, IT tools orientation (ticketing, ITSM, VPN).
- Shadow 2–3 experienced T1 analysts; listen to calls.
- Buddy system: assigned mentor for 4 weeks.
- Initial training: OS basics, remote support tools, password reset procedures.

**Days 31–60 (Hands-on Contribution):**
- Handle 10–15 tickets/day independently (ramp up from 5–8).
- Gain ticket-owning accountability; learn escalation pathways.
- First time handling your own printer setup, Active Directory user creation, basic hardware swaps.
- Manager check-ins on progress.

**Days 61–90 (Independence):**
- Reach 25–30 tickets/day baseline.
- Ownership of recurring tasks (monthly password audits, new employee onboarding).
- First small incident that wasn't routine; handled it without escalation.
- Transition planning: what's next? Specialization or next role?

#### Salary (Cited)

- **Entry-level (0–1 year):** $40,000–$55,000 (US average).
- **Help desk technician (A+ typical):** $41K–$65K per 2026 market data.
- **IT support specialist median:** $71,000 (as of November 2025).
- **2026 average:** $63,909/year.
- **Salary premium for A+ cert:** 5–15% above non-certified peers in same role.

**Sources:** ZipRecruiter (April 2026), Glassdoor (5,525 postings), Coursera salary insights.

### Cisco CCST (100-140 IT Support, 100-150 Networking)

#### What Jobs Are Actually Posted

- Help Desk Tier 1 / Tier 2
- Network Operations Center (NOC) Technician
- Network Support Specialist
- Customer Support Engineer

**CCST positioning:** Entry-level credential; stepping stone to CCNA.

#### Portfolio / Proof of Skill

- **Packet Tracer labs:** 10–15 Cisco Packet Tracer labs showing basic router/switch config, VLAN setup, static routing.
- **Real gear (optional):** Used Cisco equipment on eBay/Craigslist (2950/2960 switches, 1841 routers) for home lab.
- **Lab write-ups:** Document each lab on Medium/Dev.to with screenshots of running-config and verification commands.

#### Resume Tips

- **Keywords:** CCST, network fundamentals, routing, switching, TCP/IP, VLAN, Cisco IOS, Packet Tracer, network troubleshooting.
- **Structure:** Place CCST prominently; link to Packet Tracer labs in GitHub or personal site.

#### Interview Format Expected

- **Technical:** Explain routing vs. switching. Draw a simple network topology. What's a VLAN and why use it? Walk me through a ping.
- **Behavioral:** Time you debugged network connectivity. How you learn new protocols.
- **Hands-on:** Some employers give a Packet Tracer scenario (configure a small network; 30 minutes).

#### Where to Apply

- **Cisco-specific:** Cisco Learning Network Job Board, Cisco Career Opportunitiespage.
- **Generic:** Indeed, Dice (strong for networking roles), LinkedIn.
- **Junior-friendly:** Wellfound, Built In.

#### First 90 Days in Role

**Days 1–30:**
- NOC onboarding: ticket system, monitoring tools (Nagios, Zabbix), escalation procedures.
- Shadow NOC techs during incidents (even low-severity ones).
- Learn company's network topology (usually 20–50 pages of documentation).

**Days 31–60:**
- Handle first-level alerts: interface down, high CPU, bandwidth saturation.
- Perform basic diagnostics (ping, traceroute, show commands).
- Escalate to L2/L3 for deeper troubleshooting.

**Days 61–90:**
- Own common maintenance tasks: OS upgrades on non-critical devices, documentation updates.
- Confidence to escalate correctly (know when it's your domain vs. when to hand off).

#### Salary (Cited)

- **CCST entry-level network technician:** $55,420/year average; range $31K–$81K.
- **NOC analyst with CCST:** $58,000–$75,000 (estimated from networking technician data).
- **Path to CCNA:** Network technician → Junior Network Engineer ($93K–$160K with CCNA + experience).

**Source:** ZipRecruiter (April 2026), network technician salary aggregates.

## NETWORKING: Network Engineers & Technicians

### CompTIA Network+ (N10-009)

#### What Jobs Are Actually Posted

- Network Technician
- Network Operations Center (NOC) Analyst
- Junior Network Administrator
- Network Support Specialist

#### Portfolio / Proof of Skill

- **Packet Tracer + GNS3 labs:** 15–20 labs demonstrating subnetting, VLAN trunking, OSPF, ACLs, NAT.
- **Real network:** If access to Cisco/Juniper hardware, configure and document a small prod-like topology.
- **Write-ups:** 5–8 technical posts explaining OSI model, subnetting, OSPF concepts, with packet captures from Wireshark.

#### Resume Tips

- **Keywords:** Network+, routing, switching, VPN, firewall basics, packet analysis, Wireshark, VLAN, subnetting, TCP/IP, network troubleshooting.
- **Quantify:** "Deployed 12 VLANs across 3 switches," "Analyzed 100+ packet captures to identify misconfiguration."

#### Interview Format Expected

- **Technical:** Subnetting questions (given a network, calculate subnets). Explain routing protocols (RIP, OSPF, BGP at conceptual level). Draw a DMZ. Walk me through a VPN handshake.
- **Hands-on:** Some employers provide a Packet Tracer scenario (design a small network to spec; 45 minutes).
- **Behavioral:** Time you troubleshot a network outage. How you stay current with networking standards.

#### Where to Apply

- Dice (networking-strong), LinkedIn, Indeed.
- CompTIA Job Board (partner listings).
- Wellfound, Built In.

#### First 90 Days in Role

**Days 1–30:**
- Network documentation tour; learn topology, IP addressing scheme, naming conventions.
- Ticket system and escalation (same as CCST, but more complex issues).
- Complete vendor training (Cisco, Juniper, Arista if company uses it).

**Days 31–60:**
- Own basic network change tasks: VLAN reconfig, ACL updates on non-critical devices.
- Assist with network maintenance windows (upgrades, migrations).
- First time you touch production equipment under supervision.

**Days 61–90:**
- Independent ticket ownership on low-risk changes.
- Contribute to network runbook updates (procedure documentation).
- Interest signal for specialization (security, cloud network, wireless)?

#### Salary (Cited)

- **Network technician:** $55K–$85K.
- **NOC analyst:** $60K–$90K.
- **Junior network admin (with Network+):** $75K–$110K.
- **Salary premium for Network+ vs. non-cert:** 10–15%.

**Source:** ZipRecruiter, Glassdoor (aggregate networking role data).

### Cisco CCNA (200-301)

#### What Jobs Are Actually Posted

- Junior Network Engineer
- Network Engineer (early career)
- Network Administrator
- Network Support Engineer

**Real posting volume:** Hiring demand remains strong; CCNA is employer-recognized across all industries.

#### Portfolio / Proof of Skill

- **Advanced Packet Tracer labs:** 20–30 labs covering OSPF, EIGRP, ACLs, NAT, VPNs, QoS, VLANs, spanning-tree.
- **Real hardware (strongly preferred):** CCNA bootcamps often include hands-on lab time (CCNA boot camps: Boson Nessus labs or INE ProctorU lab).
- **Wireshark portfolio:** 5–10 packet captures annotated with analysis (TCP three-way handshake, ARP, ICMP, DNS, DHCP).
- **Blog:** 8–10 detailed posts explaining CCNA topics with diagrams (OSPF hello packets, EIGRP feasible successors, etc.).

#### Resume Tips

- **Keywords:** CCNA, routing protocols (OSPF, EIGRP, BGP), switching, VLAN, ACL, NAT, VPN, QoS, network design, packet analysis, Wireshark, Packet Tracer, network troubleshooting.
- **Quantify:** "Designed and tested OSPF migration affecting 5000+ users," "Performed network capacity planning for Q3 upgrade."
- **Structure:** CCNA prominently in certifications; 3–5 network projects in "Technical Experience" section.

#### Interview Format Expected

- **Technical (depth increases vs. Network+):** OSPF design questions. EIGRP metric calculation. ACL logic. VPN tunnel negotiation. BGP path selection (if senior role). Packet analysis scenario (here's a Wireshark dump; what's happening?).
- **Hands-on:** Many CCNA interviews include a 60–90-minute Cisco lab scenario (configure a small multi-site network to spec, troubleshoot pre-broken configs).
- **Behavioral:** Major network incident you handled or witnessed. How you approach learning new protocols.

#### Where to Apply

- **Cisco-specific:** Cisco Learning Network, Cisco Careers, Cisco partner job boards.
- **Strong:** Dice (networking-focused), LinkedIn (search "CCNA" + location), Indeed.
- **Competitive:** Built In, Wellfound (some startup networking roles).

#### First 90 Days in Role

**Days 1–30:**
- Network architecture deep-dive (company's full topology, design decisions).
- Vendor tools: Cisco Prime (if used), monitoring platforms (Cisco DNA Center, Catalyst suite).
- Ticket intake and triage process.
- Complete any internal routing protocol certification (Cisco-specific config standards).

**Days 31–60:**
- Own first routing changes (add OSPF area, EIGRP network statement) on non-critical sites.
- Assist with planned network maintenance (bug fixes, OS upgrades).
- Troubleshoot tickets escalated from NOC (root cause analysis, not just quick fix).

**Days 61–90:**
- Design small network improvements (VLAN segmentation, QoS policy for new app).
- Lead a minor migration or cutover (under supervision).
- Feedback: path to next cert (CCNP Security, CCNP Enterprise)?

#### Salary (Cited)

- **Entry-level CCNA network engineer:** $121,253/year (as of April 2026); range $93K–$160K.
- **Junior network engineer (broader role):** $112,042/year.
- **Salary premium:** CCNA + 3 years experience = $110K–$140K non-coastal metro.
- **Top industries:** Information Technology (median $108,809), Aerospace & Defense ($100,697).

**Source:** ZipRecruiter (April 2026), Glassdoor.

## SYSTEMS: Linux & Windows Administration

### Linux Foundation LFCS (Linux Foundation Certified Sysadmin)

#### What Jobs Are Actually Posted

- Junior Linux System Administrator
- Linux Support Technician
- Systems Administrator (Linux-focused)
- DevOps Engineer (often requires LFCS or equivalent)

#### Portfolio / Proof of Skill

- **Home lab:** CentOS/RHEL or Ubuntu VMs (3–5 servers, preferably on hypervisor).
- **Hands-on projects:**
  - Configure LAMP/LEMP stack (Apache/Nginx, MySQL, PHP); secure with firewall rules.
  - Set up SSH key-based authentication, user management, file permissions (umask, ACLs).
  - Partition a disk, create logical volumes, configure LVM snapshots.
  - Configure rsync-based backup solution; automate with cron.
  - Troubleshoot system performance (use top, iotop, vmstat to identify bottlenecks).
- **Write-ups:** 5–8 Medium posts explaining systemd, Linux networking, kernel tuning, security best practices.
- **GitHub repo:** Ansible playbooks or shell scripts for common tasks (user creation, package updates, log rotation).

#### Resume Tips

- **Keywords:** LFCS, Linux administration, CentOS, RHEL, Ubuntu, systemd, shell scripting (Bash), SSH, firewall (iptables, firewalld), user/group management, file permissions, LVM, package management (yum, apt), monitoring (top, ps, journalctl).
- **Quantify:** "Managed 40+ Linux servers," "Automated system patching for 50 servers using Ansible," "Reduced MTTR for Linux issues by 30% through runbook documentation."

#### Interview Format Expected

- **Technical:** Explain systemd. How do you troubleshoot high CPU usage on Linux? User permission scenario. SSH key setup. Firewall rules for a web server. Shell scripting basics.
- **Hands-on:** Many employers provide a Linux lab (log in to VM, troubleshoot a broken service, configure a new app; 60 minutes).
- **Behavioral:** Time you debugged a critical Linux system. How you stay current with Linux updates.

#### Where to Apply

- **Generic:** Indeed, LinkedIn, Glassdoor.
- **Linux-specific:** Linux Foundation Careers, Linux Foundation Job Board.
- **Strong for juniors:** Built In, Wellfound, Otta (if startup/DevOps-forward).

#### First 90 Days in Role

**Days 1–30:**
- Linux infrastructure tour (hypervisor, storage, backup, monitoring tools).
- Complete security training (especially if any financial/regulated data).
- Shadow 1–2 senior sysadmins on production tasks.
- Set up your own lab environment (same as home lab; learn company's OS baseline).

**Days 31–60:**
- Own first patching cycle (non-critical servers; follow change control process).
- Troubleshoot tickets escalated from L1: user access, disk space, service restarts.
- Assist with migrations or hardware refreshes.
- First time you impact production (under close review).

**Days 61–90:**
- Independent ownership of sysadmin tasks (user provisioning, log rotation, basic troubleshooting).
- Contribution to runbooks or system documentation.
- Interest in specialization (cloud Linux, container platforms, security hardening)?

#### Salary (Cited)

- **Junior Linux system administrator:** $104,900/year (ZipRecruiter, April 2026); range $87K–$123K.
- **Glassdoor alternative:** $136,018/year; range $108K–$172K (higher estimate reflects broader sysadmin role data).
- **Salary.com:** $93,366/year.
- **Certification premium:** LFCS holders earn 15–25% higher salaries compared to non-certified peers.

**Source:** ZipRecruiter (April 2026), Glassdoor, Salary.com, Linux Foundation salary data.

- Junior Linux Systems Administrator (Red Hat shops)
- Systems Engineer (Red Hat-focused)
- DevOps Engineer (Red Hat Ansible focus)

**Premium positioning:** RHCSA is the gold standard for Red Hat Enterprise Linux (RHEL) shops; often preferred over generic LFCS by large enterprises.

#### Resume Tips

- **Keywords:** RHCSA, RHEL, Red Hat Enterprise Linux, Ansible, systemd, SELinux, firewalld, yum, rpm, bash scripting, Linux administration, systems configuration.
- **Differentiator:** "Certified Red Hat System Administrator (EX200)" — emphasize the exam pass; employers recognize it as harder than generic LFCS.
- **Quantify:** "Managed 100+ RHEL servers," "Wrote 30+ Ansible playbooks for infrastructure automation."

#### Interview Format Expected

- **Technical:** RHEL-specific (SELinux contexts, firewalld zones, systemd targets). Troubleshoot RHEL issues. Ansible playbook review. RHEL security hardening.
- **Hands-on:** Many Red Hat employers provide RHEL lab scenario (remediate a misconfigured RHEL system; 90 minutes).
- **Behavioral:** Experience with Red Hat products. How you approach learning RHEL updates.

#### Where to Apply

- **Red Hat-specific:** Red Hat Careers, Red Hat Partner job boards (many ISVs, consulting firms hire for Red Hat specialization).
- **Generic:** Indeed, LinkedIn, Dice.
- **Enterprise-focused:** Heidrick & Struggles, CiscoJobs, IBM (owns Red Hat as of 2019; hires heavily for RHEL).

#### First 90 Days in Role

**Days 1–30:**
- Red Hat infrastructure architecture and best practices.
- RHEL security policies (SELinux context rules; compliance if regulated industry).
- Ansible Tower/AWX setup and usage (if company uses it).
- Shadow senior RHEL engineers.

**Days 31–60:**
- First RHEL change ticket: OS upgrade, yum patch, Ansible playbook execution.
- Troubleshoot SELinux denials (common gotcha for RHEL newcomers).
- Contribution to Ansible playbook library (add a new playbook or refactor existing).

**Days 61–90:**
- Owner of RHEL maintenance tasks (patches, OS upgrades, security updates).
- Lead documentation of infrastructure (runbooks, architecture diagrams).
- Interest in RHCE (Red Hat Certified Engineer) — next level?

#### Salary (Cited)

- **RHCSA junior sysadmin:** $95K–$120K (Red Hat premium vs. generic Linux; estimated 10–20% uplift).
- **Senior RHEL sysadmin (with RHCSA + 5 years exp):** $130K–$160K.

**Note:** RHCSA data is less widely published than LFCS/generic Linux. Use LFCS data as floor; add 10–15% for Red Hat premium.

## CLOUD: AWS, Azure, GCP

### AWS Cloud Practitioner (CLF-C02)

#### What Jobs Are Actually Posted

- Cloud Support Associate
- Junior Cloud Engineer
- Cloud Operations Associate
- Cloud Systems Administrator (entry)

**Real posting volume:** AWS Cloud Practitioner is the most widely held AWS cert; widely used as baseline filter for cloud-adjacent roles.

#### Portfolio / Proof of Skill

- **AWS Free Tier projects:** Deploy 3–5 simple projects:
  - Static website on S3 + CloudFront (with HTTPS).
  - EC2 instance + security groups + Elastic IP.
  - RDS database (MySQL/PostgreSQL) + basic backup config.
  - Lambda function triggered by S3 upload (simple ETL or image resize).
  - VPC with public/private subnets, NAT gateway.
- **GitHub README:** For each project, document the architecture, setup steps, cost estimate.
- **Write-ups:** 4–6 Medium posts explaining AWS core services (EC2, S3, RDS, Lambda, VPC).

#### Resume Tips

- **Keywords:** AWS Cloud Practitioner, AWS fundamentals, EC2, S3, RDS, Lambda, VPC, IAM, CloudWatch, cost management, cloud best practices.
- **Structure:** AWS projects in "Technical Projects" section with links to GitHub READMEs or deployed sites.
- **Quantify:** "Deployed 5 AWS projects costing <$2/month using Free Tier," "Wrote infrastructure-as-code for 3 application environments."

#### Interview Format Expected

- **Technical:** AWS service definitions (EC2 vs. Lightsail, RDS vs. DynamoDB). Cost management concepts. Basic security (IAM, security groups). Explain an architecture you built.
- **Hands-on:** Some employers provide AWS console scenario (deploy app to EC2, configure RDS, set up CloudWatch; 45 minutes).
- **Behavioral:** Why cloud? How you stay current with AWS updates.

#### Where to Apply

- **AWS-specific:** AWS Job Board, AWS Partner job boards (consulting firms, managed services providers).
- **Generic:** Indeed, LinkedIn, Built In, Wellfound.
- **Junior-friendly:** Otta, College Recruiter.

#### First 90 Days in Role

**Days 1–30:**
- Company's AWS account structure (multi-account strategy, billing, tagging standards).
- AWS support plan and escalation process.
- Internal AWS security policies (allowed services, IAM roles, encryption requirements).
- Shadow cloud support team on customer tickets (typical: password resets, console access, billing questions).

**Days 31–60:**
- Handle first cloud support tickets independently (AWS console navigation, basic troubleshooting).
- Escalate to L2 for infrastructure design, advanced troubleshooting.
- Complete hands-on labs (AWS Skill Builder, CloudAcademy) on frequently requested services.

**Days 61–90:**

## CROSS-CUTTING: Universal Strategies

### Resume Tips (All Roles)

1. **Tailor to job description:** Use the exact keywords from the posting. ATS filtering catches missing terms.
2. **Quantify impact:** Numbers beat adjectives. "Resolved 25 tickets/day with 98% accuracy" > "Resolved tickets efficiently."
3. **Action verbs:** Deployed, Automated, Optimized, Debugged, Designed, Migrated, Managed.
4. **Certifications section:** List prominently (top 3 if you have more). Include exam code and date.
5. **GitHub link:** If technical, include GitHub profile link prominently (ensure it's professional and current).
6. **One-page rule (until 3+ years exp):** Aim for 1–1.5 pages for entry-level.

### Interview Preparation (All Roles)

1. **STAR format:** Situation, Task, Action, Result — use for all behavioral questions.
2. **Technical prep:** Study cert exam objectives + hands-on labs (don't memorize; understand concepts).
3. **Company research:** Know their tech stack, recent news, 3 questions for them (culture, team structure, growth path).
4. **Mock interviews:** Practice with friend or on platforms like Pramp.
5. **Whiteboard/Live coding:** Expect it for developer roles. Practice on LeetCode (easy/medium) if coding role.

### Job Board Strategy

**Tier 1 (Largest reach):**
- Indeed
- LinkedIn

**Tier 2 (Niche + strong filtering):**
- Dice (networking, DevOps strong)
- Stack Overflow Jobs (developer roles)
- Built In (tech-forward companies, startups)

**Tier 3 (Underrated but valuable):**
- Wellfound (startup scene, equity transparency)
- Otta (European, but expanding; quality roles)
- Local recruiter cold outreach (LinkedIn DMs to recruiters in your city; often overlooked but effective)
- Hacker News "Who's Hiring" (monthly threads; small but high-quality postings)

**Government/Defense (if cleared or eligibile):**
- USAJobs
- ClearanceJobs

### First 90 Days: Universal Expectations

1. **Days 1–30 (Orientation):**
   - Company culture, tools, systems onboarding.
   - Shadow 1–2 senior teammates.
   - Complete compliance/security training.
   - **Goal:** Don't break production; ask questions freely.

2. **Days 31–60 (Hands-on Contribution):**
   - First ticket/task ownership (low-risk, well-scoped).
   - Regular 1-on-1s with manager; feedback cycles.
   - Learn escalation pathways.
   - **Goal:** Demonstrate you can own a task end-to-end.

3. **Days 61–90 (Independent Execution):**
   - Own 3–5 tickets/tasks independently.
   - Contribute to team documentation or tooling.
   - Identify one process improvement (propose it, implement if approved).
   - **Goal:** Prove you're ready for mid-level challenges; discuss next steps (specialization, advanced projects, promotion timeline).

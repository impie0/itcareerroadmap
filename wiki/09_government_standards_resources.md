# Free Government & Standards Body Publications for IT Professionals

A curated guide to critical, no-cost cybersecurity, risk management, and IT governance resources published by government agencies and recognized standards bodies.

---

## NIST (National Institute of Standards and Technology)

### NIST Cybersecurity Framework (CSF) 2.0
- **Status:** Free
- **URL:** https://nvlpubs.nist.gov/nistpubs/CSWP/NIST.CSWP.04162024.pdf
- **Why It Matters:** The industry-standard voluntary framework for managing cybersecurity risk. Version 2.0 (2024) adds Govern function and aligns with current threat landscape. Widely adopted across sectors and used by government, private industry, and critical infrastructure.
- **Who Should Read:** Risk managers, CISO offices, board-level governance; enterprise architects planning security programs.

### NIST SP 800-53 Rev. 5 (Security and Privacy Controls for Information Systems and Organizations)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-53/rev-5/final
- **Why It Matters:** Comprehensive control catalog for federal information security. 6+ categories covering access control, incident response, and supply chain risk. De facto standard adopted by many private organizations and critical infrastructure operators.
- **Who Should Read:** Security architects, compliance teams, anyone building or auditing security control baselines.

### NIST SP 800-171 Rev. 3 (Protecting Controlled Unclassified Information in Nonfederal Systems and Organizations)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-171/rev-3/final
- **Why It Matters:** Required for any contractor handling Controlled Unclassified Information (CUI) or supplying to DoD/federal agencies. Streamlined 14 control families; critical for defense industrial base and government contractors.
- **Who Should Read:** Defense contractors, federal suppliers, compliance officers in regulated supply chains.

### NIST SP 800-30 Rev. 1 (Guide for Conducting Risk Assessments)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-30/rev-1/final
- **Why It Matters:** Methodical risk assessment process: identify threats, vulnerabilities, likelihood, and impact. Pairs with RMF for enterprise risk governance.
- **Who Should Read:** Risk analysts, security engineers, system owners implementing risk management programs.

### NIST SP 800-37 Rev. 2 (Risk Management Framework)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-37/rev-2/final
- **Why It Matters:** Six-step lifecycle (Prepare, Categorize, Select, Implement, Assess, Authorize) for managing risk in federal systems. Aligns with CSF and used by federal agencies, contractors, and many enterprises.
- **Who Should Read:** Authorization officials, system security engineers, program managers.

### NIST SP 800-61 Rev. 2 (Computer Security Incident Handling Guide)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-61/rev-2/final
- **Why It Matters:** Incident response phases (Preparation, Detection/Analysis, Containment, Eradication, Recovery, Post-Incident Activities). Foundational for IR playbooks and SOC operations.
- **Who Should Read:** Incident responders, SOC analysts, IR planners, security operations teams.

### NIST SP 800-63-3 (Digital Identity Guidelines)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-63-3/final
- **Why It Matters:** Authentication and identity assurance levels (IAL, AAL, FAL). Guides passwordless, MFA, and identity verification practices in federal and commercial systems.
- **Who Should Read:** Identity architects, authentication engineers, access control designers.

### NIST SP 800-92 (Guide to Computer Log Management)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-92/final
- **Why It Matters:** Log sources, retention, and analysis for security monitoring. Essential for detection, forensics, and regulatory compliance.
- **Who Should Read:** SOC teams, security engineers, forensic analysts.

### NIST SP 800-115 (Technical Security Testing and Assessment Guide)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-115/final
- **Why It Matters:** Scanning, penetration testing, and security assessment methodologies for identifying vulnerabilities and control weaknesses.
- **Who Should Read:** Penetration testers, vulnerability assessors, security auditors.

### NIST SP 800-145 (A NIST Definition of Cloud Computing)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-145/final
- **Why It Matters:** Authoritative definition of cloud computing characteristics (on-demand, broad network access, resource pooling, rapid elasticity, measured service). Used globally in cloud contracts and architecture.
- **Who Should Read:** Cloud architects, procurement teams, anyone evaluating cloud services.

### NIST SP 800-190 (Application Container Security Guide)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-190/final
- **Why It Matters:** Docker, Kubernetes, and container best practices: image scanning, registry security, runtime monitoring. Critical as containers become ubiquitous.
- **Who Should Read:** DevOps engineers, container platform teams, application security architects.

### NIST SP 800-207 (Zero Trust Architecture)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-207/final
- **Why It Matters:** Blueprint for moving from perimeter-based to continuous trust verification. Increasingly adopted by enterprises and government to defend against advanced threats.
- **Who Should Read:** Network architects, identity teams, security operations strategists.

### NIST SP 800-218 (Secure Software Development Framework)
- **Status:** Free
- **URL:** https://csrc.nist.gov/publications/detail/sp/800-218/final
- **Why It Matters:** SSDF practices for government software procurement and supply chain security. Aligns with Executive Order 14028 on cybersecurity; becoming a market differentiator.
- **Who Should Read:** Software development teams, procurement security specialists, engineering leaders.

### NIST AI Risk Management Framework (AI RMF)
- **Status:** Free
- **URL:** https://airc.nist.gov/ai-risk-management-framework
- **Why It Matters:** Governance, mapping, measuring, and managing risks in AI/ML systems. Emerging standard for responsible AI deployment.
- **Who Should Read:** AI/ML engineers, governance teams, product managers deploying AI solutions.

### NIST Privacy Framework
- **Status:** Free
- **URL:** https://nvlpubs.nist.gov/nistpubs/CSWP/NIST.CSWP.01162024.pdf
- **Why It Matters:** Aligned with CSF; governs data handling, transparency, and individual rights. Critical for privacy engineering and compliance (GDPR, CCPA, etc.).
- **Who Should Read:** Privacy officers, legal teams, data engineers, compliance departments.

## CIS (Center for Internet Security)

### CIS Controls v8.1 (formerly Top 20 Critical Controls)
- **Status:** Free
- **URL:** https://www.cisecurity.org/cis-controls/
- **Why It Matters:** 18 prioritized, actionable security controls with implementation groups (IG1/IG2/IG3) based on organizational maturity. Used by thousands of organizations as a prescriptive baseline.
- **Who Should Read:** Security directors, control owners, anyone implementing a control framework.

### CIS Benchmarks
- **Status:** Free (membership unlocks some advanced features)
- **URL:** https://www.cisecurity.org/benchmarks/
- **Why It Matters:** Configuration hardening guides for operating systems, applications, cloud platforms (AWS, Azure, GCP). Specific, measurable, and widely used in audits and compliance assessments.
- **Who Should Read:** Systems administrators, infrastructure teams, security assessors.

### Multi-State Information Sharing & Analysis Center (MS-ISAC)
- **Status:** Free
- **URL:** https://www.cisecurity.org/ms-isac/
- **Why It Matters:** Threat intelligence, incident support, and security services for state/local government. Provides threat feeds and cyber situational awareness.
- **Who Should Read:** State/local government IT leadership, public sector CISOs.

## ENISA (EU Agency for Cybersecurity)

### ENISA Threat Landscape Annual Report
- **Status:** Free
- **URL:** https://www.enisa.europa.eu/threat-landscape
- **Why It Matters:** Annual assessment of cyber threats targeting Europe and critical infrastructure. Identifies emerging threats, attacker trends, and vulnerabilities.
- **Who Should Read:** Security strategists, threat intelligence teams, governance.

### ENISA Cloud Security Guidelines
- **Status:** Free
- **URL:** https://www.enisa.europa.eu/publications/cloud-security-guide
- **Why It Matters:** Advice on cloud service assessment, vendor selection, and security in cloud deployments. Aligns with NIS2 and GDPR requirements.
- **Who Should Read:** Cloud architects, procurement teams, organizations operating in EU.

### NIS2 Directive Guidance
- **Status:** Free
- **URL:** https://www.enisa.europa.eu/nis2
- **Why It Matters:** EU directive on network and information systems security (effective 2024). Mandatory for critical infrastructure and certain private companies; expanded scope compared to NIS1.
- **Who Should Read:** European organizations, CISOs, governance/compliance teams subject to NIS2.

## ISO/IEC Standards (Mostly Paid; Listed for Reference)

**Note:** Full ISO standards require purchase, but summaries, implementation guides, and mappings are often freely available from industry bodies and training organizations.

### ISO/IEC 27001 (Information Security Management System)
- **Status:** Paid (standard ~$140 USD)
- **URL:** https://www.iso.org/standard/27001
- **Why It Matters:** International standard for establishing and managing information security. Widely adopted for third-party certification and regulatory alignment.
- **Who Should Read:** ISMS practitioners, auditors, organizational governance.

### ISO/IEC 27002 (Code of Practice for Information Security Controls)
- **Status:** Paid (standard ~$200 USD)
- **URL:** https://www.iso.org/standard/27002
- **Why It Matters:** Detailed control guidance (14 domains) aligned with 27001. Practical implementation reference.
- **Who Should Read:** Security architects, control designers.

### ISO/IEC 27017 / 27018 (Cloud Security)
- **Status:** Paid
- **URL:** https://www.iso.org/standard/27017 and https://www.iso.org/standard/27018
- **Why It Matters:** Cloud-specific controls for IaaS/PaaS (27017) and PII in cloud (27018).
- **Who Should Read:** Cloud security teams, providers, consumers.

### ISO/IEC 27701 (Privacy Information Management)
- **Status:** Paid
- **URL:** https://www.iso.org/standard/27701
- **Why It Matters:** Privacy extension to 27001/27002. Aligns with privacy regulations.
- **Who Should Read:** Privacy engineers, data protection officers.

### ISO 31000 (Risk Management)
- **Status:** Paid
- **URL:** https://www.iso.org/standard/43170
- **Why It Matters:** Generic risk framework; often mapped to NIST RMF and CSF for cross-standard alignment.
- **Who Should Read:** Risk officers, governance teams.

## NSA / CISA (US Government Cybersecurity & Infrastructure Security Agency)

### CISA Known Exploited Vulnerabilities (KEV) Catalog
- **Status:** Free
- **URL:** https://www.cisa.gov/known-exploited-vulnerabilities-catalog
- **Why It Matters:** Real-time catalog of vulnerabilities actively exploited by threat actors. Used to prioritize patching and defense efforts.
- **Who Should Read:** Vulnerability managers, SOC teams, patch management teams.

### NSA Cybersecurity Information Sheets (CSIs)
- **Status:** Free
- **URL:** https://www.nsa.gov/what-we-do/cybersecurity-collaboration-center/
- **Why It Matters:** NSA threat intelligence and hardening guidance on specific technologies (e.g., Kubernetes, DNS filtering). High-signal threat intel.
- **Who Should Read:** Security engineers, infrastructure teams, threat assessment teams.

### CISA Secure by Design Publications
- **Status:** Free
- **URL:** https://www.cisa.gov/securebydesign
- **Why It Matters:** Industry guidance on building security into product design rather than bolting on later. Drives security culture in development organizations.
- **Who Should Read:** Software architects, product teams, vendor security programs.

### US-CERT Advisories & Alerts
- **Status:** Free
- **URL:** https://www.cisa.gov/news-events
- **Why It Matters:** Real-time alerts on emerging vulnerabilities and active cyber campaigns affecting critical infrastructure and the public.
- **Who Should Read:** Incident responders, threat intelligence teams, security operations.

### StopRansomware.gov
- **Status:** Free
- **URL:** https://www.stopransomware.gov/
- **Why It Matters:** Consolidated ransomware intelligence, techniques, indicators, and resources from FBI, CISA, Secret Service, and Interpol.
- **Who Should Read:** Ransomware defense teams, backup/recovery teams, incident responders.

## MITRE

### MITRE ATT&CK Framework
- **Status:** Free
- **URL:** https://attack.mitre.org/
- **Why It Matters:** The de facto adversary playbook: 100+ techniques mapped to threat actors, campaigns, and tools. Used for threat modeling, detection rule development, and red teaming.
- **Who Should Read:** Threat analysts, red teamers, detection engineers, security architects.

### MITRE D3FEND (Defensive Techniques)
- **Status:** Free
- **URL:** https://d3fend.mitre.org/
- **Why It Matters:** Inverse of ATT&CK: defensive countermeasures mapped to adversary techniques. Guides detection and mitigation strategy.
- **Who Should Read:** Detection engineers, purple teamers, security architects.

### MITRE CALDERA (Adversary Emulation Tool)
- **Status:** Free (Open Source)
- **URL:** https://github.com/mitre/caldera
- **Why It Matters:** Open-source platform for simulating adversary behavior and testing defenses. Built on ATT&CK; used for red team exercises and continuous validation.
- **Who Should Read:** Red teamers, security operations, purple team programs.

### MITRE CWE (Common Weakness Enumeration)
- **Status:** Free
- **URL:** https://cwe.mitre.org/
- **Why It Matters:** Catalog of ~900 software weaknesses; maps to CVEs and used for vulnerability prioritization and root cause analysis in application security.
- **Who Should Read:** Application security engineers, developers, penetration testers.

### MITRE CAPEC (Common Attack Pattern Expression Language)
- **Status:** Free
- **URL:** https://capec.mitre.org/
- **Why It Matters:** Abstraction of attack patterns used in threat modeling, red teaming, and risk assessment.
- **Who Should Read:** Threat modelers, architects, red team leads.

## OWASP (Open Worldwide Application Security Project)

### OWASP Top 10 (Web)
- **Status:** Free
- **URL:** https://owasp.org/www-project-top-ten/
- **Why It Matters:** The canonical list of most critical web application risks. Updated every 3-4 years; widely used in code review, testing, and AppSec programs.
- **Who Should Read:** Web developers, QA engineers, application security testers.

### OWASP API Security Top 10
- **Status:** Free
- **URL:** https://owasp.org/www-project-api-security/
- **Why It Matters:** API-specific risks (broken object-level auth, BOLA, injection, etc.). Critical as APIs become the primary attack surface.
- **Who Should Read:** API developers, backend engineers, API security testers.

### OWASP Mobile Security Top 10
- **Status:** Free
- **URL:** https://owasp.org/www-project-mobile-top-10/
- **Why It Matters:** Mobile app-specific risks (insecure storage, weak crypto, reverse engineering). Essential for iOS/Android teams.
- **Who Should Read:** Mobile developers, mobile security testers.

### OWASP ASVS (Application Security Verification Standard)
- **Status:** Free
- **URL:** https://owasp.org/www-project-application-security-verification-standard/
- **Why It Matters:** Comprehensive security requirements framework for web/mobile apps, organized by risk level (L1/L2/L3). Used in threat modeling and secure requirements.
- **Who Should Read:** Security architects, QA leads, secure code reviewers.

### OWASP SAMM (Software Assurance Maturity Model)
- **Status:** Free
- **URL:** https://owasp.org/www-project-samm/
- **Why It Matters:** Maturity model for building secure development practices. Covers governance, design, implementation, verification, and deployment.
- **Who Should Read:** AppSec program managers, development leadership.

### OWASP Cheat Sheets
- **Status:** Free
- **URL:** https://cheatsheetseries.owasp.org/
- **Why It Matters:** Quick-reference guides on specific security topics (authentication, cryptography, Docker, etc.) for developers and architects.
- **Who Should Read:** Developers, architects, anyone needing quick security guidance.

### OWASP ZAP (Zed Attack Proxy)
- **Status:** Free (Open Source)
- **URL:** https://www.zaproxy.org/
- **Why It Matters:** Dynamic application security testing (DAST) tool for finding vulnerabilities in web apps. Beginner-friendly; widely used in CI/CD pipelines.
- **Who Should Read:** QA automation engineers, security testers, developers.

## Industry-Specific Government & Standards

### HHS HIPAA Security Rule Guidance
- **Status:** Free
- **URL:** https://www.hhs.gov/hipaa/for-professionals/security/index.html
- **Why It Matters:** 18 CFR Part 164 Subpart C; defines security safeguards for Protected Health Information (PHI). Mandatory for healthcare covered entities and business associates.
- **Who Should Read:** Healthcare security teams, compliance officers, healthcare IT.

### PCI Security Standards Council
- **Status:** Free (core standards; some advanced docs paid)
- **URL:** https://www.pcisecuritystandards.org/
- **Why It Matters:** Payment Card Industry Data Security Standard (PCI-DSS), PA-DSS, P2PE. Mandatory for any organization handling payment cards.
- **Who Should Read:** Payment security teams, merchants, payment processors, compliance teams.

### FedRAMP (Federal Risk and Authorization Management Program)
- **Status:** Free
- **URL:** https://www.fedramp.gov/
- **Why It Matters:** Standardized approach for government cloud security authorization. Required for cloud services used by federal agencies.
- **Who Should Read:** Federal cloud service providers, government cloud architects, compliance teams.

### IRS Publication 1075 (Tax Information Security Guidelines)
- **Status:** Free
- **URL:** https://www.irs.gov/publications/p1075
- **Why It Matters:** Controls for handling tax return information by tax professionals, accountants, and preparers. Updated regularly with IRS safeguards.
- **Who Should Read:** Tax firms, accounting firms, IRS reporting entities.

### FFIEC IT Examination Handbook
- **Status:** Free
- **URL:** https://www.ffiec.gov/it-examination-handbook/
- **Why It Matters:** Guidance for financial institution IT security and audit. Covers business continuity, information security, and third-party risk for banks.
- **Who Should Read:** Financial services IT security, financial institution auditors.

## How to Use These Resources

### For Risk & Strategy
1. **Start:** NIST Cybersecurity Framework 2.0 → understand your five functions (Govern, Identify, Protect, Detect, Respond, Recover)
2. **Baseline:** CIS Controls v8.1 → implement controls by implementation group
3. **Assess:** NIST SP 800-30 → identify gaps
4. **Align:** Map to industry-specific standards (HIPAA, PCI, FedRAMP) if applicable

### For Incident Response & Detection
1. **Framework:** NIST SP 800-61 (Incident Handling)
2. **Intelligence:** CISA KEV Catalog → MITRE ATT&CK → NSA CSIs
3. **Tools:** MITRE CALDERA (testing) → OWASP ZAP (application testing)

### For Application Security
1. **Standards:** OWASP Top 10 → OWASP ASVS → OWASP Cheat Sheets
2. **Maturity:** OWASP SAMM (program building)
3. **Development:** Integrate NIST SP 800-218 (SSDF) into CI/CD

### For Cloud Security
1. **Framework:** NIST SP 800-145 (definition) → CSA CCM (controls)
2. **Configuration:** CIS Benchmarks (cloud-specific)
3. **Vendor:** CSA STAR Registry → FedRAMP (if government procurement)

### For Governance & Compliance
1. **Policy Foundation:** ISO 31000 (risk) mapped to NIST RMF
2. **Controls:** NIST SP 800-53 or ISO 27001/27002 (mapping available free)
3. **Privacy:** NIST Privacy Framework + GDPR/CCPA mapping
4. **Audit:** CIS Benchmarks or ISO 27001 assessment controls

## Key Takeaways

- **NIST publications** are the most comprehensive and globally trusted; prioritize SP 800-53, CSF 2.0, and RMF.
- **CIS Controls** provides actionable, implementation-group-based guidance; easier to operationalize than NIST for many organizations.
- **OWASP** is essential for development and application security teams; free and community-driven.
- **MITRE ATT&CK/D3FEND** should inform threat modeling, detection rules, and red team exercises.
- **Industry-specific standards** (HIPAA, PCI-DSS, FedRAMP) are often mandatory; treat as non-negotiable baselines.
- **Regional/national standards** (NCSC, ANSSI, ACSC) provide localized guidance and regulatory alignment for those geographies.

All resources listed are fully accessible and free to download. No subscriptions or credentials required.
